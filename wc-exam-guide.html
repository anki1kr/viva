Bluetooth, Piconets & Scatternets ⭐ Repeated 3×

Bluetooth structures close-range devices using Frequency Hopping Spread Spectrum (FHSS) to prevent interference:

  • Piconet: A local network containing up to 8 active devices: 1 Master (boss) coordinating synchronization + up to 7 Slaves.
  • Scatternet: Formed when a slave in one piconet acts as a bridge (or master) in an adjacent piconet, chaining them together.

Frequency Reuse & Cluster Math ⭐ Guaranteed 10M

Frequency Reuse Low-power antennas allow the same radio channels to be reused in faraway cells. Nearby cells are grouped into a **Cluster (N)**. Each cell inside a cluster gets a distinct set of channels.
Frequency Reuse Formulas
• Q = D / R = sqrt(3N) (Co-channel reuse ratio)
• Standard N sizes = i² + ij + j² (Common sizes: 3, 4, 7, 9, 12)

A smaller cluster size $N$ increases overall network capacity but reduces the physical distance between co-channel cells, increasing **Co-channel Interference (CCI)**.

Cellular Handoff MAHO / MCHO / NCHO ⭐ Repeated 6×

A handoff transfers an active call from one radio channel/base station to another as the subscriber crosses cell boundaries:

Handoff StrategyWho measures?Who decides?Standard System
NCHO (Network-Controlled)Base StationMSC brain switch1G Analog (Slow, ~10s lag)
MAHO (Mobile-Assisted)Mobile phone measurementsMSC switch2G GSM (Fast, ~1s lag)
MCHO (Mobile-Controlled)Mobile phoneMobile phoneDECT / Private WLAN (Instant)
Practical Considerations & Ping-Pong effect To prevent the **Ping-Pong Effect** (rapid, repetitive handoffs back and forth at the cell boundary), networks use a **Handoff Hysteresis** buffer: a handoff is triggered only if the new station's signal exceeds the old one by a specific threshold (e.g. 3 dB to 6 dB).

Unit III

03Multiple Access &
Spread Spectrum

Sharing the sky safely using time slots, frequency slots, and unique code mathematics.

Multiple Access: FDMA, TDMA, CDMA, SDMA ⭐ Repeated 4×

Access Schemes Mnemonic • FDMA: Separate users by frequency (one channel per user).
• TDMA: Separate users by time slots (all share one frequency in turns).
• CDMA: Separate users by orthogonal mathematical codes (all share same band simultaneously).
• SDMA: Separate users by pointing narrow spatial antenna beams.
Near-Far Problem in CDMA If a mobile user is physically close to the base station, its signal will completely drown out distant users on the same frequency. CDMA resolves this by enforcing **Strict Power Control** loops (checking and adjusting mobile power 800 times a second).

Spread Spectrum: DSSS vs FHSS ⭐ Repeated 2×

Spread Spectrum (SS) Intentionally spreading a narrow-band data signal across a much wider radio frequency band. This reduces interference susceptibility and enables anti-jamming.
  • DSSS (Direct Sequence): Multiplies data bits by a fast Pseudo-Noise (PN) chip code. Low probability of intercept. Used in GPS.
  • FHSS (Frequency Hopping): Carrier frequency hops rapidly across many channels according to a PN sequence. Outstanding anti-interference. Used in Bluetooth.
Unit IV

04Wireless Networking &
Routing Architectures

Network routing, ISDN architectural channels, and intelligent systems.

ISDN & AIN Architectures

  • ISDN (Integrated Services Digital Network): Integrates voice and data over standard phone lines. Channels: B Channel (Bearer, 64 kbps, carries voice/data) and D Channel (Delta, 16 kbps, handles out-of-band signaling).
  • AIN (Advanced Intelligent Network): Decouples call-routing databases from physical switches. Allows rapid deployment of custom services (toll-free numbers, call forwarding) without updating local exchange software.

Explain Grade of Service (GoS) in cellular networks. What is the standard target?
GoS is a metric defining the probability that a call is blocked during peak busy hours. The standard carrier target is GoS ≤ 2% (meaning ≤ 2 out of 100 calls are blocked).

Differentiate hard handoff and soft handoff.
Hard Handoff: "Break-before-make" — drops connection with old base station before connecting with the new one. Soft Handoff: "Make-before-break" — retains connection with both stations simultaneously (requires CDMA).

CDMA is a multiplexing access scheme where multiple users share the same frequency band simultaneously, separated by unique orthogonal code sequences. Processing Gain formula: G_p = Chip Rate / Data Rate.

List the three GSM security authentication triplets generated by the AuC.
The AuC security triplet consists of: (1) RAND (Random number), (2) SRES (Signed Response), and (3) Kc (Cipher key).

Define scatternet in Bluetooth.
A scatternet is formed when multiple Bluetooth piconets are linked together, sharing at least one common bridge device acting as a slave in one piconet and a master/slave in another.

Explain the near-far problem in TDMA and CDMA systems.
The near-far problem occurs when a mobile unit physically close to the base station transmits at a power level that completely drowns out weak signals from distant users. Enforced power control loops are required to resolve this in CDMA.

Pure ALOHA maximum throughput = 18.4% (at G = 0.5). Slotted ALOHA maximum throughput = 36.8% (at G = 1.0) because forced clock-slot boundaries halve the collision vulnerability window.

Q2Unit 1 Highlight15M
Describe the GSM System Architecture in detail with a neat block diagram. Detail the role of Mobile Station (MS), Base Station Subsystem (BSS), and Network Switching Subsystem (NSS) with a table.
Introduction

GSM (Global System for Mobile communications) is the canonical 2G digital cellular standard standardised by ETSI in 1990 (originally Groupe Spécial Mobile). Its architecture decouples components into four main subsystems connected by standard interfaces (Um, Abis, A), ensuring multi-vendor hardware compatibility.

The Three Subsystem Groups
Subsystem GroupNetwork ModuleRole & Function
Mobile Station (MS)SIM Card + HandsetSIM card stores the IMSI, authentication keys, and user profile data, insulating device hardware from subscription status.
Base Station Subsystem (BSS)BTS (Base Transceiver Station)Houses radio transmitters/receivers, handles modulation/demodulation and the Um air interface.
BSC (Base Station Controller)Manages radio channel allocation, frequency hopping, and local handover execution across multiple BTS towers.
Network Switching Subsystem (NSS)MSC (Mobile Services Switching Centre)The core routing engine. Coordinates call setup, routing switching, billing, and handoffs across different controller zones.
HLR (Home Location Register)The permanent master database storing subscriber profiles, billing details, and current location registers.
VLR (Visitor Location Register)A dynamic local cache database storing temporary profiles of guest mobile users currently active in the MSC zone.
AuC (Authentication Centre)Generates RAND, SRES, and Kc keys using A3/A8 algorithms, providing the MSC with security verification.
Operations Support Subsystem (OSS)OMC (Operations & Maintenance Centre)Network-wide monitoring, fault management, configuration and performance management — keeps the whole GSM network healthy.
Examiner Trap — HLR vs VLR Do not mix up HLR and VLR. HLR is the permanent master database owned by your home network (one per operator). VLR is a temporary local registry cache at each MSC holding profiles of guest mobiles currently in that zone — it avoids redundant queries back to the home HLR. Mixing them up costs 2-3 marks every time.
Modern Relevance

Although 5G NR is the headline 2026 wireless standard, GSM (2G) remains the universal fallback layer for voice. Major carriers (Vodafone, Airtel, T-Mobile) still operate GSM cores for VoLTE fallback and IoT M2M devices. The MS/BSS/NSS partition introduced by ETSI in 1990 also shaped the 3GPP architecture for UMTS, LTE EPC (with MME/HSS replacing MSC/HLR) and 5G SBA — proving the decoupling principle still rules cellular design.

Conclusion

GSM's four-subsystem layered architecture remains the textbook reference for cellular system design: clean separation between handset, radio access, switching, and operations enables vendor interoperability, seamless mobility, and roaming. Mastering MS/BSS/NSS/OSS and the AuC security triplet unlocks every subsequent generation question (3G/4G/5G).

Q3Unit 2 Numerical15M
A total bandwidth of 25 MHz is allocated to a cellular system with N=7 frequency reuse cluster. If each simplex channel is 25 kHz (duplex pair = 50 kHz), and 1 MHz is reserved for control channels:
(a) Find the total duplex channels in the system.
(b) Compute voice & control channels per cell.
(c) If cell radius R = 2 km, find co-channel reuse ratio Q and co-channel cell distance D.
Introduction

Frequency reuse is the cornerstone of cellular capacity, formalised by V.H. MacDonald (Bell Labs, 1979) in the seminal AMPS design paper. Same frequency channels are spatially reused at cells separated by a co-channel distance D, organised into clusters of size N where N = i² + ij + j² (allowed N: 3, 4, 7, 9, 12, 13…).

(a) Total Duplex Channels
Step 1
Compute duplex channel width: each call requires one uplink and one downlink simplex channel. Duplex size = 2 × 25 kHz = 50 kHz = 0.05 MHz.
Step 2
Compute total system duplex channels: Total Channels = 25 MHz / 0.05 MHz = 500 duplex channels.
(b) Voice & Control Channels per Cell
Step 3
Compute system-wide control channels from reserved bandwidth: 1 MHz / 0.05 MHz = 20 control channels.
Step 4
Compute system-wide voice channels: 500 total − 20 control = 480 voice channels.
Step 5
Compute voice and control channels per cell (using cluster size N = 7):
• Voice channels per cell = 480 / 7 ≈ 68.57 → 68 channels per cell (rounded down).
• Control channels per cell = 20 / 7 ≈ 2.86 → 3 channels per cell (rounded up).
(c) Reuse Ratio Q & Co-channel Distance D
Step 6
Compute co-channel reuse ratio Q: Q = sqrt(3 N) = sqrt(21) ≈ 4.58.
Step 7
Compute co-channel cell distance D (using cell radius R = 2 km): D = Q × R = 4.58 × 2 km = 9.16 km.

Answers summary: (a) 500 duplex channels, (b) 68 voice & 3 control channels/cell, (c) Q ≈ 4.58, D ≈ 9.16 km.

Examiner Trap — D/R ratio The most common mis-step is computing D = √(3N) without the × R factor — Q is dimensionless (D/R), but the actual co-channel distance D = Q × R has units of km. Drop the multiplication and you lose 1.5–2 marks. Also: when channels-per-cell isn't an integer, voice channels round down (no half-channel for voice), control channels round up (you can't have fractional signalling).
Modern Relevance

While classic N=7 reuse was the 2G GSM staple, 4G LTE and 5G NR use universal reuse (N=1) with fractional frequency reuse (FFR) and soft frequency reuse (SFR) — interior cell users get the full band, edge users get orthogonal sub-bands. The cluster geometry survives in cell-planning tools like Atoll and ICS Telecom.

Conclusion

Frequency reuse remains the most powerful capacity multiplier in any spatially-bounded radio system: by halving the cluster size you double the system capacity, at the cost of tighter interference management. Modern networks push this principle to its physical limit with N=1 + interference coordination.

Q4Unit 2 Highlight15M
Compare MAHO, MCHO, and NCHO handoff strategies. Discuss the practical handoff considerations: hysteresis, dwell time, guard channels, and the ping-pong effect. Trace the 4-stage handoff sequence.
Introduction

A handoff (handover) transfers an active call from one radio channel/base station to another without breaking the user session, as the subscriber crosses a cell boundary. The three control strategies — NCHO, MAHO, MCHO — were progressively standardised by ITU-T and ETSI through the 1990s as cellular evolved from 1G analog to 2G/3G digital.

Handoff Strategies Comparison
Handoff StrategyMeasurement CollectionSwitching Decision MakerStandard Implementation
NCHO (Network-Controlled)Base Station measurementsMSC central switch1G Analog (Very slow, 10s latency)
MAHO (Mobile-Assisted)Mobile phone gathers signal dataMSC switch2G GSM / 3G (Fast, 1s latency)
MCHO (Mobile-Controlled)Mobile phone measurementsMobile phone itselfDECT / Private WLAN (Instant)
Practical Handoff Considerations
  • Hysteresis: A margin by which the new candidate station's signal must exceed the current station's signal to trigger handover (prevents accidental swapping due to brief fading).
  • Dwell Time: The continuous duration over which a mobile remains in the boundary zone before handoff initiates (avoids sudden spikes).
  • Guard Channels: Reserving a pool of channels in each cell exclusively for incoming handovers (prevents call drops).
  • Ping-Pong Effect: Rapid, repetitive back-and-forth handovers at the cell boundary caused by fluctuations. Prevented by combining hysteresis + dwell time.
The 4-Stage Handoff Sequence
1
Measure Phase: MS continuously measures downlink signal strengths (RSS) from neighbouring base stations and reports to current BSC.
2
Decide Phase: BSC detects that RSS falls below threshold (e.g. -100 dBm) and hysteresis is met. Initiates handoff request to MSC.
3
Execute Phase: MSC locates new target BS, reserves a traffic channel, and instructs current BS to send a handover command to MS.
4
Complete Phase: MS retunes to the new frequency, establishes link with the new BS, and old BS releases the previous channel back to the pool.
Examiner Trap — MAHO vs NCHO The classic confusion: NCHO uses base station measurements (mobile is dumb, network is smart) — used in slow 1G AMPS. MAHO uses mobile measurements but the MSC still decides — used in 2G GSM. Don't merge the two; an examiner who reads "mobile measures and decides" for MAHO marks it wrong (that's MCHO, used in DECT).
Modern Relevance

4G LTE and 5G NR replace classical hard handover with soft handover (CDMA2000/UMTS) and make-before-break X2/Xn handover (LTE/NR), where the mobile is briefly connected to both source and target cells — eliminating call drops. Future 6G architectures (3GPP Rel-19+) target zero-interruption mobility using ML-driven handover prediction.

Conclusion

Handoff sits at the heart of cellular mobility — the trade-off between measurement accuracy, decision latency, and signalling overhead has driven the entire evolution from NCHO → MAHO → MCHO → soft handover. Coupled with hysteresis, dwell time, and guard channels, modern networks achieve sub-50 ms break times.

Q5Unit 2 Highlight15M
Contrast Cell Splitting, Cell Sectoring, and Microzone cell designs in detail. Explain how each increases coverage or capacity, with a detailed comparison table.
Introduction

As cellular subscriber density grows, operators must scale capacity without acquiring expensive new frequency spectrum. Three architectural scaling options — cell splitting, cell sectoring, and microzone deployment — were formalised in Lee's seminal cellular design literature (Bell Labs, 1980s) and remain the textbook capacity-extension toolkit.

The Capacity Upgrades Comparison
AxisCell SplittingCell SectoringMicrozone Cell
MechanismSubdivides congested cell into 4 micro-cells, reducing antenna radius by R/2.Replaces omni-directional antenna with 3 directional sector antennas (120°).Distributes low-power active zone antennas within a single cell.
Capacity Gain4× capacity per area (since more channels repeat in same space).Medium gain (reduces co-channel interference $S/I$, enabling smaller cluster size N).High local capacity (no handoffs between zones).
CCI ImpactNo change in $S/I$ ratio.Reduces CCI by limiting interference from omni to only 2 directions.Drastically reduces CCI; power is strictly localized to active zones.
Key DrawbackHigh hardware cost. Increases handover signaling load dramatically.Reduces trunking efficiency by splitting channel sets into isolated groups.High cabling/fiber installation cost between zone antennas and base station.
Trunking & Handoff Trade-offs
  • Cell Splitting is the only way to achieve infinite raw capacity growth, but it requires new tower acquisitions.
  • Cell Sectoring is a cheap antenna swap at the top of an existing tower, but it reduces the size of channel pools.
  • Microzone avoids the handover storm of cell splitting by keeping all low-power zone antennas mapped under a single, central base station controller.
Examiner Trap — Over-sectoring A common over-engineering mistake: dividing a cell into 6 or 12 sectors instead of the standard 3. While this further reduces co-channel interference, it multiplies handoff load (every user now hands off when crossing each sector boundary inside the same cell) and dramatically degrades trunking efficiency (Erlang-B blocks faster on smaller channel groups). Stick to 3-sector 120° as the practical sweet spot unless dense urban traffic demands otherwise.
Modern Relevance

2026 networks heavily favour 5G small-cell densification — a refinement of cell splitting where pico/femto cells (10–100 m radius) are deployed in malls, stadiums, and street-furniture. Massive-MIMO + 3D beamforming in 5G NR effectively realises adaptive sectoring (256 spatial beams replace 3 fixed sectors), and distributed antenna systems (DAS) productise the microzone concept across enterprise indoors and venue stadiums.

Conclusion

Cell splitting, sectoring, and microzone form the cellular capacity playbook — each trades capital cost, signalling overhead, and interference geometry differently. Modern small-cell + Massive MIMO designs blend all three approaches, but the underlying physics — same-channel reuse at safe distance — has not changed since Bell Labs 1979.

Q6Unit 3 Highlight15M
Compare FDMA, TDMA, CDMA, and SDMA multiple access techniques in detail with a comparison table. Discuss the code separation concept in CDMA and show a worked bit-level despreading example.
Introduction

Multiple access defines how multiple subscribers share a single physical wireless channel. The four canonical schemes — FDMA, TDMA, CDMA, SDMA — partition along orthogonal axes: frequency, time, code, space. FDMA traces back to Bell Labs AMPS (1983); CDMA was pioneered by Andrew Viterbi at Qualcomm (1989, IS-95); SDMA is the latest beam-forming era championed by Massive-MIMO 5G NR.

Multiple Access Schemes Comparison
SchemeUser Separation AxisPrimary AdvantageMajor ChallengeStandard Use
FDMAFrequency (Isolated channels)Simple analog hardware logic. No clock sync.Spectral waste. Guard bands needed.1G AMPS
TDMATime slots (Shared frequency)Flexible data rates. Low-power transmitters.Strict synchronization required. Guard times.2G GSM
CDMAMathematical Codes (PN sequences)Infinite soft capacity. High privacy.Near-far power control limits.3G WCDMA
SDMASpatial coordinates (Directed beams)Massive frequency reuse in same cell.High smart antenna array complexity.5G Massive MIMO
CDMA Code Separation & Orthogonality

In CDMA, every active mobile is assigned a unique Pseudo-Noise (PN) code vector from a set of **orthogonal Walsh codes**. The inner product of any two distinct codes $C_A \cdot C_B = 0$, while $C_A \cdot C_A = N$ (where $N$ is chip length). This enables multiple signals to share the same frequency band simultaneously without interfering.

Worked Bit-Level CDMA Example

User A has code C_A = [1, -1, 1, -1]. User B has code C_B = [1, 1, -1, -1]. Note: $C_A \cdot C_B = 1(1) - 1(1) - 1(1) + 1(1) = 0$ (orthogonal).

Step 1
User A wants to send bit 1 (represented as $+1$) ⇒ Transmits S_A = +1 × C_A = [1, -1, 1, -1].
Step 2
User B wants to send bit -1 ⇒ Transmits S_B = -1 × C_B = [-1, -1, 1, 1].
Step 3
Combined over-the-air signal: S = S_A + S_B = [0, -2, 2, 0].
Step 4
Receiver A recovers A's bit by taking the inner product of S and $C_A$:
Result = S · C_A = 0(1) + (-2)(-1) + 2(1) + 0(-1) = 0 + 2 + 2 + 0 = +4.
Divide by chip length N = 4: +4 / 4 = +1 (Bit 1 successfully recovered!).
Examiner Trap — TDMA time sync TDMA's biggest practical Achilles heel is tight time synchronisation: every mobile must fire its burst within a ±¼-symbol guard time of its assigned slot, or it collides with neighbours. GSM mobiles correct this via Timing Advance commands from the BTS. Forgetting to mention sync & guard time for TDMA loses 2 marks — examiners check for it specifically.
Modern Relevance

4G LTE and 5G NR use OFDMA (Orthogonal Frequency Division Multiple Access) — a refined FDMA where subcarriers are mutually orthogonal, eliminating guard bands. The uplink in LTE uses SC-FDMA for lower PAPR. Massive-MIMO in 5G is essentially SDMA at scale (64–256 antennas), and the next 6G generation (3GPP Rel-20+) explores non-orthogonal multiple access (NOMA) — multiple users on the same time-frequency resource separated by power level.

Conclusion

FDMA, TDMA, CDMA, and SDMA represent the four orthogonal partitionings of the wireless medium; CDMA's mathematical orthogonality unlocked the 3G capacity revolution, and SDMA's spatial dimension now powers 5G Massive-MIMO. Real systems combine multiple axes (OFDMA + SDMA + power-domain NOMA) to squeeze maximum capacity from finite spectrum.

  • Sectoring: directional sector antennas. Less CCI
  • Microzone: Low-power local zone antennas. No HO

  • FDMA: Separate users by Frequency channels
  • TDMA: Separate users by Time slots (turns)
  • CDMA: Separate users by unique PN Codes
  • SDMA: Separate users by Directed Beams
Spread Spectrum
  • DSSS: Multiplies bits by fast PN chip code
  • FHSS: Hops carrier frequency rapidly
  • Anti-jamming + Privacy + Multipath resistance
GoS Blocked Target
  • GoS = Probability a call is blocked at peak
  • Standard carrier target = GoS ≤ 2%
  • Calculated using Erlang-B formulas
ISDN Channels

  • D Channel: Delta, 16 kbps, out-of-band signaling
  • AIN: Decouples call-routing from switches
