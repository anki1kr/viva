1.1 Evolution of Wireless Communications always 1.5–10 M

Definition Wireless communication is the transfer of information between two or more points without an electrical conductor, using electromagnetic waves (radio, microwave, infrared, or light) as the carrier medium.

The field has grown across five generations over roughly five decades, with each generation broadly increasing data rate by an order of magnitude while shrinking cell size and tightening spectral efficiency.

1.4 CDMA — 3G Wireless Network ⭐ 10M

Definition CDMA (Code Division Multiple Access) is a spread-spectrum multiple-access technique where every user transmits over the same frequency at the same time, distinguished only by a unique orthogonal spreading code (PN sequence). The receiver decodes its signal by correlating with the known code; other users appear as low-level noise.
CDMA basics Chip rate Rc = code chips per second Data rate Rb = user info bits per second Processing gain Gp = Rc / Rb = BWSS / BWdata For IS-95 Rc = 1.2288 Mcps · Rb ≈ 9.6 kbps → Gp ≈ 128 (21 dB)

Areas where CDMA is used (BCA25-2a literal ask)

  • Cellular telephony — IS-95 (cdmaOne), CDMA2000 1xRTT, EV-DO (3G), WCDMA inside UMTS
  • GPS — each satellite has a unique Gold code; receiver correlates to find pseudo-range
  • Military — anti-jam, low-probability-of-intercept comms (its original use)
  • Wi-Fi (802.11b) — direct-sequence variant
  • Bluetooth — frequency-hopping variant (FHSS)
  • Satellite phones — Globalstar, ICO
Three killer advantages (1) Soft handoff — call held by 2+ base stations simultaneously, no hard cut-over → no dropped calls. (2) Universal frequency reuse — K = 1, every cell uses the same band → spectral efficiency 5–8× FDMA/TDMA. (3) Inherent secrecy — without the code, the signal looks like white noise.
Near-far problem (asked separately as 1.5-M short Q) A nearby strong transmitter swamps a distant weak one because both share the same band → CDMA must use fast closed-loop power control (800 Hz in IS-95) keeping every received signal at the same level. This is why "near/far effect on TDMA" (BCA25-1h) is a trick: TDMA is largely immune because users are separated in time, not code.

1.5 4G Wireless Networks 1.5M list-style

Definition 4G (Fourth Generation) is an all-IP packet-switched mobile broadband system standardised by 3GPP as LTE and LTE-Advanced (peak ≥ 100 Mbps mobile / 1 Gbps stationary), and by IEEE as WiMAX (802.16e). It uses OFDMA on the downlink, SC-FDMA on the uplink, and 2×2/4×4 MIMO.

Four advantages of 4G (BCA25-1b verbatim list)

  1. High data rate — 100 Mbps mobile, 1 Gbps stationary → HD video streaming, large-file downloads.
  2. Low latency & All-IP core — <50 ms round-trip → enables VoLTE, real-time gaming, video calls.
  3. Spectral efficiency via OFDMA + MIMO — multiple antennas exploit multipath instead of fighting it.
  4. Seamless inter-RAT mobility — handover to 2G/3G/Wi-Fi without dropping; one core for voice + data.
Bonus for higher marks Add SDR-friendly air interface, flat IP architecture (no MSC, only eNodeB + EPC), and QoS classes (QCI 1–9). These three lines move a 1.5-M answer into "examiner-pleasing 2.5-M territory" if the answer slot is generous.

1.6 Wireless in Local Loop (WLL) ⭐ 7.5M

Definition WLL (also Radio-in-the-Loop / Fixed Wireless Access) replaces the copper "last-mile" between the local telephone exchange and the subscriber with a radio link, so a fixed-location customer accesses PSTN services via a small antenna instead of a wired drop.

Architecture (one-line per block)

  • Switching Function (SF) — local exchange interfacing PSTN / ISDN
  • WLL Controller (WLLC) — manages radio resources, billing, OAM
  • Base Station (BS) — radio transceiver covering a sector / cell
  • Subscriber Unit (SU) — fixed customer-premises radio + telephone jack (RJ-11)
  • Air interface — typically TDMA/CDMA in 1.9 / 2.4 / 3.5 GHz licensed bands
Why WLL beats wired in developing markets Trenching copper across a rural Indian district costs ~₹50,000/km; a WLL BTS covers 5 km at a tenth of the cost. Roll-out time drops from months to weeks. This is the canonical "developing-country last-mile" answer for the 7.5-M variant.
Compression hint 5-mark version → drop OAM details, keep 4-block diagram + 3 lines on each. 7.5-mark version → add 2 advantages (cost, deployment speed) and 1 limitation (lower bandwidth vs FTTH).

1.8 Bluetooth, Personal Area Networks & Scatternet ⭐ 8M (BCA25-5a)

Definition Bluetooth (IEEE 802.15.1) is a short-range (1–100 m), low-power wireless standard in the 2.4 GHz ISM band, using Frequency-Hopping Spread Spectrum (FHSS) at 1600 hops/sec across 79 (Classic) or 40 (BLE) channels. It forms ad-hoc Personal Area Networks (PANs) with a master/slave star topology.

How Bluetooth works (the 8-mark answer in 5 lines)

  1. Two devices discover each other via an INQUIRY message and 4096-pattern inquiry hops.
  2. One becomes master, the other slaves to it; up to 7 active slaves form a piconet.
  3. The master's clock + BD_ADDR seed the FHSS sequence; all members hop in lock-step (1600 hops/s).
  4. Communication uses TDD time slots (625 µs each) with optional ACL (data) or SCO (voice) links.
  5. Multiple piconets sharing a slave form a scatternet — the bridge slave time-divides between piconets.

Scatternet (BCA25-1e literal 1.5-M ask) A scatternet is the interconnection of two or more piconets that share a common device (the "bridge"). The bridge slave time-multiplexes between the two piconets, enabling larger ad-hoc networks while preserving the 7-slave-per-piconet limit.

2.1 Spectrum Allocation 5–8M

Definition Spectrum allocation is the process by which a national or international regulator (ITU-R globally, WPC/TRAI in India, FCC in the USA) assigns specific frequency bands to specific services (cellular, broadcasting, satellite, defence) and licences them to operators for a fixed period, usually via auction.

Why spectrum is finite (and why it's regulated)

  • Physical limit — usable mobile spectrum sits between ~30 MHz and ~6 GHz (recently ~24–60 GHz mmWave). Below 30 MHz: ionospheric reflection ⇒ unreliable cellular. Above 60 GHz: oxygen absorption ⇒ short reach.
  • Interference — two unlicensed transmitters on the same band collide; regulation prevents the tragedy-of-the-commons.
  • Revenue — auctions monetise a national resource (e.g., India's 5G auction raised ₹1.5 lakh crore in 2022).

2.2 Basic Cellular System & Performance Criteria ⭐ 5–8M · 4× repeat

Definition A cellular system divides the service area into many small hexagonal cells, each served by its own low-power base station; cells in a cluster use unique frequencies and the same frequency set is reused in cells far enough apart that co-channel interference is acceptable.

Why hexagonal cells (BCA25-1c verbatim)

  1. No gaps, no overlaps — only three regular polygons tessellate the plane: triangle, square, hexagon. The hexagon is closest to a circle ⇒ minimum overlap with adjacent cells.
  2. Maximum coverage per base station — for a given radius R, hexagon area = (3√3/2)·R² ≈ 2.598 R², larger than square (2R²) but using the same number of BSs.
  3. Equidistant centres — six neighbours all at distance √3·R, which simplifies co-channel reuse maths.
  4. Realistic propagation shape — omni-antennas radiate roughly circularly; the hexagon is the most "circle-like" tessellating shape.

Performance criteria of a cellular system (asked 4× — BCA25-2b, 24BTECH-7b, 23BTECH-4b, 7BTECH-various)

2.4 Frequency Reuse ⭐⭐ 7× repeat across 6 papers

Definition Frequency reuse is the design principle of allocating the same set of channel frequencies to multiple, geographically separated cells in a cellular network. Cells using the same frequency are called co-channel cells. The separation is engineered so that co-channel interference stays below an acceptable level.

2.5 Channel Assignment Strategies ⭐ FCA vs DCA 4× repeat

Definition Channel assignment is the policy by which the network decides which physical channel (frequency / timeslot / code) a new call gets when it arrives in a cell. Two main families: Fixed (FCA), Dynamic (DCA), with Hybrid and Borrowing as variants.

Variants you must name

  • Borrowing strategy (23BTECH-1j literal 1.5-M) — a hybrid: cell uses its own FCA channels first; if exhausted, "borrows" an idle channel from a neighbour that has free capacity. The borrowed channel is locked in the donor cell to prevent co-channel interference. Used in older AMPS networks.
  • Hybrid Channel Assignment (HCA) — split pool: A% fixed + B% dynamic. Captures both worlds.
  • Channel re-arrangement — when a call ends, the system can move ongoing calls to compact assignments and reduce future blocking.

2.6 Handoff / Handover Strategies ⭐⭐ 6× repeat — almost guaranteed

Definition Handoff (US) / Handover (Europe) is the process of transferring an ongoing call or data session from one base station / channel to another while maintaining the conversation. It is triggered by deteriorating signal quality — typically as a subscriber moves between cells.

Triggers — what makes a handoff happen?

  • RSS (Received Signal Strength) falls below threshold
  • BER (Bit Error Rate) rises above limit
  • Distance / propagation delay exceeds the cell radius
  • Traffic balancing — the MSC may force a handoff to offload a congested cell

Hard vs Soft Handoff (IT25-1d 1.5-M)

Hard handoff (break-before-make) The mobile disconnects from the old base station before connecting to the new one. Brief audio gap (~50–100 ms). Used in FDMA/TDMA (AMPS, GSM) where the two BSs are on different frequencies — the mobile cannot tune both at once.
Soft handoff (make-before-break) The mobile maintains simultaneous connections with two or more base stations during the transition. Possible only in CDMA because all BSs share the same frequency — the mobile combines (rake-receives) signals from both for diversity gain. No call drop, no audio gap.
Common pitfalls (1) Threshold too low → ping-pong handoffs between two BSs. Cure: use hysteresis margin (e.g., 6 dB) and dwell timer. (2) Soft handoff is NOT possible in GSM — many students mix this up. (3) NCHO is obsolete with GSM onwards; don't claim modern networks use it.

2.8 Tracking · Grade of Service · Blocked-Call Models 1.5M definition

Definition · GoS Grade of Service (GoS) is the probability that a call request is blocked or experiences a delay greater than a specified value during the busy hour. Typically expressed as a probability (e.g., GoS = 0.02 means 2 calls per 100 are blocked at peak).
Erlang B (blocked-calls-cleared) B(C, A) = (AC/C!) / Σk=0..C Ak/k! A = offered traffic (Erlangs) · C = number of channels B = blocking probability (= GoS for blocked-cleared systems) Used when a blocked call is dropped (no queue) — typical for voice.

Three blocked-call models you must name

  1. Blocked-Calls-Cleared (BCC) / Erlang B — blocked caller hangs up; classic for cellular voice. Memoryless arrivals (Poisson) + exp holding times.
  2. Blocked-Calls-Delayed (BCD) / Erlang C — blocked caller waits in queue; used for call-centre paging and 1G dispatch.
  3. Blocked-Calls-Held (BCH) — blocked call holds the channel for its full duration but cannot actually communicate; rare, used for analytical comparison.
Blocked-call delay system (24BTECH-1a literal) = Erlang C (BCD) — a blocked call is held in queue until a channel becomes free; service quality is measured by average waiting time rather than blocking probability.
Tracking ≈ "where is the mobile right now?" — handled by paging, periodic Location-Area updates and HLR/VLR sync. Add a 2-line mention if a "tracking + GoS" question is combined.

2.9 Cell Splitting ⭐ 5–7.5M

Definition Cell splitting is the process of subdividing a congested cell into smaller cells (microcells), each with its own base station and reduced transmit power. The new cells preserve the original frequency-reuse pattern but multiply capacity per unit area.
Capacity gain If original cell radius = R, new = R/2, then area scales by (1/2)² = 1/4 ⇒ 4 microcells per old cell → 4× the channels per old footprint Tx power scales as Rn: Pnew = Pold · (R/2)n/Rn = Pold/2n For n = 4 (urban) → 16× less power than the original BS.

How to implement (the 7.5-M answer flow)

  1. Identify hotspot — a cell whose offered traffic exceeds Erlang-B capacity at current GoS.
  2. Reduce R by half and locate new base stations at points spaced R/2 apart (or at the existing cell vertices).
  3. Reduce Tx power on both old and new BSs to keep co-channel interference unchanged.
  4. Reallocate channels respecting the same N-cluster pattern.
  5. Update neighbour lists and handover thresholds in the BSC.
Pitfalls Doubling BS density = double the handovers = doubled signalling load. The MSC must handle 4× the handovers per call. Also, micro-cells have worse coverage at edges due to lower Tx power — careful site survey is essential.

2.10 Cell Sectoring 5–7.5M

Definition Cell sectoring replaces the omni-directional antenna at the BS with 3 (120°) or 6 (60°) directional antennas. Each sector behaves as a separate logical cell. The base station hardware is shared but the cell area is split by angle, not by radius.

How sectoring reduces co-channel interference

With an omni antenna, 6 first-tier co-channel cells contribute interference. With 120° sectoring, only 2 of those 6 interfere with a given sector (the others point away). With 60° sectoring, only 1 interferes.

2.11 Microcell / Zone / Microzone Concept ⭐ 5–7.5M (BCA25-6b)

Definition The microcell zone concept covers a large cell with multiple low-power zone transmitters linked to a single base station by fibre or microwave. The mobile uses whichever zone has the strongest signal but stays under the same BS — so a zone change is not a handoff (no MSC involvement).

Why it beats plain cell-splitting

  • No additional handoffs when a user moves between zones — only intra-cell zone selection.
  • Lower interference — directional zone antennas keep the signal localised.
  • Reuses the BS — only zone transmitters are added; no MSC re-planning.
  • Better in-building coverage — zone Tx can be tucked inside a building or tunnel.

Intelligent / smart cells (24BTECH-6b)

Intelligent cell A cell whose antenna array dynamically steers beams (smart antennas) and adapts radio resources based on real-time user location and traffic — combining sectoring, microcell-zone, and beamforming. Implemented via SDMA (see Sec. 3.4) and used heavily in 4G/5G.

Methods to achieve an intelligent system: (i) phased-array smart antennas with DoA estimation, (ii) SDMA, (iii) AI-based dynamic channel allocation, (iv) cell-zone splitting with self-organising network (SON) algorithms.

3.2 Spread Spectrum — DSSS · FHSS · CDMA ⭐ 10M (BCA25-6a)

Definition Spread Spectrum (SS) is a transmission technique in which the signal bandwidth is intentionally spread over a far wider range than the minimum required by the data rate, using a pseudo-random spreading code known only to sender and intended receiver.

Why spread the spectrum? (BCA25-6a literal)

  1. Anti-jamming — narrowband jammer can only mask a fraction of the wide signal.
  2. Low probability of intercept (LPI) — without the spreading code, the signal looks like noise.
  3. Multiple access (CDMA) — different codes let many users share the same band.
  4. Multipath rejection — wideband signal can be rake-received with diversity gain.
  5. Privacy — code-based scrambling prevents casual eavesdropping.

3.3 Packet Radio & Random Access (ALOHA, CSMA) 5–7M

Definition Packet Radio = forwarding short, addressed data packets over a shared radio channel using random-access MAC protocols. The grandfather of ALL modern wireless data (WLAN, Zigbee, LoRa).

ALOHA family (IT24-1e literal)

  • Pure ALOHA (1971) — transmit any time; if collision, wait random back-off and retry. Max throughput ≈ 18.4 % at G = 0.5. Vulnerability = 2× packet duration.
  • Slotted ALOHA — transmissions begin only at synchronised slot boundaries → vulnerability halved → max throughput ≈ 36.8 % at G = 1.
  • CSMA — sense the channel first, transmit only if idle. 1-persistent, non-persistent, p-persistent variants. Used in 802.11 Wi-Fi (CSMA/CA — collision avoidance + RTS/CTS).
Why packet radio matters in cellular Common-channel signalling (Unit IV), GPRS/EDGE data, LTE random-access channel (PRACH) all descend from packet-radio principles. A "packet ratio" Q means "what fraction of the channel is occupied by useful packet payload" — efficiency ≈ throughput / offered load.

3.4 SDMA — Space Division Multiple Access ⭐ 7.5M (24BTECH-5b)

Definition SDMA reuses the same frequency, time slot, and code in different spatial regions by exploiting directional / smart antennas, beamforming, and MIMO. It is the multiplicative axis on top of FDMA/TDMA/CDMA.

Implementation methods

  1. Sectorisation (the simplest SDMA) — 120°/60° antennas at one BS.
  2. Smart / adaptive antennas — phased-array steers a beam at each desired user using DoA estimation.
  3. MIMO + spatial multiplexing — N transmit × M receive antennas send independent streams in the same band.
  4. Massive MIMO (5G) — 64+ antennas per cell, beamforms to 8+ users in parallel.

Features & applications

  • Capacity grows linearly with number of beams (up to 8–16 typically).
  • Interference rejection — null-steering can place zeros toward known interferers.
  • Range extension — beamforming concentrates power on a user, gaining 10–20 dB.
  • Applications — 4G LTE-Advanced, 5G NR, satellite spot beams, Wi-Fi 6/7 MU-MIMO.

3.5 Capacity of a Cellular System 5–10M

Definition Cellular capacity = number of simultaneous active users (or Erlangs of traffic) the network can support per unit area for a given GoS. Three drivers: spectrum, reuse efficiency, spectral efficiency of the air interface.
Capacity drivers Nusers ∝ (BWtotal / BWchannel) · (1 / Ncluster) · ηair spectrum reuse efficiency modulation/coding gain Capacity per unit area = Nchannels / (cell area · N) grows with smaller cells, smaller N, better η

Two capacity-enhancement schemes (IT23-3b literal)

  • Cell splitting — area shrinks, channels/km² grows ~4×.
  • Sectoring + S/I-based reuse reduction — sectoring improves S/I, letting you drop N from 7 → 4 → 3 ⇒ more channels/cell.
  • (Bonus) SDMA / Smart antennas, Hierarchical cells (macro + micro), Adaptive modulation.

3.6 Near/Far Effect on TDMA & CDMA 1.5M trick Q

Near/far effect Occurs when a transmitter close to the receiver overpowers a distant transmitter sharing the same resource — the strong signal swamps the weak one's reception.

On TDMA (BCA25-1h): mostly immune because each user owns its own time slot — only that user's signal is decoded at any instant. Adjacent-slot interference can occur if guard times are too short, but it is a minor issue compared to CDMA.

On CDMA: severe — all users overlap in the same band; without fast power control, the nearest user's signal masks all others. IS-95 mitigates with closed-loop power control at 800 Hz, keeping every received signal at the same level (~ −110 dBm).

4.2 Development of Wireless Networks 5M

Reuse the Sec. 1.1 evolution table — but for this unit emphasise the networking dimension rather than the radio dimension.

  • Pre-cellular (1940–1980) — Mobile Telephone Service (MTS) and Improved MTS (IMTS): single high-power transmitter per city, manual operator, ~25 channels total.
  • 1G cellular (1980s) — frequency-reuse breakthrough; analog AMPS in US, TACS in UK, NMT in Nordics. Still circuit-switched within the PSTN.
  • 2G digital (1990s) — GSM introduces SS7-based signalling, MAP protocol over SS7, HLR/VLR location DB, SIM, SMS as a signalling-channel by-product.
  • 2.5G/3G (2000s) — GPRS/UMTS adds packet-switched core (SGSN, GGSN), bridging cellular to the IP world.

4.4 Traffic Routing in a Wireless Network ⭐ 7M (BCA25-5b)

Definition Traffic routing in a wireless network = the procedure by which the MSC and SS7 signalling network decide the path a call takes from caller to callee, accounting for the callee's mobility via HLR/VLR location lookup.

The 7-step mobile-terminated call flow (the 7-M answer)

  1. PSTN caller dials the MSISDN (mobile number) of the subscriber.
  2. PSTN routes the call to the GMSC (gateway MSC) of the subscriber's home network using the country/network code.
  3. GMSC queries the HLR using the MSISDN to find the subscriber's current MSC/VLR address.
  4. HLR queries the serving MSC/VLR for a temporary MSRN (Mobile Station Roaming Number) and returns it to the GMSC.
  5. GMSC routes the call to the serving MSC using the MSRN.
  6. Serving MSC pages the mobile in its current Location Area (broadcast on all BTSs of the LA).
  7. Mobile responds, traffic channel assigned, voice path connected. End-to-end.

Mobile-originated call (IT23-2a + 23BTECH-3a)

  1. MS sends RR-CHAN_REQ on RACH (random-access channel) to BTS.
  2. BSC assigns SDCCH (stand-alone dedicated control channel) for signalling.
  3. Authentication (A3) + ciphering (A5) exchange.
  4. MS sends CC-SETUP with dialled number to MSC.
  5. MSC routes to GMSC → PSTN if external; or to terminating MSC if mobile-to-mobile.
  6. Traffic channel (TCH) assigned, ALERT/CONNECT messages, voice path open.

4.5 Common Channel Signalling & Wireless Data Services 5–7M

CCS — Common Channel Signalling A signalling system that uses a separate, dedicated digital channel for all call-control messages, distinct from the voice trunks themselves. The dominant standard is SS7 (Signalling System No. 7) from ITU-T. The opposite (older) approach is CAS — Channel-Associated Signalling — where dial-tones, ringing, etc. share the voice channel itself.

Why CCS / SS7 won

  • Faster call setup — control messages don't wait for trunks; SS7 messages are short and packetised.
  • New services possible — caller-ID, call-forwarding, free-phone (800), SMS, roaming — all use SS7 application-layer messages (TCAP, MAP).
  • Trunk efficiency — trunks carry only voice, not in-band tones.
  • Security — control plane separated from user plane.

Wireless data services (taught alongside CCS)

  • SMS / MMS — short message ride on SS7 (MAP) signalling channels.
  • GPRS / EDGE — 2.5G packet data over GSM TS using SGSN/GGSN nodes.
  • CDPD (23BTECH-6b literal) — Cellular Digital Packet Data, IP packets opportunistically in idle AMPS voice slots; superseded.
  • ARDIS / RAM Mobile Data — early commercial mobile data services (Motorola/RAM) pre-GPRS.
  • 4G data — all-IP, EPS bearers, QoS classes (QCI 1–9).

4.7 Advanced Intelligent Networks (AIN) 5–7M

Definition AIN is an architectural framework (Bellcore, 1990s) that separates switching (handled by the SSP) from service logic (handled by the SCP) so that new telecom services can be deployed without re-programming every switch.

Three core elements

  • SSP — Service Switching Point — the call-handling switch (a MSC or PSTN tandem) that recognises a "trigger" and pauses to query the SCP.
  • SCP — Service Control Point — central database / service logic server reachable via SS7.
  • STP — Signal Transfer Point — packet switch for SS7 messages between SSPs and SCPs.

Advantages (23BTECH-1f literal 1.5-M)

  • Rapid service deployment — one SCP update, all switches benefit.
  • Vendor independence — SCP from one vendor, SSPs from another.
  • Personalisation — user-specific routing, prepaid balance lookup, VPN.
  • Reduced switch software complexity — switches stay generic.
  • Foundation for modern services — toll-free (800/1800), call-forwarding, voicemail, mobile pre-paid, number portability.

Cell splitting (Sec. 2.9) — shrink R, add BSs; capacity ↑4× per split level. Cell sectoring (Sec. 2.10) — split cell by angle using 3×120° or 6×60° antennas; capacity ↑3–6× and S/I ↑5–8 dB. What is the need of spreading a spectrum? Explain the different types of spread spectrum. Explain FHSS in detail.

Appeared as: BCA 2025 6a (10M) · BTech ENC 2023 1c (DSSS, 1.5M) · BTech IT 2025 1c (SSMA, 1.5M)
Block 1 · Why spread the spectrum

Per Sec. 3.2: 5 reasons — (1) anti-jamming, (2) low-probability-of-intercept, (3) multiple access via codes (CDMA), (4) multipath rejection via rake combining, (5) privacy. Originally a military technique (1940s); commercialised in IS-95 cellular and 802.11b WLAN.

Block 2 · Two main types — DSSS vs FHSS comparison

Full 6-row table from Sec. 3.2: how spread, chip/hop rate, BW use, used-in, jammer resistance, implementation complexity.

Block 3 · FHSS — slow vs fast variants

Slow FHSS = hop rate < data rate (multiple bits per hop). Example: Bluetooth Basic Rate (1600 hops/s, 1 Mbps data ⇒ ~625 bits per hop).

Fast FHSS = hop rate > data rate (multiple hops per bit). Used in military radios for maximum jam resistance; rarely in commercial systems because of synthesiser complexity.

Block 4 · FHSS math & processing gain
Hop sequence: f₁ → f₅ → f₂ → f₇ → … (controlled by PN code) Number of hop channels: N Spread BW: BWSS = N · BWchannel Processing gain: Gp = BWSS / BWdata = N Jamming margin: MJ = Gp − (S/N)min (dB) Bluetooth: 79 hop channels × 1 MHz = 79 MHz spread, data at 1 Mbps ⇒ Gp ≈ 19 dB
Block 5 · Application + limitation

Application: Bluetooth (FHSS), military secure radios, GSM optional slow-hopping for anti-fading. Limitation: synchronisation between sender and receiver hop sequences is hard; loss of sync = total loss of link. Also wastes spectrum when only one user is active (CDMA/OFDMA are more efficient for multi-user).

Q7 · Unit 2 15 M 4× repeat

Distinguish between Fixed Channel Assignment and Dynamic Channel Assignment in cellular networks. Explain borrowing and hybrid variants. Which suits bursty traffic better and why?

Appeared as: BTech ENC 2023 7a · BTech ENC 2023 1j (borrowing) · BTech IT 2024 1f · BTech IT 2023 2b
Block 1 · Definitions

Per Sec. 2.5: FCA = each cell pre-assigned a fixed channel set at system-design time. DCA = channels in a central pool, allocated on demand per call.

Block 2 · 7-axis comparison table

Full table from Sec. 2.5: allocation, decision time, complexity, performance at uniform vs bursty traffic, CCI control, examples.

Block 3 · Borrowing strategy + Hybrid Channel Assignment

Borrowing: FCA hybrid — when a cell exhausts its fixed set, borrow an idle channel from a neighbour; donor locks that channel to preserve CCI. Used in old AMPS.

Hybrid Channel Assignment (HCA): split pool, e.g., 70 % fixed + 30 % dynamic. Captures FCA simplicity for normal load and DCA flexibility for bursts.

Channel re-arrangement: when a call ends, the system can move other ongoing calls to compact assignments and pre-empt future blocking.

Block 4 · Which suits bursty traffic + why

DCA wins clearly for bursty / non-uniform traffic because it adapts to load: a hotspot cell can pull from the entire system pool rather than queueing within its own fixed set. FCA cells with bursty traffic block heavily even when neighbours have idle capacity — wasteful.

Block 5 · Real-world deployment

1G AMPS used pure FCA (simple). Early GSM used FCA with frequency hopping for interference smoothing. 3G UMTS and 4G LTE use packet-level DCA on top of OFDMA — every TTI (1 ms) the eNodeB scheduler allocates physical resource blocks dynamically. So in modern networks, DCA is the norm at sub-second granularity.

What are performance criteria of cellular system? Explain in detail and discuss how each is measured.

Appeared as: BCA 2025 2b (5M) · BTech ENC 2024 7b (7.5M) · BTech ENC 2023 4b (7M)
Block 1 · Why criteria matter

Cellular operators must meet regulator + customer expectations on multiple dimensions. KPIs are how operators measure, report, and optimise. Improving one KPI usually costs another (the central trade-off).

Block 2 · The seven KPIs (master table)

Full 7-row table from Sec. 2.2: voice quality (MOS ≥ 4), coverage (≥ 95 %), capacity (Erlangs/cell), GoS (≤ 2 %), call-drop rate (≤ 2 %), handoff success (≥ 98 %), spectral efficiency (max bits/sec/Hz/cell).

Block 3 · How each is measured
  • MOS — Mean Opinion Score from listener panels or PESQ algorithmic estimation.
  • Coverage — drive-test RSS percentile across service area.
  • Capacity — Erlang B/C calculation from peak-hour traffic logs.
  • GoS — count blocked-call events / total attempts in busy hour.
  • Call-drop — count abnormal terminations / completed-call attempts.
  • Handoff success — count successful HO / total HO attempts in BSC logs.
  • Spectral efficiency — total bits delivered / spectrum used / cell area.
Block 4 · Trade-offs between KPIs

Reducing N improves capacity but hurts S/I ⇒ voice quality drops. Adding microcells improves capacity but increases handoff load. Aggressive power control extends battery + reduces interference but can hurt coverage at cell edge. The art of cellular design is balancing all seven.

Block 5 · 4G/5G evolution of KPIs

Modern networks add latency (≤ 1 ms for URLLC slices), reliability (5 9's), connection density (10⁶ devices/km² for IoT), and energy efficiency (J/bit) — but the original 7 remain foundational for voice service.

Q9 · Unit 2 15 M 3× repeat

Explain co-channel interference and its impact on cellular performance. Derive the S/I expression. List and explain at least 5 techniques to reduce CCI.

Appeared as: BTech ENC 2024 4a (7.5M) · BTech IT 2023 6a (10M) · BCA 2025 1f (inference, 1.5M)
Block 1 · Definition + impact

Per Sec. 2.7: Co-channel interference (CCI) = unwanted signal received from another cell using the same frequency in a reuse pattern. Impact: degrades voice quality (MOS), raises call-drop rate, limits achievable cell radius and capacity.

Block 2 · Diagram of the 6 first-tier interferers

Hex cluster with the target cell in the centre and 6 co-channel cells surrounding it at distance D = R√(3N). Each contributes interference power proportional to D−n.

Block 3 · S/I derivation
Signal power S ∝ R−n (received from own BS at distance R) Interference I = Σᵢ Dᵢ−n (6 co-channel BSs at distance D) Approximating all 6 equidistant at D: S/I = R−n / (6·D−n) = (1/6) · (D/R)n = (1/6) · Qn For n=4 (urban): N=7 → Q=4.58 → S/I = (1/6)·4.58⁴ ≈ 73 ≈ 18.7 dB
Block 4 · Five techniques to reduce CCI
  1. Increase cluster size N — pushes co-channel cells further apart; trade-off = fewer channels/cell.
  2. Cell sectoring (120° / 60°) — directional antennas drop the number of effective interferers from 6 to 2 (or 1) ⇒ ~5–8 dB S/I gain.
  3. Antenna tilt (downtilt) — mechanical or electrical tilt shrinks the cell's outward radiation footprint without losing primary coverage.
  4. Power control — mobile transmits only the minimum power needed; reduces uplink CCI to neighbour cells.
  5. Frequency hopping (GSM) / spreading (CDMA) — averages or removes CCI in the time/code dimensions.

Bonus: fractional frequency reuse (LTE), inter-cell interference coordination (ICIC), CoMP (4G/5G).

Block 5 · Other interference types & pitfalls

Acknowledge in 2 lines: adjacent-channel interference (filter leakage between neighbouring channels — mitigated by sharper filters + don't allocate adjacent channels in same cell), intermodulation (PA non-linearity creates 2f₁−f₂ products), near-far (CDMA-specific, solved by power control). Pitfall: in real urban deployments, S/I is much worse than the idealised hex-grid formula due to buildings, terrain, and Tx site location constraints.

Per Sec. 4.6: ISDN (ITU-T) = end-to-end digital transmission of voice + data + signalling over the existing copper subscriber loop. Motivation: replace analog modems (≤ 56 kbps) with native digital — supporting simultaneous voice + data on one line with out-of-band signalling.

Block 2 · Architecture diagram

Block 3 · BRI vs PRI table

Use the 3-row table from Sec. 4.6: BRI = 2B+D = 144 kbps for home/SOHO. PRI Europe = 30B+D ≈ 2 Mbps (matches E1). PRI US = 23B+D ≈ 1.544 Mbps (matches T1). B = bearer (voice/data, 64 kbps), D = signalling.

Block 4 · ISDN vs AIN

ISDN = transport layer (digital bits between user + exchange). AIN = service layer (programmable service logic — SCP queried by SSP via SS7). ISDN moves bits; AIN moves services. AIN runs on top of ISDN signalling.

Block 5 · ISDN vs xDSL and obsolescence

xDSL uses the same copper but achieves Mbps (ADSL 8 Mbps, VDSL2 100 Mbps) by exploiting much higher frequencies above the voice band. ISDN is symmetric at 144 kbps; ADSL is asymmetric at megabits. ISDN was killed commercially by xDSL (1990s) and then fibre — but the PRI variant still survives in PBX trunks for VoIP gateways and SIP-trunking transition setups.

Trade-off table: sectoring gives S/I gain but adds intra-cell handovers; splitting gives capacity but adds inter-cell handovers + new BSs.

The intuition is geometric: a 120° directional antenna can only "see" co-channel cells that fall inside its main lobe, so the effective interferer count drops from 6 → 2. The S/I gain is therefore a direct consequence of antenna geometry, not extra transmit power — emphasise this when deriving the dB improvement.

Key takeaways · revise these 5
  • Omni cell sees 6 first-tier interferers; 120° sector sees 2; 60° sector sees 1.
  • S/I gain ≈ 10 log10(6/2) ≈ 5 dB (120°) and ≈ 10 log10(6/1) ≈ 8 dB (60°).
  • Sectoring needs no new BS site — only an antenna swap — cheapest capacity / quality lever.
  • Cost: each sector becomes its own logical cell, so movement across sectors triggers intra-cell (softer) handovers.
  • vs splitting: sectoring boosts S/I + capacity at one site; splitting adds new sites for raw capacity but multiplies inter-cell HOs.

Part I drill · 1.5 M each · cover the entire syllabus

1. Define frequency-reuse distance and state its formula.
U2The co-channel distance D between two cells reusing the same frequencies, given by D = R · √(3N) where R is the cell radius and N is the cluster size. Larger D ⇒ less CCI but lower capacity.
2. What is the difference between TDD and FDD?
U3FDD uses two paired frequencies for UL and DL (e.g. GSM 890–915 / 935–960 MHz); TDD uses one frequency with alternating time slots (e.g. WiMAX, 5G NR TDD). FDD needs a duplexer; TDD only a switch. TDD allows asymmetric UL/DL ratios.
3. What is GoS and what is the cellular target?
U2Grade of Service = probability that a call attempt is blocked (Erlang B) or excessively delayed (Erlang C) during busy hour. Cellular target: GoS ≤ 0.02 (2%) — at most 2 of every 100 calls blocked at peak.
4. Define scatternet.
U1Interconnection of two or more Bluetooth piconets via a shared bridge slave that time-multiplexes between them. Extends Bluetooth coverage past the 7-slave-per-piconet limit.
5. List 4 advantages of 4G mobile networks.
U1(1) 100 Mbps – 1 Gbps peak data rate · (2) all-IP packet core with < 50 ms latency · (3) spectral efficiency via OFDMA + MIMO · (4) seamless inter-RAT handover to 2G/3G/Wi-Fi.
6. Why hexagonal cells in cellular system?
U2Tessellate without gaps (only 3 polygons do); maximum area per radius; six equidistant neighbours simplify reuse maths; best match to omni-antenna circular pattern.
7. State the 5 main applications of CDMA.
U3Cellular (IS-95 cdmaOne, CDMA-2000, 3G WCDMA), GPS, military anti-jam comms, Wi-Fi 802.11b (DSSS variant), satellite (Globalstar).
8. Explain the term "soft handoff" and where it is used.
U2Make-before-break handover where the mobile maintains simultaneous links with 2+ base stations during the transition. Only possible in CDMA (all BSs share same frequency, rake-combined for diversity gain). Eliminates the audio gap and call drops at cell edges.
9. What is borrowing strategy in channel assignment?
U2FCA hybrid — when a cell exhausts its fixed channel set, it borrows an idle channel from a neighbour, locking that channel in the donor cell to preserve CCI. Used in old AMPS overflow.
10. Write the BRI and PRI rates of ISDN.
U4BRI = 2B + D = 2×64 + 16 = 144 kbps. PRI Europe = 30B + D = 30×64 + 64 = 1.984 Mbps (≈ E1). PRI US = 23B + D = 23×64 + 64 = 1.544 Mbps (≈ T1).
11. Why does CDMA need fast power control?
U3To solve the near-far problem: a nearby strong transmitter swamps a distant weak one because both share the same frequency and time. IS-95 uses closed-loop power control at 800 Hz so every received signal arrives at ~ −110 dBm.
12. Define processing gain in spread spectrum.
U3Gp = BWSS / BWdata = chip-rate / data-rate (DSSS) or N (number of hop channels, FHSS). Represents the "spreading factor" — directly translates to jamming margin and interference suppression. IS-95 Gp ≈ 128 (21 dB).
