2.1 What is R? History & Features

Definition R is an open-source, interpreted, high-level language and environment for statistical computing and graphics. Created by Ross Ihaka & Robert Gentleman at the University of Auckland (1993); current stable maintained by the R Core Team and distributed via CRAN.

Key features:

  • Vectorised operations — built-in element-wise arithmetic (no explicit loop for x + y).
  • 18,000+ packages on CRAN (statistics, ML, viz, bioinformatics).
  • First-class statistics — distributions, hypothesis tests, GLMs built in.
  • Reproducible reports via R Markdown / Quarto (PDF, HTML, Word).
  • Cross-platform & free (GPL-2).

Limitations: single-threaded by default · slower than C/Python for heavy loops · steeper learning curve for non-statisticians · weaker production-deployment story than Python.

Definition CRAN (Comprehensive R Archive Network) — a worldwide network of FTP and web servers that store identical, up-to-date versions of R, contributed packages, documentation and source code. Main site: cran.r-project.org.

3.1 Function Creation & Scripts

R is functional first — every operation is a function call. UDFs are first-class objects.

# Anatomy: name <- function(args) { body }
greet <- function(name="friend", greeting="Hello") {
  paste(greeting, name)
}
greet()              # "Hello friend"
greet("Ankit")       # "Hello Ankit"
greet(greeting="Hi", name="A")  # named args

# Anonymous functions (lambda)
sapply(1:5, function(x) x^2)   # 1 4 9 16 25

# R 4.1+ lambda shorthand
sapply(1:5, \(x) x^2)

Features: default args · lazy evaluation · last expression auto-returned · lexical scoping · pass-by-value.

Scripts: save as .R, run via source("file.R") or Rscript file.R from shell.

3.2 Logical Operators & Conditional Statements

# if / else if / else
x <- 5
if (x > 0) {
  print("positive")
} else if (x == 0) {
  print("zero")
} else {
  print("negative")
}

# Inline (ternary equivalent)
sign <- if (x > 0) "pos" else "neg"

# Vectorised: ifelse()
v <- c(-1, 0, 5, -3)
ifelse(v >= 0, "non-neg", "neg")
# "neg" "non-neg" "non-neg" "neg"
if vs ifelse — common 5M trap if is a control-flow statement that takes one scalar condition. ifelse() is a vectorised function that returns a vector of the same length as its first argument. Inside for/while bodies, prefer if; for whole-vector decisions, ifelse().

3.4 switch() Statement 2023 Q3b · 2025 Part-A(g)

# Replaces long if-else if chains
day <- "mon"
switch(day,
  mon = "Start of week",
  fri = "TGIF!",
  sat = ,                # falls through
  sun = "Weekend",
  "Mid-week"             # default (no name)
)

Behaviour: evaluates the first arg, matches against named alternatives, returns the corresponding value. An unnamed last argument is the default. Empty cases fall through to the next.

3.8 Object-Oriented Programming in R ⭐ 2023 Q6a, 2025 Q3b (10M)

R supports three OOP systems. Each defines classes & method dispatch differently.

S3 — informal, the original

# Any object can become a class via class()
x <- list(name="A", marks=85)
class(x) <- "Student"

# Methods are functions named generic.class
print.Student <- function(o, ...) {
  cat("Student:", o$name, "Marks:", o$marks, "\n")
}
print(x)        # dispatches to print.Student

# Inheritance via class vector
class(x) <- c("HonorsStudent", "Student")

S4 — formal, slot-based

setClass("Student",
  representation(name="character", marks="numeric"))
s <- new("Student", name="A", marks=85)
s@name; s@marks            # @ for slot access

setGeneric("greet", function(x) standardGeneric("greet"))
setMethod("greet", "Student",
  function(x) cat("Hi", x@name))
greet(s)                   # "Hi A"

isVirtualClass("Student")
validObject(s)             # checks types

Reference Classes (R5) — mutable, Java-like

Account <- setRefClass("Account",
  fields = list(balance = "numeric"),
  methods = list(
    deposit  = function(x) { balance <<- balance + x },
    withdraw = function(x) { balance <<- balance - x }
  ))
a <- Account$new(balance=100)
a$deposit(50); a$balance    # 150 — MUTATED in-place

The 6 layers in prose — one paragraph each

  • 1 · Data — the data frame fed to ggplot() is the foundation: every layer above derives its values from these columns. One plot can refer to a single data frame or layer-specific overrides. Example: ggplot(data = students) binds the students data frame to the plot.
  • 2 · Aesthetics (aes()) — the mapping from data columns to visual channels (x, y, colour, fill, size, shape, alpha). Aesthetics describe what variable controls what visual, never raw values. Example: aes(x = hours, y = marks, colour = grade) maps three columns to three channels.
  • 3 · Geometries (geom_*) — the visual marks drawn for each row: points, bars, lines, boxplots, polygons. Geometries consume aesthetics and emit shapes; multiple geoms can stack on the same aesthetic mapping. Example: geom_point(size = 3) draws a scatter, geom_bar(stat = "count") draws bars.
  • 4 · Statistics (stat_*) — optional transformations applied before drawing: smoothing, binning, density estimation, summary statistics. Every geom has a default stat (e.g. geom_bar uses stat_count); explicit stats let you overlay fits or summaries. Example: stat_smooth(method = "lm") overlays a linear-regression line with confidence band.
  • 5 · Facets (facet_*) — split the plot into small multiples by one or two categorical variables, so subgroups can be compared side-by-side without overlapping. Replaces base-R's manual par(mfrow=). Example: facet_wrap(~ grade) creates one panel per grade; facet_grid(sex ~ year) builds a 2-D matrix.
  • 6 · Themes (theme_*) — non-data styling: fonts, gridlines, panel background, legend position, axis ticks. Themes never change the data or its mapping, only the presentation. Example: theme_minimal() strips chartjunk for publication; theme(legend.position = "bottom") tweaks one element.

Compose order: Data → Aesthetics → Geometries are mandatory; Statistics, Facets and Themes are optional refinements. 4.8 Hypothesis Testing ⭐ 2023 Q4a, 2025 Q6a (8–10 M)

Definition Hypothesis testing is a formal statistical procedure to decide between two competing claims about a population parameter using sample data and a chosen significance level α.

The six steps (memorise the order)

  1. State H₀ & H₁ — null (no effect, equality) and alternative (claim to test).
  2. Choose α — significance level, typically 0.05 (5%).
  3. Select test statistic — z, t, χ², F, depending on data.
  4. Compute test stat & p-value from the sample.
  5. Decision rule — reject H₀ if p < α (or |stat| > critical value).
  6. Interpret in context — translate back to the original problem.

Covariance vs Correlation (2024 Part-A h) Covariance measures joint variability in original units — magnitude is hard to interpret. Correlation standardises covariance by SDs, giving a unit-free [−1, 1] number → directly comparable across datasets. cor(x,y) = cov(x,y) / (sd(x) × sd(y)).

Explain object-oriented programming in R with examples of S3, S4, and Reference Classes. ↗ Sec. 3.8

OOP in R

R supports object-oriented programming through three distinct class systems, each adding more formality. Method dispatch and slot access differ across them.

1. S3 — informal
x <- list(name="Ankit", marks=85)
class(x) <- "Student"
print.Student <- function(o, ...)
  cat("Student:", o$name, "→", o$marks, "\n")
print(x)        # → Student: Ankit → 85
2. S4 — formal, slot-based
setClass("Student",
  representation(name="character", marks="numeric"))
s <- new("Student", name="Ankit", marks=85)
s@name                # "Ankit"  ('@' for slots)
setGeneric("show2", function(x) standardGeneric("show2"))
setMethod("show2", "Student",
  function(x) cat(x@name, "→", x@marks))
show2(s)
3. Reference Class (R5) — mutable
Account <- setRefClass("Account",
  fields = list(balance="numeric"),
  methods = list(
    deposit = function(x) { balance <<- balance + x }
  ))
a <- Account$new(balance=100)
a$deposit(50); a$balance     # 150 — modified in place
Contrast

Both belong to R's apply family — functional alternatives to for loops. They apply a function FN to each element of an input X.

Describe various measures of central tendency and explain how to calculate them using R. ↗ Sec. 4.7

Definitions

Measures of central tendency are single values that describe the centre of a distribution. They tell us "what is a typical value?".

1. Mean (arithmetic average)
x̄ = (Σ xᵢ) / n

R: mean(x). Best for symmetric numeric data; sensitive to outliers.

2. Median

The middle value when data is sorted. R: median(x). Robust to outliers; preferred for skewed data and income/price data.

3. Mode

The most frequently occurring value. No built-in in R — write a UDF:

getMode <- function(v) {
  u <- unique(v)
  u[which.max(tabulate(match(v, u)))]
}

Best for categorical / nominal data.

Worked example
x <- c(2, 4, 4, 4, 5, 5, 7, 9)

mean(x)        # 5.0
median(x)      # 4.5
getMode(x)     # 4
sd(x)          # 2.07
var(x)         # 4.286

summary(x)
# Min. 1st Qu.  Median  Mean 3rd Qu. Max.
# 2.0  4.0     4.5     5.0   5.5     9.0
Application & interpretation

For a symmetric distribution, mean ≈ median ≈ mode. In a right-skewed distribution (e.g. salaries), mean > median > mode — outliers pull the mean. Reporting all three tells a complete story.

Common mistake Reporting only the mean for skewed data is misleading. Combine with median + SD for a fair summary.

What is hypothesis testing? Write steps for performing a hypothesis test in R with an example. ↗ Sec. 4.8

Definition

Hypothesis testing is a formal statistical procedure to decide between two competing claims about a population parameter using sample data and a chosen significance level α.

Six steps
  1. State H₀ and H₁ — null (status quo, equality) vs alternative.
  2. Choose significance level α — usually 0.05.
  3. Select the appropriate test statistic — z, t, χ², F.
  4. Compute the test statistic and p-value from sample data.
  5. Apply decision rule — reject H₀ if p < α.
  6. Interpret in context of the original problem.
Example in R — one-sample t-test

A factory claims the mean weight of its product is 50 g. A sample of 10 yields:

weights <- c(48, 52, 49, 51, 53, 47, 50, 51, 49, 52)

# Step 1–2:
# H0: mu = 50    H1: mu != 50    alpha = 0.05

# Step 3–4:
t.test(weights, mu = 50)
# t = 0.395, df = 9, p-value = 0.7022
# 95% CI: [49.0, 51.6]

# Step 5: p (0.70) > alpha (0.05) → fail to reject H0
# Step 6: Insufficient evidence to claim the mean differs from 50 g.

(See Fig 4.8 for the pipeline view.) Common variants: two-sample t.test(x, y), paired t.test(x, y, paired=TRUE), proportions prop.test(), chi-square chisq.test().

Common pitfalls (1) p > α does not prove H₀, only fails to reject. (2) α must be fixed before looking at data. (3) Statistical significance ≠ practical significance.

What is Poisson distribution? Explain with examples.

The Poisson distribution models the probability of a given number of events occurring in a fixed interval of time/space, when those events occur independently at a constant average rate λ.

P(X = k) = (λᵏ · e⁻λ) / k! , k = 0, 1, 2, …

Mean = λ, Variance = λ (equi-dispersion is a defining feature).

Use cases: calls arriving at a help desk per hour · defects per metre of fabric · arrivals at an airport gate per minute · clicks on a web page per second.

# Example: a call centre receives λ=4 calls/hour on average.
# P(exactly 2 calls in next hour)
dpois(2, lambda = 4)        # 0.1465

# P(at most 3 calls)
ppois(3, lambda = 4)        # 0.4335

# Simulate 100 hours
sim <- rpois(100, lambda = 4)
mean(sim); var(sim)         # both ≈ 4

When to use Poisson: rare-event count data with low p · approximation to Binomial when n is large and p is small (λ = np).

Discuss about one-sample T-test with example.

A one-sample t-test determines whether the mean of a single sample differs significantly from a known/hypothesised population mean μ₀, when the population standard deviation is unknown and n is small.

t = (x̄ − μ₀) / (s / √n) , degrees of freedom = n − 1

Assumptions: data is approximately normal · observations are independent.

Example: A bakery claims its loaves weigh 500 g on average. A sample of 8 yields 498, 503, 497, 505, 499, 502, 501, 500.

loaves <- c(498, 503, 497, 505, 499, 502, 501, 500)

t.test(loaves, mu = 500)
# t = 0.451, df = 7, p-value = 0.6657
# 95% CI: [498.6, 502.4]
# Conclusion: p > 0.05 → fail to reject H0.
# No evidence that mean weight differs from 500 g.

#1 · U3 · 1.5–5 M ⭐ 3× (2023·2024·2025)

Explain lapply() and sapply() with examples. Differentiate their output.

Appeared: 2023 Part-A(h), 2024 Part-A(i), 2025 Q7(b) · ↗ Sec. 3.7

Both belong to R's apply family — functional, vectorised alternatives to explicit for loops. They share the contract "apply FN to each element of X" but differ in output type.

lapply(X, FN) — always returns a list of the same length as X. Preserves structure even when results vary in length/type.

sapply(X, FN) — simplified lapply: if every result is a scalar of the same type, returns a vector; if each is a same-length vector, returns a matrix; otherwise falls back to a list.

Worked comparison
nums <- 1:4

# lapply — list of 4
lapply(nums, sqrt)
# [[1]] 1  [[2]] 1.414  [[3]] 1.732  [[4]] 2

# sapply — vector of length 4
sapply(nums, sqrt)
# 1.000 1.414 1.732 2.000

# When sapply CANNOT simplify, it returns a list
sapply(list(1:2, 1:3), identity)   # list (lengths differ)

# Matrix simplification
sapply(1:3, function(i) c(i, i^2))
#      [,1] [,2] [,3]
# [1,]    1    2    3
# [2,]    1    4    9
Why apply family beats loops
  • Functional style — code expresses intent, not bookkeeping.
  • Faster — internal C loops avoid R-level overhead.
  • Composable — output of one apply can feed the next.
Compression for 1.5–2 M version One-line definition: "lapply applies FN to each element of X and returns a list; sapply simplifies that list to a vector when possible." Add one example: sapply(1:3, sqrt) → c(1, 1.414, 1.732).
Key takeaways · revise these 5
  • Input shape: both accept a vector or list; both call FN on each element exactly once.
  • Output shape: lapply always returns a list of the same length; sapply attempts simplification.
  • When sapply simplifies: every result is a scalar of the same atomic type → vector; every result is a same-length vector → matrix.
  • When sapply falls back to list: results differ in length or type (e.g. sapply(list(1:2, 1:3), identity)).

What is hypothesis testing? Describe types and steps with an R example.

Appeared: 2023 Q4(a) 8M, 2025 Q6(a) 10M · ↗ Sec. 4.8

Hypothesis testing is a formal statistical procedure to decide between two competing claims about a population parameter using sample data and a chosen significance level α.

Types
  • By number of tails: one-sided (H₁: μ > μ₀) vs two-sided (H₁: μ ≠ μ₀)
  • By parameter: parametric (z, t, F, χ²) vs non-parametric (Wilcoxon, Kruskal–Wallis)
  • By sample count: one-sample, two-sample (independent), paired
Six steps

(See Sec. 4.8 and Fig 4.8.) State H₀/H₁ → choose α → select test statistic → compute p-value → decide (reject if p<α) → interpret.

R example — one-sample t-test
x <- c(48,52,49,51,53,47,50,51,49,52)
t.test(x, mu = 50)
# t = 0.395, df = 9, p-value = 0.7022
# Since p > 0.05, fail to reject H0.

Key takeaways · revise these 5
  • Definition: formal procedure to decide between H₀ (status quo) and H₁ (claim) using sample evidence and a pre-set α.

Appeared: 2023 Q3(b) 8M, 2025 Part-A(g) 1.5M · ↗ Sec. 3.4

switch() selects one of several values based on a single key, replacing long if/else if chains. Cleaner when branches are mutually exclusive and based on a single expression.

dayName <- function(d) {
  switch(d,
    mon="Monday", tue="Tuesday", wed="Wednesday",
    thu="Thursday", fri="Friday",
    sat=, sun="Weekend",          # fall-through
    "Unknown")                    # default
}
dayName("sat")    # "Weekend"

(Full if vs ifelse contrast at Sec. 3.2.)

Key takeaways · revise these 5
  • Signature: switch(expr, case1 = val1, case2 = val2, default) — returns the matched value; the last unnamed argument is the fallback.

(Use the full 7-row contrast + 3 code snippets from Sec. 3.8.) The "BankAccount" rotation idea — implement the same class three ways to highlight that S3 is informal and copy-on-modify, S4 enforces typed slots, and R5 alone supports true mutation via <<-:

# --- S3 (immutable) ---
deposit_s3 <- function(acc, x) { acc$bal <- acc$bal + x; acc }
acc1 <- structure(list(bal=100), class="acc")
acc1 <- deposit_s3(acc1, 50)        # must reassign

# --- S4 (slot-validated, still copy-on-modify) ---
setClass("Acc", representation(bal="numeric"))
acc2 <- new("Acc", bal=100)
acc2@bal <- acc2@bal + 50           # creates a copy in fn scope

# --- Reference Class (mutable in place) ---
Acc <- setRefClass("Acc",
  fields=list(bal="numeric"),
  methods=list(deposit=function(x) bal <<- bal + x))
acc3 <- Acc$new(bal=100)
acc3$deposit(50); acc3$bal          # 150 — mutated
Key takeaways · revise these 5
  • S3 BankAccount: a plain list with class<-"acc"; mutation requires reassigning the whole object — copy-on-modify.
  • S4 BankAccount: setClass("Acc", representation(bal="numeric")); slot-validated at construction; still copy-on-modify.
  • R5 BankAccount: setRefClass with fields and methods; only R5 supports <<- mutation persisting across method calls.
  • Method dispatch: S3 looks up fn.class by name; S4 / R5 use formal generics — S4 enforces argument types at registration.
  • Pick by need: S3 for quick prototypes, S4 for typed reproducible packages (Bioconductor), R5 for stateful objects (DB handles, GUIs).

Explain measures of dispersion (range, variance, SD, IQR) along with measures of central tendency. Apply all on a real dataset in R.

Model answer

Central tendency: mean, median, mode (Sec. 4.7).

Dispersion:

  • Range = max − min · R: diff(range(x))
  • Variance = Σ(x − x̄)²/(n−1) · R: var(x)
  • SD = √variance · R: sd(x)
  • IQR = Q3 − Q1 (robust) · R: IQR(x)
  • Coefficient of variation = SD / mean (unit-free)
x <- c(2, 4, 4, 4, 5, 5, 7, 9)

# Tendency
mean(x); median(x)                # 5, 4.5

# Dispersion
diff(range(x))                    # 7
var(x); sd(x); IQR(x)             # 4.29, 2.07, 1.0
sd(x) / mean(x)                   # 0.41 = CV

summary(x)                        # 5-number summary

Interpretation: tendency tells you "where the centre is", dispersion tells you "how spread out the data is". Together they characterise a distribution at first glance.

Key takeaways · revise these 5
  • Range = max(x) - min(x); cheapest dispersion, but defined by the two extreme values — fragile to outliers.
  • Variance = Σ(x − x̄)²/(n−1); R: var(x). Penalises large deviations quadratically. Units = (unit of x)².
  • SD = √variance; R: sd(x). Same units as x; the standard companion to the mean.
  • IQR = Q3 − Q1; R: IQR(x). Robust to outliers — the workhorse spread for skewed data; paired with median.
  • Coefficient of variation = SD / mean — unit-free spread, useful when comparing variability across data sets with different scales.
