Binary Codes & Self-Complementary Codes ⭐ Repeated

Definition A code is a pattern of bits used to represent numbers, letters, or symbols. Weighted codes assign a fixed weight to each bit position (BCD 8-4-2-1); non-weighted codes do not (Gray, Excess-3).
CodeTypeKey PropertyUse
BCD (8421)WeightedEach decimal digit = 4 bits (0000–1001). 1010–1111 are invalid.Digital displays, calculators.
Excess-3 (XS-3)Non-weighted, self-complementaryBCD + 3. 9's complement = bit complement.Old arithmetic units.
Gray codeNon-weighted, reflectedOnly one bit changes between successive values.Encoders, K-maps (no glitches).
ASCIIAlphanumeric7-bit code for 128 characters.Keyboards, text.
Self-Complementary Code A code is self-complementary if the code of a decimal digit D is the bitwise (1's) complement of the code of 9 − D. Examples: Excess-3, 2-4-2-1, 8-4-(−2)-(−1). For each, the sum of a digit's code and the code of its 9's-complement is all-ones. This makes 9's-complement subtraction trivial — just invert the bits.
Professor Pattern "What are self-complementary codes?" is a recurring 1.5M Part-A. Define + name Excess-3 + show one worked pair (e.g. 4 → 0111, 5 → 1000 are complements) for full marks.

Error-Detecting & Error-Correcting Codes ⭐ Repeated both years

Why error codes? Noise on a transmission line can flip bits. Redundant (extra) bits are added so the receiver can detect (and sometimes correct) such flips.

Parity (single-bit error detection)

  • Even parity: append a parity bit so the total number of 1s is even.
  • Odd parity: append a parity bit so the total number of 1s is odd.
  • Detects any single-bit error (and any odd number of errors). Cannot detect even-numbered errors, cannot correct.

Hamming Distance

Hamming Distance (d) The number of bit positions in which two code words differ. The minimum Hamming distance of a code decides its power:
• Detect up to d − 1 errors. • Correct up to ⌊(d − 1)/2⌋ errors.
  • Single parity → minimum distance 2 → detect 1, correct 0.
  • Hamming code → minimum distance 3 → detect 2, correct 1.

Hamming Code — single-error correction

Parity bits are placed at positions that are powers of 2 (1, 2, 4, 8…). The number of parity bits r for m data bits satisfies:

Hamming Relation 2r ≥ m + r + 1 — pick the smallest r that fits the data. Each parity bit checks a fixed set of positions; the binary syndrome of failed checks points straight to the erroneous bit.

Full step-by-step single-error correction is worked in Predicted Q3.

Professor Pattern A 15M Hamming-correction numerical appeared in both 2024 and 2025 papers. Memorise the P1/P2/P4 position-checking scheme and the syndrome rule.

Digital Logic Families & Fan-Out ⭐ Repeated both years

Logic Family A group of digital ICs built with the same circuit technology so they share compatible voltage levels, speed, and power. Main families: TTL (bipolar), Schottky TTL (faster TTL), CMOS (MOSFET, low power), ECL (fastest), tri-state outputs.

Key Characteristics of Digital ICs

  • Fan-out: the maximum number of standard gate inputs one output can drive reliably. CMOS has the highest fan-out (~50, because gate inputs draw almost no DC current) vs standard TTL (~10).
  • Fan-in: the number of inputs a gate has.
  • Propagation delay: time for a change at the input to reach the output. TTL/ECL are fast; CMOS is slower (but improving).
  • Noise margin: the maximum noise voltage tolerated before logic level is misread. CMOS has the best noise margin (≈ 45% of VDD).
  • Power dissipation: CMOS draws almost zero static power (only on switching); TTL draws steady power.
One-liner answer "Which family supports maximum fan-out?" → CMOS, because its MOSFET inputs are voltage-driven and draw negligible input current.

Full TTL vs Schottky vs CMOS comparison table is in Predicted Q2(b).

Tri-State Logic & Buffers ⭐ Repeated both years

Tri-State Logic A gate output that can take three states: logic 0, logic 1, and a third high-impedance (Hi-Z) state where the output is electrically disconnected from the line. Controlled by an ENABLE pin.
  • When ENABLE is active, the buffer passes the input to the output normally (0 or 1).
  • When ENABLE is inactive, the output goes to Hi-Z — as if the device were removed from the circuit.
  • Signal management benefit: many tri-state outputs can share one common bus wire. Only one device is enabled at a time; the rest sit in Hi-Z, so there is no bus contention (no two drivers fighting). This is exactly how a CPU data bus and memory bus work.

FET Basics & CMOS↔TTL Interfacing compressed

FET (Field-Effect Transistor) A voltage-controlled, unipolar device — the gate voltage controls drain-to-source current with almost no gate current. The MOSFET (used in CMOS) comes in n-channel (NMOS) and p-channel (PMOS) types; a complementary pair forms a CMOS gate.
  • Interfacing problem: TTL and CMOS use different voltage and current levels, so they cannot always be wired directly.
  • TTL → CMOS: TTL's logic-HIGH (~3.3 V) may be too low for CMOS at 5 V. Fix: add a pull-up resistor on the TTL output to lift HIGH to VDD.
  • CMOS → TTL: CMOS can source enough voltage but a low-power CMOS may not sink TTL's input current. Fix: use a CMOS buffer (e.g. 4049/4050) that has enough current drive, or a level-shifter.

Carry-Look-Ahead Adder (CLA) ⭐ Repeated both years

The Ripple-Carry Problem In a ripple-carry adder, each full adder must wait for the carry from the stage below. For an n-bit add, the carry ripples through all n stages, so delay grows with n — slow for wide words.

The carry-look-ahead adder computes all carries in parallel directly from the inputs, using two signals per bit:

  • Generate Gi = Ai·Bi — this stage creates a carry on its own.
  • Propagate Pi = Ai ⊕ Bi — this stage passes an incoming carry forward.
  • Carry equation: Ci+1 = Gi + Pi·Ci, expanded so every carry depends only on inputs and C0.
Operational Advantage Because all carries are generated simultaneously from the input bits, the carry no longer ripples. The total add delay becomes nearly constant (≈ 2 gate delays) regardless of word width — far faster than a ripple-carry adder, at the cost of more gates.
1.5M one-liner "Operational advantage of CLA?" → It computes all carry bits in parallel from the inputs, removing the ripple delay, so addition speed is roughly independent of word length.

BCD Arithmetic & BCD Adder ⭐ Repeated both years

BCD Addition Rule Add the two BCD digits with a normal 4-bit binary adder. If the result is greater than 9 (1001) or a carry-out is produced, add 0110 (6) to correct it back to valid BCD and generate a carry to the next digit.

BCD subtraction is done by 9's or 10's complement addition. Worked BCD subtraction (8 − 3) is in Predicted Q1(e); the full BCD-adder logic with the +0110 correction circuit is in Predicted Q5(a).

Popular MSI: Comparator, Parity, Encoder, Decoder ⭐ Encoder repeated

  • Digital comparator: compares two n-bit numbers A and B, producing three outputs A>B, A=B, A<B. A=B is true when every bit pair is equal (XNOR of all bits). IC 7485 is the classic 4-bit comparator.
  • Parity generator/checker: a tree of XOR gates. The generator produces the parity bit at the transmitter; the checker XORs all received bits to flag an error (output 1 = error for even parity).
  • Priority encoder: a 2n→n encoder where, if several inputs are active at once, only the highest-priority one is encoded. Solves the plain-encoder ambiguity. An extra "valid" output flags whether any input is active.
  • Decoder/driver: an n→2n decoder activates exactly one output line per input code. 7-segment decoder/drivers (e.g. 7447) convert a BCD digit into the seven segment-drive signals (a–g) for a display.

The 4-input priority encoder (truth table + equations) is worked in Predicted Q4(b).

Professor Pattern "Explain a priority encoder" is a recurring Part-A. Stress: resolves multiple simultaneous active inputs by encoding only the highest-priority line.

Code Converters ⭐ BCD↔XS-3

Code Converter A combinational circuit that translates a number from one code to another (BCD ↔ Excess-3, Binary ↔ Gray). Design steps: build the truth table mapping input code → output code, draw a K-map for each output bit, simplify, and realise with gates.
  • Binary → Gray: MSB stays the same; each lower Gray bit = XOR of adjacent binary bits (Gi = Bi+1 ⊕ Bi).
  • Gray → Binary: MSB stays; each lower binary bit = XOR of the current Gray bit with the previous binary bit.
  • BCD → Excess-3: add 0011 to the BCD digit. Realised as a 4-input/4-output converter via K-maps.

Full BCD-to-Excess-3 converter (truth table + 4 K-maps + circuit) is worked in Predicted Q5(b).

Quine-McCluskey (Tabular) Minimization ⭐ 10M both years

Why Quine-McCluskey (Q-M)? K-maps become impractical beyond 5–6 variables and rely on human pattern-spotting. The Quine-McCluskey method is a systematic, tabular algorithm that always finds the minimal SOP — suitable for hand-work and for software.

The Algorithm

1
List all minterms in binary, grouped by the number of 1s.
2
Compare adjacent groups; combine terms differing in exactly one bit (replace that bit with a dash -). Tick combined terms.
3
Repeat combining on the new lists until no further combination is possible.
4
Un-ticked terms are prime implicants (PIs).
5
Build a PI chart (PIs × minterms). A column with a single tick gives an essential prime implicant. Select EPIs first, then cover remaining minterms minimally.

The ALU (Arithmetic Logic Unit) combines an adder/subtractor with logic units under select lines to perform arithmetic and logic ops on operands — elementary ALU design uses a full-adder array plus multiplexed logic functions. A full Q-M numerical is worked in Predicted Q4(a).

Flip-Flops, Latches & Excitation Tables ⭐ FF vs latch repeated

Latch vs Flip-Flop A latch is level-triggered — its output follows the input as long as the enable level is held. A flip-flop is edge-triggered — it changes state only on a clock edge (rising or falling). Flip-flops are the building blocks of synchronous sequential circuits.

Asynchronous (Ripple) vs Synchronous Counters ⭐ Repeated both years

Counter A sequential circuit that cycles through a fixed sequence of states on successive clock pulses. A MOD-N counter has N states and needs ⌈log2N⌉ flip-flops.
ParameterAsynchronous (Ripple)Synchronous

1.5M one-liner Async: each FF triggered by the previous FF's output → carry ripples → slow. Sync: all FFs share the same clock → all change at once → fast and glitch-free.

04Analog-to-Digital
Converters

Turning continuous voltages into bits — sampling, quantization, encoding, and the four classic ADC architectures.

Sampling, Quantization, Encoding & Resolution ⭐ Repeated both years

The ADC Pipeline 1. Sampling — measure the analog signal at fixed time intervals (Nyquist: fs ≥ 2·fmax). 2. Quantization — round each sample to the nearest of 2n discrete levels. 3. Encoding — represent each level as an n-bit binary code.
Resolution The resolution of an n-bit ADC is the smallest analog change it can detect — the voltage of one step (1 LSB):
Resolution = VFS / (2n − 1) (or VFS/2n). More bits → finer resolution → smaller quantization error (max ± ½ LSB).
  • 1.5M answer for "Define resolution": the minimum input-voltage change that produces a one-LSB change in the digital output; equals full-scale voltage divided by the number of steps.
  • Quantization error: the unavoidable rounding error, bounded by ± half a step (± ½ LSB).

The Four Classic ADC Architectures ⭐ Repeated both years

Trade-off axis All ADCs trade speed against cost/complexity. Flash is fastest but costliest; dual-slope is slow but most accurate and noise-immune.
  • Flash (parallel-comparator): uses 2n−1 comparators against a resistor-ladder reference; all compare at once. Fastest (one clock), but hardware doubles per added bit — costly.
  • Successive-approximation (SAR): a binary-search ADC. A SAR register sets each bit MSB→LSB, a DAC feeds back a trial voltage, a comparator keeps or clears the bit. Takes exactly n clock cycles for n bits. Best speed/cost balance.
  • Counting (digital-ramp): a counter drives a DAC; the DAC output ramps up until it exceeds the input, then the count is latched. Worst case 2n clock cycles. Slow, simple.
  • Dual-slope (integrating): integrate the input for a fixed time, then integrate a reference of opposite sign until the integrator returns to zero; the count is proportional to the input. Very accurate, noise-rejecting (averages noise over the integration window), but slow. Used in digital multimeters.

Full comparison + worked conversion-time numericals are in Predicted Q7.

Conversion Time & ADC Specifications ⭐ Numerical both years

Conversion-Time Formulas (memorise) • SAR ADC: Tconv = n × Tclk (n = bits) — fixed.
• Counting ADC: Tconv(max) = (2n) × Tclk (worst case).
• Flash ADC: ≈ one propagation delay (effectively a single clock).
• Dual-slope: Tconv = (2 × 2n) × Tclk worst case (fixed up-ramp + variable down-ramp).
  • Other specs: resolution (bits), conversion time/sampling rate, accuracy, linearity (DNL/INL), full-scale range, missing codes.
  • Example ICs: ADC0804/0808 (8-bit SAR), ICL7106/7107 (dual-slope, used in 3½-digit DMMs), TLC5510 (flash).

What is tri-state logic and how does it help in signal management?
Tri-state logic is a gate whose output can be logic 0, logic 1, or a third high-impedance (Hi-Z) state, selected by an ENABLE pin. In Hi-Z the output is electrically disconnected. (a) Introduction — CMOS & TTL

TTL (Transistor-Transistor Logic) is a bipolar family: fast, but draws steady power and has limited fan-out. CMOS (Complementary MOS) uses complementary NMOS + PMOS transistors: very low static power, high noise margin, and high fan-out. Because the two families use different voltage and current levels, interfacing them needs care.

NAND Gate using CMOS

A 2-input CMOS NAND uses two PMOS in parallel (pull-up) and two NMOS in series (pull-down). Output is LOW only when both A and B are HIGH (both NMOS conduct).

NOR Gate using CMOS

A CMOS NOR is the dual: two PMOS in series (pull-up) and two NMOS in parallel (pull-down). Output is HIGH only when both A and B are LOW.

Modern Relevance

In 2026, virtually all VLSI — CPUs, GPUs, SoCs, microcontrollers — is built in CMOS because of its near-zero static power, scalability, and high noise immunity. TTL/Schottky survive only in legacy glue logic and education. Level-translator ICs (TI 74LVC, NXP autorange) handle the mixed-voltage interfacing that this question describes whenever a 1.8 V SoC must talk to 3.3 V or 5 V peripherals.

Conclusion

CMOS and TTL interface by fixing two mismatches — voltage level (pull-up for TTL→CMOS) and current drive (buffer for CMOS→TTL). CMOS gates are built from complementary pull-up (PMOS) and pull-down (NMOS) networks: parallel-P/series-N gives NAND, series-P/parallel-N gives NOR. CMOS wins on power, fan-out, and noise margin, while Schottky-TTL wins on raw speed — which is why CMOS dominates modern chips.

Q3★★★ Error/Hamming codes · both years15M
Explain error-detecting and error-correcting codes — parity, block coding and Hamming distance. A 7-bit message is received with even parity as 1110101. Locate and correct the single-bit error (show parity-bit positions P1 P2 P4, the parity checks, the syndrome and the corrected word).
Introduction

When data travels over a noisy channel, bits can flip. Error-detecting codes add redundancy so the receiver can spot an error; error-correcting codes add enough redundancy to also locate and fix it. The Hamming code is the classic single-error-correcting (SEC) code.

Key Concepts
  • Parity: one extra bit so the total number of 1s is even (even parity) or odd (odd parity). Detects any single-bit error but cannot correct it.
  • Block coding: data is grouped into fixed-size blocks; redundant check bits are computed per block. Hamming codes are a block code.
  • Hamming distance (d): number of bit positions in which two code words differ. A code with minimum distance d can detect d−1 and correct ⌊(d−1)/2⌋ errors. Hamming code has d = 3 → corrects 1, detects 2.
  • Parity-bit positions: bits at positions that are powers of 2 (1, 2, 4, 8…) are parity bits; the rest are data.
Worked Single-Error Correction

The 7-bit word uses Hamming(7,4): positions 1,2,4 are parity (P1,P2,P4) and 3,5,6,7 are data. Number positions left → right as 1…7. Received word:

P1 checks positions 1,3,5,7: bits 1,1,1,1 → sum = 4 → even → C1 = 0 ✓
C2
P2 checks positions 2,3,6,7: bits 1,1,0,1 → sum = 3 → odd → C2 = 1 ✗
C4
P4 checks positions 4,5,6,7: bits 0,1,0,1 → sum = 2 → even → C4 = 0 ✓

Syndrome = C4 C2 C1 = 0 1 0 (binary) = 2. A non-zero syndrome means an error at position 2.

fix
Flip bit at position 2: received 1 1 1 0 1 0 1 → corrected 1 0 1 0 1 0 1.

Corrected word = 1010101 ✓ (the single-bit error was P2 at position 2).

Modern Relevance

Hamming-style ECC is everywhere in 2026: ECC RAM in servers (SECDED on every 64-bit word), NAND-flash controllers, spacecraft telemetry, and QR codes (Reed-Solomon, a stronger relative). The same syndrome idea underlies CRC checks in Ethernet and storage.

Conclusion

Parity detects but cannot correct; Hamming codes add log-many parity bits placed at power-of-2 positions so the failed-check syndrome points straight to the bad bit. For the received word 1110101, the syndrome 010 located the error at position 2, giving the corrected word 1010101. This single-error-correction is the foundation of all modern ECC memory.

(a) BCD Adder — Working Principle

A BCD adder adds two BCD digits (each 0–9) and produces a valid BCD result. It uses a normal 4-bit binary adder, then a correction circuit.

  • Step 1: add the two 4-bit BCD numbers with a binary adder → sum S3S2S1S0 + carry K.
  • Step 2 — detect invalid result: a correction is needed when the binary sum > 9 (1010–1111) or a carry-out K is produced. The condition is C = K + S3·S2 + S3·S1.
  • Step 3 — correct: if C = 1, add 0110 (6) to the sum using a second adder, and send carry to the next BCD digit.

Q7★★★ ADC types + conv-time numerical · both years15M
Explain and compare successive-approximation, dual-slope and counting-type A/D converters with block diagrams. Discuss sampling, quantization and encoding. Solve: (i) conversion time of a 12-bit counting ADC with a 1 µs clock; (ii) the number of clock cycles a SAR ADC needs for an 18.1 V input with a 32 V reference (8-bit).
Introduction — The ADC Pipeline

An ADC turns a continuous voltage into bits in three stages: sampling (measure at fixed time steps, fs ≥ 2·fmax), quantization (round each sample to one of 2n levels, error ≤ ½ LSB), and encoding (write each level as an n-bit code). The architecture decides the speed-vs-cost trade-off.

1. Successive-Approximation (SAR) ADC

A binary-search converter. A SAR register sets bits MSB→LSB; a DAC feeds the trial voltage to a comparator, which keeps or clears each bit. Takes exactly n clock cycles.

2. Counting (digital-ramp) ADC

A counter drives a DAC whose output ramps up; when it just exceeds Vin, the comparator stops the counter and the count is the result. Worst case 2n clocks.

3. Dual-Slope (integrating) ADC

Integrate Vin for a fixed time T1, then integrate a reference of opposite sign until the integrator returns to zero (time T2 ∝ Vin). The count during T2 is the output. Slow but very accurate and noise-rejecting — averages noise over T1.

Sampling, quantization and encoding are common to all — the architecture only changes how the quantization search is performed.
