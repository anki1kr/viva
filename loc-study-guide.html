01 Memory Organisation & Hierarchy

A computer needs storage that is fast, large, and cheap — but no single technology gives all three. The solution is a hierarchy: small fast memory close to the CPU, large slow memory far away. Each level acts as a cache for the level below it.

1.1 What is Memory Organisation?

Definition Memory organisation — arrangement of storage units so the CPU receives data with minimum delay at minimum cost.

Key design goals

  • Speed — match CPU clock as closely as possible
  • Capacity — hold the entire program + data
  • Cost per bit — keep total system price reasonable
  • Reliability — minimise errors, support persistence where required

1.2 Hierarchical Memory Organisation

Definition Memory hierarchy — layered storage where each level closer to the CPU is faster, smaller, and costlier per bit; exploits locality of reference.

The pyramid (top = fastest)

Memory Hierarchy Pyramid
Fast, small, costly storage near the CPU; slow, large, cheap storage at the base.

1.3 Locality of Reference — the engine of the hierarchy

Temporal Locality
A location used now is likely to be used again soon. e.g. loop counter i.
Spatial Locality
A location used now means nearby locations are likely to be used too. e.g. walking an array.

1.4 Performance — Effective (Average) Access Time

Definition With a hit rate h in the upper level (e.g. cache) and access times T1 (cache) and T2 (main memory):

Tavg = h · T1 + (1 − h) · (T1 + T2)
Example Cache: 10 ns, Main memory: 100 ns, hit rate 90 %.
Tavg = 0.9 × 10 + 0.1 × (10 + 100) = 9 + 11 = 20 ns

Quick Quiz — Topic 1

Q1. Why is a memory hierarchy used instead of one big fast memory?
Because no single technology offers high speed, large capacity, and low cost per bit at the same time. The hierarchy exploits locality of reference to deliver near-cache speeds at near-disk capacities.
Q2. Order from fastest to slowest: SSD, Cache, Register, HDD, Main memory.
Register → Cache → Main memory → SSD → HDD.
Q3. Cache 5 ns, RAM 80 ns, hit rate 95 %. Compute Tavg.
0.95 × 5 + 0.05 × (5 + 80) = 4.75 + 4.25 = 9 ns.
Q4. Distinguish temporal vs spatial locality.
Temporal = same address re-accessed soon (loop variable). Spatial = nearby addresses accessed (sequential array walk).

02 Semiconductor Memory & Interleaving

Modern main memory and cache are built from semiconductor chips. We classify them by how they store data (RAM vs ROM, static vs dynamic, volatile vs non-volatile), and we use interleaving to make slow DRAM look faster to the CPU.

2.1 Semiconductor Memory Technologies

Definition Semiconductor memory stores bits in transistor/capacitor circuits on silicon; fast (ns access), most types volatile — data lost on power-off.

Family tree

2.2 Memory Interleaving

Definition Memory interleaving splits main memory into independent banks accessed in parallel or overlapped. Consecutive addresses spread across banks, multiplying effective bandwidth.

Why it works

With 4 banks, four successive accesses start back-to-back, dropping average time per word to ~25 ns.

Two flavours

High-Order Interleaving
  • MSB bits select the bank module
  • Consecutive words land in same bank
  • No parallelism; used for fault isolation
Low-Order Interleaving
  • LSB bits select the bank module
  • Consecutive words spread across all banks
  • Maximum parallelism; used by CPUs
Address split — 4 banks, 16 words each (6-bit address)
Example — speed-up DRAM bank cycle = 100 ns. With 4-way low-order interleaving, a sequential burst of 4 words finishes in ≈ 100 + 3×25 = 175 ns instead of 4×100 = 400 ns. Effective bandwidth ≈ 4×.
Pitfall Interleaving helps only when accesses hit different banks. Same-bank access (stride = number of banks) causes a bank conflict.

Quick Quiz — Topic 2

Q1. Why does DRAM need refreshing while SRAM does not?
DRAM stores each bit on a tiny capacitor that leaks charge in milliseconds. The contents must be periodically read and written back to restore the charge. SRAM uses a flip-flop which holds state as long as power is on.
Q2. Which ROM type can be erased with UV light?
EPROM (Erasable PROM). EEPROM and Flash are erased electrically.
Q3. With 4-way low-order interleaving and bank cycle time 80 ns, what is the time to fetch 4 consecutive words?
~ 80 + 3 × 20 = 140 ns if accesses are pipelined every 20 ns; otherwise 80 ns to start all four overlapping ⇒ ≈ 80 ns total when fully pipelined.
Q4. Which interleaving scheme exposes a contiguous address chunk per module?
High-order interleaving — high address bits select the module, so a whole contiguous block sits in one bank.

03 Cache Memory — Mapping, Size, Replacement

Cache is small, fast SRAM placed between the CPU and main memory. The questions every designer asks are: "Where can a memory block sit in the cache?" (mapping), "How big should blocks be?" (size trade-off), and "Which block do we evict on a miss?" (replacement).

3.1 What is Cache Memory?

Definition Cache memory — fast SRAM between CPU and RAM; exploits locality to cut average access time.

Hit, Miss, Hit-Ratio

  • Hit — requested word is found in cache.
  • Miss — not in cache; must fetch the entire block from main memory.
  • Hit ratio (h) = hits / total accesses.
  • Miss penalty = extra time spent on a miss.

3.2 Cache Memory Mapping Techniques

Definition Mapping function decides which cache line a block occupies: Direct, Fully Associative, or Set Associative.

Standard address split

Address fields vary by mapping scheme:

SchemeAddress Split
Direct| TAG | LINE | OFFSET |
Fully Associative| TAG | OFFSET |
Set Associative| TAG | SET | OFFSET |

(a) Direct Mapping

Definition Each main-memory block is mapped to exactly one cache line, given by:
line index = (block address) mod (number of cache lines)
Pros
  • One comparator — simple hardware
  • Fastest lookup, lowest cost
Cons
  • Conflict misses: hot blocks evict each other

(b) Fully Associative Mapping

Definition A block can sit in any cache line; all tags compared in parallel on lookup.
Pros
  • No conflict misses
  • Highest hit ratio for given size
Cons
  • Comparator per line — expensive and slow

(c) Set-Associative Mapping

Definition k-way set associative: S sets of k lines; block maps to one set, occupies any line within it.
set index = (block address) mod (number of sets)
Example 64 lines, 4-way ⇒ 16 sets; block 100 → set 100 mod 16 = 4 (any of 4 lines).

3.3 Cache Size vs Block Size — the trade-off

Cache size

Larger cache ⇒
  • ↑ Hit ratio (more data fits)
  • ↑ Cost & chip area
  • ↑ Access time (bigger arrays)
  • ↑ Power consumption
Smaller cache ⇒
  • ↓ Hit ratio
  • ↓ Cost & access time
  • Fits closer to CPU (e.g. L1)

Block (line) size

Hit Ratio vs Block Size
Hit ratio peaks for moderate block sizes; falls when blocks are too small (poor spatial locality) or too large (cache pollution).
  • Too small: spatial locality wasted; frequent misses.
  • Sweet spot: locality exploited; miss penalty manageable.
  • Too large: fewer blocks fit; causes cache pollution.
Rule of thumb Block size scales with cache size (e.g., 4 KB → 16–32 B; 1 MB L2 → 64–128 B).

On a miss into a full cache, a replacement policy picks which block to evict.

Quick Quiz — Topic 3

Q1. A cache has 64 lines. Which line does block 200 map to under direct mapping?
200 mod 64 = 8 ⇒ line 8.
Q2. Why does fully associative mapping have the highest cost?
It needs one tag comparator per cache line so all tags can be checked in parallel. With n lines that's n comparators — silicon-expensive and power-hungry.
Q3. Why doesn't direct mapping need a replacement algorithm?
Each block has exactly one possible cache line. On a miss the resident block is simply overwritten — there is no choice to make.
Q4. What happens to hit ratio if block size becomes too large?
Hit ratio falls. Fewer blocks fit in the cache (more capacity / conflict misses) and miss penalty rises because more bytes must be transferred per miss — this is cache pollution.
Q5. Reference string 7 0 1 2 0 3 0 4, 3-line cache, LRU. Count misses.
Misses on 7, 0, 1, 2 (evict 7), 0 (hit), 3 (evict 1), 0 (hit), 4 (evict 2) ⇒ 6 misses, 2 hits.

04 I/O Device Interface & Transfer Techniques

CPUs run at GHz speeds; keyboards run at human speeds; disks at ms speeds. The I/O interface bridges these mismatches, and the transfer technique decides who babysits the transfer — the CPU itself, an interrupt mechanism, or a dedicated controller.

4.1 I/O Device Interface

Definition I/O interface (I/O module/controller) — hardware bridge between a device and the system bus, exposing uniform CPU-facing registers.

Why we need an interface

  • Speed mismatch — CPU is millions of times faster than typical devices.
  • Data-format mismatch — device sends serial bits, CPU works with words.
  • Signal-level mismatch — device may use 12 V, CPU uses 1 V.
  • Variety — many devices, one CPU bus.

Internal structure of an interface

I/O Interface — Internal Structure
Standard register layout the CPU uses to talk to any I/O device.

4.2 I/O Transfer Techniques — overview

Three classical techniques, with rising sophistication and falling CPU load:

1. Programmed (Polled) I/O 2. Interrupt-Driven I/O 3. Direct Memory Access (DMA)

4.3 Program-Controlled (Programmed / Polled) I/O

Definition CPU polls device status register in a tight loop; every byte travels through a CPU register. 
// Polling loop pseudocode
while (status_reg.READY == 0) // busy-wait
    ;
data = data_reg;            // CPU does the move
store_to_memory(data);
Pros
  • Simplest hardware — no interrupt logic
  • Deterministic, predictable timing
Cons
  • CPU is 100% busy during wait
  • Wastes cycles on slow devices
  • Cannot multitask while polling

4.4 Interrupt-Driven I/O

Definition CPU issues I/O, resumes other work; device raises interrupt when ready, CPU runs ISR, then returns.

Step-by-step

  1. CPU issues a "start I/O" command to the device.
  2. CPU goes back to running other instructions.
  3. Device finishes ⇒ raises INT line.
  4. CPU acknowledges, pushes PC + flags onto stack.
  5. CPU jumps to the device's ISR (via interrupt vector).
  6. ISR moves data, clears interrupt, returns (IRET).
  7. CPU resumes the interrupted program.
Interrupt-Driven I/O Timeline
CPU runs unrelated work during device busy phase; an interrupt diverts it to the ISR only when the device is ready.

Handling multiple interrupts

  • Polling — ISR checks each device flag in turn (slow, simple).
  • Vectored — device supplies an address that points directly to its ISR (fast).
  • Daisy-chain — devices are wired in series; INTA signal propagates until a requesting device responds.
  • Priority interrupt controller (PIC) — dedicated chip (e.g. 8259) decides which device wins.
Maskable vs Non-Maskable Maskable: disabled via interrupt-enable flag; CPU may ignore them.
NMI (power failure, hardware faults): cannot be ignored.
Pros
  • CPU free during device wait
  • Supports multitasking
  • More responsive than polling
Cons
  • Data still flows through CPU
  • Context-switch overhead per interrupt
  • Not ideal for high-rate devices

4.5 Direct Memory Access (DMA)

Definition DMA controller transfers data between I/O device and memory without CPU touching every word. CPU sets source, destination, count; gets one interrupt when the block is done.

DMA controller registers

  • Memory Address Register — where in RAM to read/write
  • Word Count Register — how many words to move
  • Control Register — direction (read/write), mode
  • Status Register — done / error flags

Three DMA transfer modes

ModeHow it shares the busBest for
Burst (Block) TransferDMA grabs the bus and transfers the whole block in one go; CPU is starved.Tape, disk — high-rate devices
Cycle StealingDMA takes the bus for one word, releases it, takes it again later. CPU runs in between.Moderate-rate devices
Transparent / HiddenDMA uses only those bus cycles when CPU isn't using the bus (e.g. internal compute cycles).Best when CPU and DMA must run concurrently

Steps in a DMA transfer

  1. CPU programs the DMA controller (memory address, count, direction).
  2. Device asserts DMA Request (DREQ).
  3. DMA asserts HOLD; CPU releases bus, replies HLDA.
  4. DMA places address on address bus, generates RD/WR — data flows directly between device and memory.
  5. Count = 0: DMA releases bus, raises interrupt to CPU.
DMA Controller Wiring
DMA arbitrates the bus with HOLD/HLDA, hand-shakes with the device via DREQ/DACK, and moves data straight between memory and device.
Example — disk read Reading a 4 KB sector with polled I/O could occupy CPU for thousands of microseconds. With DMA, CPU spends ≈ 1 µs on setup, runs other code for the entire transfer, and gets a single interrupt on completion.

Quick Quiz — Topic 4

Q1. Why is programmed I/O unsuitable for a hard disk?
The CPU would busy-wait inside the polling loop for milliseconds per access, wasting millions of cycles and blocking all other work.
Q2. What signals does the CPU and DMA exchange to share the bus?
The DMA asserts HOLD (a.k.a. BR — Bus Request); the CPU completes the current cycle, releases the bus, and asserts HLDA (Bus Grant). When done, the DMA deasserts HOLD.
Q3. Difference between cycle-stealing and burst DMA.
Cycle stealing moves one word per bus acquisition then releases the bus, letting CPU run. Burst mode transfers the entire block while holding the bus, starving the CPU until done.
Q4. What is an NMI? Why is it useful?
Non-Maskable Interrupt: cannot be disabled; reserved for catastrophic events like power failure or hardware errors.
Q5. Which I/O technique still needs the CPU to move every word but doesn't make it busy-wait?
Interrupt-driven I/O — the CPU is free between interrupts, but each transfer still passes through a CPU register inside the ISR.

05 Control Unit — Hardwired vs Microprogrammed

The Control Unit (CU) is the conductor of the CPU. For each instruction it generates a precisely-timed sequence of control signals that tell the ALU, registers, memory, and I/O what to do. There are two classical ways to build it.

5.1 Role of the Control Unit

Definition The control unit decodes each fetched instruction and generates the sequence of binary control signals that drive the data path through the fetch → decode → execute cycle.

Inputs and outputs

Inputs:                            Outputs (control signals):
  - IR  (instruction register)       - Read / Write to memory
  - Flags (Z, C, N, V ...)           - Register select & enable
  - Clock                            - ALU function select
  - Status from data path            - Bus drivers, latches

5.2 Hardwired Control Unit

Definition A hardwired control unit uses fixed combinational/sequential logic to generate control signals; wiring is permanent after fabrication.

Design steps

  • List instructions and their micro-operations.
  • Build a state diagram of execution sequence.
  • Write Boolean function: signal = f(opcode, state, flags)
  • Minimise with K-map / Q-M.
  • Implement: combinational logic + state counter.

Block diagram

Hardwired Control Unit — Block Diagram
Fixed combinational logic decodes opcode + state + flags into control signals; a clocked state counter drives sequencing.
Pros
  • Very fast — pure gate delays
  • Used by RISC CPUs (small, regular ISAs)
  • Efficient for small instruction sets
Cons
  • Inflexible — adding an instruction = redesign + refab
  • Hard to debug
  • Becomes very complex for large ISAs

5.3 Microprogrammed Control Unit

Definition A microprogrammed CU stores control signals as microinstructions in Control Memory (ROM/PROM); executing an instruction steps through its stored microprogram.

Key components

  • Control Memory (CM) — ROM/PROM holding all microinstructions.
  • Control Address Register (CAR / µPC) — points to the next microinstruction.
  • Control Buffer Register (CBR) — holds the current microinstruction being executed.
  • Sequencer — decides the next CAR value (PC+1, branch, mapping from opcode).

Quick Quiz — Topic 5

Q1. Why is hardwired control faster than microprogrammed?
Hardwired uses pure gate logic; microprogrammed must fetch a microinstruction from ROM each clock, adding latency.
Q2. Which control style suits CISC and why?
Microprogrammed — CISC's many complex instructions are easier to encode in ROM than in hardwired gates.
Q3. Compare horizontal vs vertical microinstructions in one line.
Horizontal = one bit per control signal — wide, fast, parallel. Vertical = encoded fields — narrow, slower, needs a decoder.
Q4. What does the CAR (Control Address Register) do?
CAR holds the address of the next microinstruction — the µPC of control memory.

06 Pipelining & Pipeline Hazards

Pipelining overlaps the execution of multiple instructions like an assembly line — while instruction i is being executed, instruction i+1 is being decoded, and i+2 is being fetched. Throughput multiplies, but only when nothing breaks the flow.

6.1 What is Pipelining?

Definition
  • Splits each instruction into k stages.
  • Multiple instructions overlap; CPI ≈ 1 at steady state.

Classical 5-stage RISC pipeline

StageNameWhat happens
1IF — Instruction FetchRead next instruction from memory; PC ← PC + 4
2ID — Instruction Decode / Register ReadDecode opcode, read operands from register file
3EX — Execute / Address CalcALU computes result or address
4MEM — Memory AccessLOAD/STORE accesses data memory
5WB — Write BackWrite result back to register file

Time-space diagram

5-Stage Pipeline — Time-Space Diagram
Staggered stages; one instruction completes per cycle once full.

Speed-up & throughput

For an ideal k-stage pipeline executing n instructions:

  Time non-pipelined  =  n · k · t
  Time pipelined      =  (k + n − 1) · t
  Speed-up   S        =  (n · k) / (k + n − 1)
                      →  k        as  n → ∞
Example
  • 5-stage, 100 instructions, t = 1 ns.
  • Non-pipelined: 500 ns; pipelined: 104 ns; S ≈ 4.81×.

6.2 Pipeline Hazards — when the line jams

Definition A pipeline hazard prevents the next instruction from executing in the next clock cycle, forcing stalls (bubbles) that reduce throughput.

The three families of hazards

(a) Structural Hazard — resource conflict

Definition Occurs when two instructions in different stages want the same hardware resource in the same cycle (e.g. one memory port for both IF and MEM).
  • Cause: single memory shared by IF and MEM, single ALU shared by EX and address calc, etc.
  • Cure: duplicate the resource — separate I-cache and D-cache (Harvard architecture), or stall.

(b) Data Hazard — the value isn't ready yet

Definition Occurs when an instruction depends on the result of a previous instruction that hasn't yet written its output.

Three sub-types based on the dependency direction:

TypePatternExample
RAW (Read After Write) — true dependencyI1 writes, I2 readsADD R1,… / SUB …, R1
WAR (Write After Read) — anti-dependencyI1 reads, I2 writes same regReorder issues
WAW (Write After Write) — output dependencyI1 writes, I2 writes same regOut-of-order pipelines

Cures for data hazards

  • Stalling (interlock) — insert NOPs / bubbles until value is ready.
  • Forwarding (bypassing) — feed the ALU output of one stage directly into the input of an earlier stage in the next cycle.
  • Operand forwarding from MEM/WB — for LOADs, value available one cycle after EX.
  • Compiler scheduling — reorder independent instructions to fill the slot.

(c) Control (Branch) Hazard

Definition Occurs when a branch changes the PC; already-fetched instructions behind the branch may be wrong and must be discarded.
  • Stall until branch resolved — simple but slow (2-3 bubble cycles).
  • Branch prediction — guess taken/not-taken; flush on mis-predict.
  • Delayed branch — execute the instruction in the "delay slot" regardless of branch outcome.
  • Branch target buffer (BTB) — cache recently-taken branch targets.

Control (branch)PC change after fetchBranch prediction, delayed branch, flush
Memory hook — three hazards "S D C — Structure, Data, Control". Data sub-types: RAW · WAR · WAW (only RAW is a true dependency).
Pitfall Deeper pipelines win on clock frequency but pay a much higher flush cost per branch mis-predict.

Quick Quiz — Topic 6

Q1. Compute the speed-up of a 4-stage pipeline running 50 instructions (vs non-pipelined).
S = (50·4) / (4 + 50 − 1) = 200 / 53 ≈ 3.77×.
Q2. What hazard does the following cause? LW R1, 0(R2); ADD R3, R1, R4
A RAW (load-use) hazard — ADD needs R1 before LW's MEM stage finishes; requires one bubble even with forwarding.
Q3. How does a Harvard architecture eliminate one structural hazard?
By providing separate instruction and data memories/caches, IF and MEM can both access "memory" in the same cycle without conflict.
Q4. What is a "delay slot"?
The instruction position immediately after a branch that is always executed, regardless of whether the branch is taken — used by some RISC ISAs (e.g. classic MIPS) to hide branch latency.
Q5. Which of WAR / WAW / RAW is a "true" data dependency?
Only RAW (Read After Write) — WAR and WAW are name dependencies removable by register renaming.

x86 = canonical CISC ISA (Intel 8086 → modern Core/Xeon); parallel processors = multiple PEs working simultaneously, classified by Flynn's taxonomy.

7.1 x86 Architecture — overview

Definition x86 = CISC ISA family (Intel 8086, 1978); variable-length instructions, rich addressing modes, segmented memory; Intel and AMD today.

The 8086 register set (foundation of all x86)

GroupRegistersPurpose
General-purpose (16-bit)AX, BX, CX, DXArithmetic, addressing, counter, data

Each GP register splits into 8-bit halves (AH/AL…); 32-bit = EAX; 64-bit = RAX + R8–R15 + XMM/YMM/ZMM SIMD.

Two functional units in the 8086 — overlap of fetch and execute

8086 — BIU / EU Split
BIU prefetches up to 6 bytes ahead while EU decodes and executes — primitive pipelining inside the 8086.

Memory segmentation (real-mode 8086)

Definition The 8086 has 20 address pins (1 MB physical) but only 16-bit registers. Memory is divided into segments; the physical address is computed as:
Physical = (Segment × 16) + Offset
Example CS = 1234h, IP = 0005h ⇒ Physical = 12340h + 0005h = 12345h

7.2 Parallel Processors

Definition Multiple PEs working simultaneously; parallelism spans bit, instruction, data, and task levels.

Flynn's Taxonomy — the four classes

ClassInstructionDataExample
SISDSingleSingleClassical uniprocessor (8086, single-core CPU)
SIMDSingleMultipleGPU, vector units (AVX), Cray-1
MISDMultipleSingleRare — fault-tolerant systolic arrays
MIMDMultipleMultipleMulticore CPUs, clusters, supercomputers
Memory hook — Flynn "SI & MI · SD & MD". Two letters = two axes. SI/MI = how many instruction streams; SD/MD = how many data streams.

MIMD — two memory models

Shared-Memory (Tightly Coupled)
All CPUs see one common physical memory. Communication via load/store. e.g. multicore PCs. Bottleneck: bus / cache coherence (MESI protocol).
Distributed-Memory (Loosely Coupled)
Each CPU has private memory; communicate by message passing. e.g. clusters, supercomputers (MPI). Scales to thousands of nodes.

Forms of parallelism — quick map

LevelGranularityExample
Bit-levelWithin one operation64-bit add in one cycle

Data-level (DLP)Same op on many data itemsSIMD, GPU, vector processor
Task-level (TLP)Independent threads / processesMulticore, cluster, MapReduce

Interconnection networks (how processors talk)

  • Bus — simple, cheap, doesn't scale beyond ~ 16 cores.
  • Crossbar — non-blocking, expensive (n² switches).
  • Mesh / Torus — scalable, used in supercomputers.
  • Hypercube — log2(n) hops between any two nodes.
  • Network-on-Chip (NoC) — modern many-core processors.

Amdahl's Law — limit of speed-up

Definition If P is the fraction of a program that can be parallelised and N is the number of processors:

Speed-up = 1 / ( (1 − P) + P / N )
Example P = 0.9, N = 10 ⇒ S = 1 / (0.1 + 0.09) = 5.26×. Even with N → ∞, max ≈ 1 / 0.1 = 10×. The serial fraction caps the gain.

Quick Quiz — Topic 7

Q1. Compute the physical address from CS = 2000h, IP = 0034h.
2000h × 16 + 0034h = 20000h + 0034h = 20034h.
Q2. What does the BIU do in the 8086?
The Bus Interface Unit calculates physical addresses, fetches instructions into a 6-byte queue, reads/writes data from memory and I/O — all in parallel with the EU's execution.
Q3. Classify a modern multicore CPU under Flynn's taxonomy.
MIMD — each core has its own instruction stream and operates on its own data, while sharing memory.
Q4. A program is 80 % parallelisable. With 16 processors, what is the speed-up?
S = 1 / (0.2 + 0.8/16) = 1 / (0.2 + 0.05) = 1 / 0.25 = 4×.
Q5. Which Flynn class do GPUs fit best?
SIMD (sometimes called SIMT) — one instruction is broadcast to many cores that all process different data lanes in lock-step.
Q6. Distinguish shared-memory and distributed-memory MIMD.
Shared-memory: all processors access one address space (loads/stores); needs cache coherence. Distributed-memory: each processor owns its memory and communicates by explicit message passing — scales further but harder to program.

1. Memory Organisation
  • Def: Arrangement of storage so CPU gets data with min delay + min cost.
  • Goals: Speed · Capacity · Cost/bit · Reliability.
  • Exam: Define → 3 goals → EAT formula.
2. Hierarchical Memory Org.
  • Def: Layered storage — faster/smaller/costlier near CPU; exploits locality.

  • Formula: Tavg = h·T1 + (1−h)(T1+T2).
  • Exam: Pyramid diagram + comparison table + EAT example.
3. Semiconductor Memory
  • Def: Bits in transistor/capacitor circuits on silicon.
  • SRAM: 6T flip-flop · no refresh · fast → cache.

  • ROM family: MROM→PROM→EPROM→EEPROM→Flash.
  • Exam: Family tree + SRAM/DRAM table. Mnemonic: "My Programmer Eats Every Flash".
4. Memory Interleaving
  • Def: Splits memory into N banks; consecutive addresses across banks → parallel access.
  • Low-order (LSB): max parallelism — CPU uses this.
  • High-order (MSB): same bank for sequential → no parallelism.
  • Speed-up: 4-bank: 4×100 ns → ≈175 ns.
  • Exam: Def + low/high address split diagram + speed example. Pitfall: bank conflict when stride = N.
5. Cache Memory
  • Def: Fast SRAM between CPU and RAM; exploits locality to cut Tavg.
  • Hit / Miss / Hit-ratio h = hits / total accesses.
  • EAT: h·T1 + (1−h)(T1+T2).
  • Exam: Def + hit/miss + EAT formula + numeric example. Write-through vs write-back.
6. Cache Mapping Techniques
  • Direct: line = block mod #lines. 1 comparator. Conflict misses.
  • Fully Assoc: any line. N comparators. Best hit rate, most expensive.
  • Set-Assoc (k-way): set = block mod #sets; any of k lines. Best balance.
  • Address split: TAG|LINE|OFFSET / TAG|OFFSET / TAG|SET|OFFSET.
  • Exam: All 3 definitions + comparison table (5 rows). Mnemonic: "Direct=One·Fully=Any·Set=Any-in-One".
7. Cache Size vs Block Size
  • Larger cache: ↑ hit rate · ↑ cost · ↑ access time.
  • Too-small block: poor spatial locality → frequent misses.
  • Too-large block: cache pollution → fewer blocks fit → hit rate falls.
  • Optimum: 32–128 B (scales with cache size).
  • Exam: Bell-curve graph + trade-off bullets.
8. Cache Replacement Algorithms
  • FIFO: evict longest-resident. Simple, ignores usage.
  • LRU: evict least-recently-used. Best practical hit rate.
  • LFU: evict least-frequently-used. Old popular blocks may persist.
  • Random: trivial hardware, near-LRU in practice.
  • OPT: theoretical best — needs future knowledge.
  • Exam: Table + LRU reference-string worked example.
9. I/O Device Interface
  • Def: Hardware bridge between device and system bus.
  • Why needed: speed / data-format / signal-level mismatch.

  • MMIO vs Port-mapped: same addr space vs separate I/O space.
  • Exam: Def → why needed → 4 registers → block diagram → MMIO table.
10. Programmed (Polled) I/O
  • Def: CPU polls device status register in tight loop; every byte through CPU.
  • CPU: 100% occupied during transfer — cannot multitask.
  • Use: simple/embedded devices only.
  • Exam: Pseudocode + pros/cons table.
11. Interrupt-Driven I/O
  • Def: CPU issues I/O, continues other work; device raises INT when ready.
  • 7 steps: START → run → INT → push PC+flags → ISR → IRET → resume.
  • Still: data flows through CPU per word (inside ISR).
  • Exam: 7-step list + timeline diagram (CPU lane + Device lane + INT arrow).

  • Def: DMA controller moves data device↔memory WITHOUT CPU touching each word.

Q11. Define a flip-flop. List four common types.

Answer (4 marks)
  • SR FF: Set/Reset; invalid state when S = R = 1.

  • T FF: Toggle on T = 1; built from JK with J = K = T. Used in counters.

Von Neumann Architecture
CPU (ALU + CU + Registers) talks to a single memory over one shared bus — the classic stored-program design.

Components

  • CPU — has the ALU (arithmetic/logic), CU (decodes & sequences), and registers (PC, IR, MAR, MDR, AC).
  • Memory — one address space holds both instructions and data.
  • I/O subsystem — keyboards, displays, disks via I/O controllers.
  • System bus — address, data, and control lines connecting the three.

Advantages & limitation

  • + Simple, flexible — same hardware runs any program (just load different code).
  • + Cheap: only one memory and one bus to design.
  • − Von Neumann bottleneck — fetch and data access compete on one bus; cache mitigates this.

Q7. Explain DMA — definition, working, and block diagram.

Definition Dedicated controller moves data blocks device ↔ memory without per-word CPU involvement. CPU initiates, gets one IRQ on completion.
Scoring template — 8 marks
  • Definition (dedicated controller, no per-word CPU involvement) — 1 mark
  • HOLD/HLDA and DREQ/DACK signal pairs (bus arbitration) — 2 marks
  • Block diagram (CPU · DMA · Memory · I/O Device labelled) — 2 marks
  • 5-step transfer sequence (load → start → DREQ → move → IRQ) — 2 marks

Steps in a DMA transfer

  1. CPU loads DMA with: src addr, dst addr, byte count, direction.
  2. CPU issues start → resumes other work.
  3. Device signals DREQ → DMA seizes bus (burst or cycle-steal).
  4. DMA moves one word/cycle, decrements counter.
  5. Counter = 0 → DMA raises IRQ to CPU.
DMA Controller — Block Diagram

Modes

  • Burst mode: DMA holds the bus until the entire block is transferred (fast but stalls CPU).
  • Cycle-stealing: DMA grabs the bus for one word, releases it, repeats — minimal CPU disruption.
  • Transparent mode: DMA only uses the bus when the CPU is not.

Advantages

  • Frees CPU from per-word I/O work — useful for high-bandwidth devices (disk, NIC, GPU).
  • Higher throughput than programmed or interrupt-driven I/O for large blocks.
  • Lower interrupt overhead (one IRQ per block, not per byte).

Scoring template — 8 marks

Definition Cache mapping — the rule that determines which cache line(s) a main-memory block may occupy.

(a) Direct Mapping

  • Each main-memory block maps to exactly one cache line.
  • Line number = Block number mod (Total cache lines).
  • Address split: | Tag | Line index | Block offset |
  • Pro: Fast, simple hardware. Con: High conflict-miss rate (two blocks mapping to the same line evict each other).
Cache line = Block address  mod  N
  (N = number of cache lines)

  Example: 8 cache lines, block 5 → line 5 mod 8 = 5
           block 13 → line 13 mod 8 = 5  (conflict!)

(b) Fully Associative Mapping

  • A memory block may go into any cache line.
  • Address split: | Tag | Block offset | (no line-index field).
  • Requires Content-Addressable Memory (CAM) to search all tags in parallel.
  • Pro: No conflict misses. Con: Expensive hardware; slow for large caches.

(c) Set-Associative Mapping (k-way)

  • Cache divided into sets; each set holds k lines (k-way).
  • Block maps to one set (like direct), but within that set can go into any of the k lines (like fully associative).
  • Address split: | Tag | Set index | Block offset |
  • 2-way and 4-way are most common in real CPUs.
  • Pro: Balances cost and miss rate. Con: Needs replacement policy (LRU/FIFO) within each set.

Definition Pipelining — splits each instruction into k stages executed by dedicated hardware; multiple instructions overlap so throughput ≈ 1 instruction/cycle at steady state.

Classical 5-stage pipeline

StageShortWork done
1IFFetch instruction from memory; PC ← PC + 4
2IDDecode opcode; read register operands
3EXALU executes; address calculation
4MEMLOAD/STORE accesses data memory
5WBWrite result to register file

Speed-up formula

Without pipeline: Time = n × k  (n instructions, k stages)
With pipeline:    Time = k + (n − 1)  (fill + drain)

Speed-up S = n × k / (k + n − 1)
  → As n → ∞,  S → k  (ideal speed-up = number of stages)

Example: k = 5 stages, n = 100 instructions
  S = (100 × 5) / (5 + 99) = 500 / 104 ≈ 4.8×

Q3. State the difference between MAR and MDR.
MAR holds the address of the memory location to be accessed; MDR holds the data being read from or written to that location.
Q4. Why are NAND and NOR called universal gates?
Because every Boolean function — and therefore every digital circuit — can be implemented using only NAND gates (or only NOR gates). This makes IC fabrication simpler and cheaper.
Q5. In a memory of cache 8 ns and RAM 80 ns with hit ratio 0.85, find Tavg.
Tavg = 0.85·8 + 0.15·(8+80) = 6.8 + 13.2 = 20 ns.
Q6. Why is DMA preferred over interrupt-driven I/O for disks?
Disks transfer large blocks at high speed. Interrupting the CPU per word would consume most of its time on context-switching. DMA moves the entire block directly to memory and raises one interrupt at the end, freeing the CPU.
