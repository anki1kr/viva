Exp 01

User Input, Display Values, R Version

InputOutputreadline
Aim

Take user input (name and age), display the values, and print the R version.

R Code
# Take input from user
name <- readline(prompt = "Enter your name: ")
age  <- readline(prompt = "Enter your age: ")

# Display the values
cat("Name:", name, "\n")
cat("Age:", age, "\n")

# Print installed R version
cat("R Version:", R.version$version.string, "\n")
Key Concepts
readline() reads console input as character — takes a prompt argument shown to the user
cat() prints without quotes and supports \n for newlines; print() shows the object with type info
R.version$version.string gets the full installed R version string
Viva Q&A

  • Q1: What is readline() in R?

    • Reads a line of input from the console
    • Takes a prompt argument — text shown to user
    • Always returns input as a character string
    • Example: name <- readline("Enter name: ")

  • Q2: What is the difference between cat() and print()?

    • cat(): prints without quotes, supports \n for newlines
    • print(): shows the object with type info, adds quotes for strings
    • Use cat() for formatted output
    • Use print() to inspect objects

  • Q3: How do you check the R version?

    • Use R.version$version.string for version string
    • Or type R.version for complete details
    • Or version gives a list of all version info
    • Example output: "R version 4.3.1 (2023-06-16)"

  • Q4: What is the difference between <- and = in R?

    • Both assign values to variables
    • <- is the traditional R assignment operator (preferred style)
    • = is used mainly inside function arguments
    • Example: mean(x = c(1,2,3)) vs x <- c(1,2,3) outside functions

  • Q5: What does \n mean in cat()?

    • \n is the newline escape character
    • Moves cursor to next line (like pressing Enter)
    • \t is tab, \\ is a backslash
    • Example: cat("Line1\nLine2\n") prints on two separate lines
Exp 02

Sequence, Mean, Sum

seqmeansum
Aim

Create sequence from 20 to 50, find mean of 20–60, find sum of 51–91.

R Code
# Sequence from 20 to 50
s1 <- seq(20, 50)
cat("Sequence 20-50:\n"); print(s1)

# Mean of 20 to 60
cat("Mean of 20-60:", mean(seq(20, 60)), "\n")

# Sum of 51 to 91
cat("Sum of 51-91:", sum(seq(51, 91)), "\n")
Key Concepts
seq(from, to) generates an integer sequence between two values
mean() calculates arithmetic mean; sum() adds all elements in a vector
length() returns the number of elements in a vector
Viva Q&A

  • Q1: What is the seq() function?

    • Generates a regular sequence of numbers
    • seq(from, to) gives integer sequence
    • Optional: by= controls step size (e.g., seq(1,10,by=2) gives 1,3,5,7,9)
    • Example: seq(20,50) gives 20,21,22,...50

  • Q2: What is the difference between seq() and seq_len()?

    • seq(a, b): generates from a to b
    • seq_len(n): generates 1 to n (faster for simple 1:n sequences)
    • seq_along(x): generates indices 1 to length(x)
    • Example: seq_along(c('a','b','c')) gives 1 2 3

  • Q3: How do you find mean in R?

    • Use mean() function
    • Example: mean(c(10,20,30)) gives 20
    • For a range: mean(seq(20,60)) gives mean of 20 to 60
    • NA values: use mean(x, na.rm=TRUE) to ignore missing values

  • Q4: What does sum() do?

    • Adds all values in a vector
    • Example: sum(1:10) = 55
    • Works on numeric vectors
    • cumsum(x) gives running total (cumulative sum)

  • Q5: What is the formula for arithmetic mean?

    • Mean = Sum of all values / Count of values
    • Mean of 20 to 60 = (20+21+...+60) / 41 = 40
    • In R: sum(seq(20,60)) / length(seq(20,60))
    • Equivalent to mean(seq(20,60))
Exp 03

Vector with Random Integers

Vectorsamplerandom
Aim

Create a vector with 10 random integer values between -50 and +50.

R Code
# sample() picks n random values from a range
# replace=FALSE means no repetition
vec <- sample(-50:50, size = 10, replace = FALSE)

cat("10 random integers between -50 and +50:\n")
print(vec)
cat("Length:", length(vec), "\n")
Key Concepts
sample(x, size, replace) picks random samples from a pool
replace=FALSE means no repeats; replace=TRUE allows same element multiple times
-50:50 is shorthand for seq(-50, 50) — a vector is an ordered collection of same-type elements
Viva Q&A

  • Q1: What is a vector in R?

    • Most basic data structure in R
    • Ordered collection of elements of the SAME type
    • Created with c() function
    • Example: c(10, 20, 30) is numeric vector; c("a","b","c") is character vector

  • Q2: What is sample() in R?

    • Randomly selects elements from a vector or range
    • Parameters: x (pool to pick from), size (how many), replace (repeat or not)
    • Example: sample(1:100, 5) picks 5 random numbers from 1 to 100
    • replace=FALSE: no repetition; replace=TRUE: can repeat

  • Q3: Difference between replace=TRUE and replace=FALSE?

    • replace=FALSE (default): each element can only be picked ONCE
    • replace=TRUE: same element can be picked MULTIPLE times
    • Use FALSE for lottery-style selection
    • Use TRUE for bootstrap sampling

  • Q4: What does -50:50 mean in R?

    • Creates integer sequence from -50 to 50
    • Shorthand for seq(-50, 50, by=1)
    • Total 101 values: -50,-49,...,0,...,49,50
    • The : operator is the fastest way to create simple integer sequences

  • Q5: How do you set a seed for reproducible random numbers?

    • Use set.seed(number) before sample()
    • Example: set.seed(42); sample(1:10, 5) gives SAME result every run
    • Important for reproducible research and debugging
    • Any integer can be used as the seed
Exp 04

Factors of a Number

LoopModulofor
Aim

Find all factors of a given number using a for loop and the modulo operator.

R Code
n <- as.integer(readline(prompt = "Enter a number: "))

cat("Factors of", n, ":")
for (i in 1:n) {
  if (n %% i == 0) {    # %% is modulo — remainder after division
    cat(i, "")
  }
}
cat("\n")
Key Concepts
%% is the modulo operator — returns remainder after division; if n %% i == 0, then i is a factor
for (i in 1:n) { ... } is R's for loop syntax — iterates i from 1 to n
c(vector, value) appends a new value to a vector; as.integer() converts character input to integer
Viva Q&A

  • Q1: What is the modulo operator in R?

    • %% returns the remainder after integer division
    • Example: 10 %% 3 = 1 (10 divided by 3 leaves remainder 1)
    • If n %% i == 0, then i is a factor of n
    • Also used to check even/odd: n %% 2 == 0 means even

  • Q2: How do you write a for loop in R?

    • Syntax: for (variable in sequence) { body }
    • Example: for (i in 1:5) { print(i) }
    • The loop variable takes each value from the sequence
    • Use break to exit early, next to skip current iteration

  • Q3: What is a prime number?

    • A number with exactly 2 factors: 1 and itself
    • Examples: 2, 3, 5, 7, 11, 13, 17, 19
    • 1 is NOT a prime number
    • Check: if length(factors) == 2 then prime

  • Q4: How do you append to a vector in R?

    • Use c(existing, new_value)
    • Example: v <- c(v, 5) adds 5 to vector v
    • R creates a NEW vector each time (immutable style)
    • For large loops, append() function is more efficient

  • Q5: What does as.integer() do?

    • Converts a value to integer data type
    • readline() always returns a character string
    • as.integer("25") converts string "25" to number 25
    • Needed for arithmetic operations on user input
Exp 05

Maximum and Minimum of Vector

maxminsort
Aim

Find maximum and minimum value of a given vector.

R Code
vec <- c(23, 45, 12, 67, 8, 34, 56)
cat("Vector:", vec, "\n")
cat("Maximum:", max(vec), "\n")
cat("Minimum:", min(vec), "\n")
cat("Position of max:", which.max(vec), "\n")
cat("Position of min:", which.min(vec), "\n")
Key Concepts
max() / min() return the largest / smallest value in a vector
which.max() / which.min() return the INDEX (position) of max/min, not the value itself
range(v) returns c(min, max); sort(v) arranges in ascending order (use decreasing=TRUE for descending)
Viva Q&A

  • Q1: What is the difference between max() and which.max()?

    • max(v): returns the maximum value in vector
    • which.max(v): returns the index (position) of the maximum value
    • Example: max(c(3,1,9,5)) = 9; which.max(c(3,1,9,5)) = 3
    • Similarly: min() and which.min()

  • Q2: What does range() return?

    • Returns a vector of two values: c(minimum, maximum)
    • Example: range(c(3,8,1,5)) gives 1 8
    • Equivalent to c(min(x), max(x))
    • Useful to quickly see the spread of data

  • Q3: How do you sort a vector in R?

    • sort(vec): returns sorted vector in ascending order
    • sort(vec, decreasing=TRUE): descending order
    • Original vector is NOT modified (immutable)
    • rev(vec) reverses the vector order

  • Q4: What is the difference between sort() and order()?

    • sort(v): returns the sorted values
    • order(v): returns the indices that would sort the vector
    • Example: order(c(30,10,20)) gives 2 3 1
    • Use order() to sort data frames by a column

  • Q5: Can max() work on character vectors?

    • Yes! For character vectors, max() returns the last value alphabetically (lexicographic order)
    • Example: max(c("banana","apple","cherry")) gives "cherry"
    • Similarly min() gives "apple"
    • Comparison is case-sensitive by default
Exp 06

Convert Matrix to 1D Array

matrixas.vectordim
Aim

Create a matrix and convert it to a 1-dimensional array.

R Code
# Create a 3x3 matrix (fills column by column)
mat <- matrix(1:9, nrow = 3, ncol = 3)
cat("Original Matrix:\n"); print(mat)

# Convert to 1D array using as.vector()
arr <- as.vector(mat)
cat("1D Array:", arr, "\n")
Key Concepts
matrix() creates a 2D matrix, filling elements column-by-column by default
as.vector() flattens a matrix to 1D, reading elements column by column (column-major order)
dim() returns c(rows, cols); nrow() and ncol() get dimensions separately
Viva Q&A

  • Q1: What is a matrix in R?

    • A 2-dimensional data structure with rows and columns
    • ALL elements must be the SAME type
    • Created with matrix(data, nrow, ncol)
    • Dimensions accessed with dim(), nrow(), ncol()

  • Q2: How does matrix() fill elements by default?

    • By column by default (column-major order)
    • matrix(1:9, nrow=3): column 1 gets 1,2,3; column 2 gets 4,5,6; column 3 gets 7,8,9
    • Use byrow=TRUE to fill row by row instead
    • Follows Fortran convention (column-major)

  • Q3: How do you fill a matrix by row?

    • Add parameter byrow=TRUE to matrix()
    • Example: matrix(1:9, nrow=3, byrow=TRUE) fills each row left to right
    • Default is byrow=FALSE (fills by column)
    • Use when your data is logically arranged in rows

  • Q4: What does as.vector() do to a matrix?

    • Converts matrix to a 1D vector
    • Elements are read column by column (left to right)
    • Example: 3x3 matrix → 9-element vector
    • Equivalent to using c(mat) — both give same result

  • Q5: What is the difference between a matrix and an array in R?

    • Matrix: always exactly 2D (rows × columns)
    • Array: can be n-dimensional (2D, 3D, 4D, ...)
    • A matrix IS a 2D array
    • Use array for higher dimensions: array(data, dim=c(r,c,l))
Exp 07

Array of Two 3×3 Matrices

array3Ddim
Aim

Create an array of two 3×3 matrices from two given vectors.

R Code
vec1 <- 1:9    # first 3x3 matrix data
vec2 <- 10:18  # second 3x3 matrix data

# dim = c(rows, cols, layers)
arr <- array(c(vec1, vec2), dim = c(3, 3, 2))

cat("Matrix 1:\n"); print(arr[,,1])
cat("Matrix 2:\n"); print(arr[,,2])
Key Concepts
array(data, dim=c(r,c,layers)) creates a multi-dimensional array
dim=c(3,3,2) means 3 rows, 3 cols, 2 layers — total elements = 3×3×2 = 18
Access individual layers with arr[,,layer_number] — blank means "all" for that dimension
Viva Q&A

  • Q1: What is an array in R?

    • Multi-dimensional data structure
    • All elements must be the same type
    • Created with array(data, dim=c(...))
    • More general than matrix (which is just 2D array)

  • Q2: How is an array different from a matrix?

    • Matrix: exactly 2 dimensions only
    • Array: 2 or MORE dimensions
    • A matrix is a special case of array
    • Arrays are used when data has 3+ dimensions (like layers of matrices)

  • Q3: What does dim=c(3,3,2) mean?

    • 3 rows, 3 columns, 2 layers
    • Like 2 pages of a 3×3 grid
    • Total elements = 3×3×2 = 18
    • Must equal length of data vector

  • Q4: How do you access the 2nd matrix in a 3D array?

    • arr[, , 2] — all rows, all columns, layer 2
    • Empty index means "all" for that dimension
    • arr[1, 2, 1] = element at row 1, col 2, layer 1
    • Think of it like: arr[row, col, layer]

  • Q5: How does array fill elements?

    • Fills in column-major order (column by column)
    • Then fills next layer in same way
    • Example: array(1:18, dim=c(3,3,2)) — first 9 fill layer 1, next 9 fill layer 2
    • Same order as matrix filling
Exp 08

3D Array Using dim()

dimarrayreshape
Aim

Create a 3-dimensional array of 24 elements using the dim() function.

R Code
# Create vector of 24 elements
data <- 1:24

# Assign dimensions to reshape into 3D array
dim(data) <- c(2, 3, 4)  # 2 rows, 3 cols, 4 layers

cat("3D Array (2x3x4):\n")
print(data)
cat("Dimensions:", dim(data), "\n")  # 2 3 4
Key Concepts
dim(x) <- c(r,c,l) can SET dimensions on a vector, converting it to an array in place
dim(x) also GETS the dimensions of an existing array or matrix
2×3×4 = 24 — dimensions must multiply to match the vector length exactly
Viva Q&A

  • Q1: What does dim() do in R?

    • Two uses: GET dimensions with dim(x)
    • SET dimensions with dim(x) <- c(r,c,l)
    • Getting: dim(matrix(1:9,3,3)) returns 3 3
    • Setting: dim(x) <- c(2,3,4) reshapes x into 3D array

  • Q2: Why must dimensions multiply to match vector length?

    • Array just reshapes existing data — no new elements created
    • 2×3×4 = 24, so data must have exactly 24 elements
    • If lengths don't match, R gives an error
    • Think of it as folding a 1D vector into an n-D shape

  • Q3: What is the difference between using dim() vs array()?

    • Both achieve the same result
    • dim(x) <- c(r,c,l): modifies existing vector IN PLACE
    • array(data, dim=c(r,c,l)): creates a NEW array object
    • Both follow column-major fill order

  • Q4: How do you get the number of dimensions in R?

    • length(dim(x)) gives the number of dimensions
    • A vector: dim() returns NULL (no dimension)
    • Matrix: length(dim(m)) = 2
    • 3D array: length(dim(a)) = 3

  • Q5: What happens when you assign dim() to a regular vector?

    • Vector becomes an array/matrix
    • Elements rearranged into given shape (column-major order)
    • The actual data doesn't change, just the shape
    • Example: a 24-element vector becomes a 2×3×4 array
Exp 09

Statistical Summary of Data Frame

data.framesummarystr
Aim

Get statistical summary and nature of data in a given data frame.

R Code
# Create sample data frame
df <- data.frame(
  Name  = c("Alice", "Bob", "Carol", "Dave"),
  Age   = c(20, 21, 19, 22),
  Marks = c(85, 72, 90, 68)
)

print(summary(df))  # min, max, mean, median, quartiles
str(df)             # structure: column types and sample values
Key Concepts
data.frame() creates tabular data where each column can have a different type
summary() gives min, 1st quartile, median, mean, 3rd quartile, max for numeric columns
str() shows structure: class, number of rows/cols, column names with data types and sample values
Viva Q&A

  • Q1: What is a data frame in R?

    • 2D table where each column can have a different data type
    • Like a spreadsheet — rows are observations, columns are variables
    • Most common data structure for data analysis in R
    • Created with data.frame() function

  • Q2: What does summary() show?

    • For numeric columns: min, 1st quartile, median, mean, 3rd quartile, max
    • For character/factor columns: count of each unique value
    • Shows all columns at once
    • Very useful for quick data exploration

  • Q3: What does str() show?

    • Structure of the object
    • Class type (data.frame, matrix, etc.)
    • Number of observations (rows) and variables (columns)
    • Column names, data types, and first few sample values

  • Q4: What is the difference between a data frame and a matrix?

    • Matrix: ALL elements same type (all numeric, or all character)
    • Data frame: each column can have DIFFERENT type (mix of numeric, character, factor)
    • Data frames are more flexible for real-world data
    • Matrices are faster for mathematical operations

  • Q5: What is stringsAsFactors?

    • Old R (before 4.0): character columns auto-converted to factors
    • stringsAsFactors=FALSE: keeps character columns as strings
    • R 4.0+: default changed to FALSE (no auto-conversion)
    • Factors are useful for categorical data with fixed levels
Exp 10

Extract First Two Rows from Data Frame

headtailindexing
Aim

Extract the first two rows from a given data frame.

R Code
df <- data.frame(
  Name  = c("Alice", "Bob", "Carol", "Dave", "Eve"),
  Age   = c(20, 21, 19, 22, 20),
  Score = c(85, 72, 90, 68, 78)
)

# head(df, n) returns first n rows
print(head(df, 2))
Key Concepts
head(df, n) returns first n rows; tail(df, n) returns last n rows (default 6)
df[rows, cols] is the indexing syntax — leaving blank selects all of that dimension
1:2 means rows 1 and 2; df[1:2, ] selects rows 1–2 and all columns
Viva Q&A

  • Q1: What is head() in R?

    • Returns the first n rows of a data frame or elements of a vector
    • Default: 6 rows/elements
    • head(df, 2) returns first 2 rows
    • Useful to quickly preview large datasets

  • Q2: What is tail() in R?

    • Returns the LAST n rows (default 6)
    • Opposite of head()
    • tail(df, 3) returns last 3 rows
    • Also works on vectors: tail(c(1:10), 3) gives 8 9 10

  • Q3: How does data frame indexing work?

    • df[row, col] — specific element
    • df[1, ] — row 1, all columns
    • df[ ,2] — all rows, column 2
    • Blank index means "select all" for that dimension

  • Q4: How do you select a column from a data frame?

    • Three ways: df$ColumnName (by name, most common)
    • df[["ColumnName"]] (double bracket, returns vector)
    • df[ ,column_number] (by position)
    • df["ColumnName"] (single bracket, returns data frame)

  • Q5: How do you filter rows based on a condition?

    • df[df$Age > 20, ] — rows where Age > 20
    • subset(df, Age > 20) — same result, cleaner syntax
    • df[df$Name == "Alice", ] — filter by character value
    • Can combine: df[df$Age > 20 & df$Score > 80, ]
Exp 11

Create Matrix from Vector

matrixnrowbyrowdimnames
Aim

Create a matrix taking a given vector of numbers as input and display the matrix.

R Code
vec <- c(1, 2, 3, 4, 5, 6, 7, 8, 9)

# Create 3x3 matrix from vector
mat <- matrix(vec, nrow = 3, ncol = 3)
cat("Matrix:\n")
print(mat)
Key Concepts
matrix(data, nrow, ncol) creates a 2D matrix from a vector
byrow=TRUE fills the matrix row by row; default is column by column
dimnames=list(rowNames, colNames) adds labels to rows and columns
Viva Q&A

  • Q1: What parameters does matrix() take?

    • matrix(data, nrow, ncol, byrow, dimnames)
    • data: input vector
    • nrow/ncol: number of rows/columns
    • byrow: fill by row if TRUE (default FALSE = fill by column)

  • Q2: If you provide nrow but not ncol, what happens?

    • R automatically calculates ncol based on data length
    • Example: matrix(1:12, nrow=3) — 12 elements, 3 rows → ncol=4
    • Number of elements must be divisible
    • R gives warning if elements don't fill evenly (recycles)

  • Q3: What is byrow=TRUE?

    • By default (byrow=FALSE): matrix fills column by column
    • byrow=TRUE: matrix fills row by row (left to right)
    • matrix(1:9, nrow=3, byrow=TRUE): row 1 gets 1,2,3; row 2 gets 4,5,6
    • Choose based on how your data is logically organized

  • Q4: What is dimnames?

    • Assigns labels to rows and columns
    • Format: dimnames = list(row_names_vector, col_names_vector)
    • Example: dimnames=list(c("R1","R2","R3"), c("C1","C2","C3"))
    • Accessed with rownames(mat) and colnames(mat)

  • Q5: How do you transpose a matrix?

    • Using t() function
    • Swaps rows and columns
    • A 3×4 matrix becomes 4×3 after t(mat)
    • Example: t(matrix(1:6, nrow=2, ncol=3)) gives 3×2 matrix
Exp 12

Matrix Indexing

indexingmatrixsubmatrix
Aim

Access element at 3rd column and 2nd row, only the 3rd row, only the 4th column.

R Code
mat <- matrix(1:16, nrow = 4, ncol = 4)
cat("Matrix:\n"); print(mat)

# Element at row 2, column 3
cat("Element [row2, col3]:", mat[2, 3], "\n")

# Only 3rd row (all columns)
cat("3rd row:", mat[3, ], "\n")

# Only 4th column (all rows)
cat("4th column:", mat[, 4], "\n")
Key Concepts
mat[row, col] for a single element; blank means "select all" for that dimension
mat[3, ] selects entire row 3; mat[ ,4] selects entire column 4
Use ranges like mat[1:2, 2:3] to extract a sub-matrix; negative index like mat[-2, ] excludes that row
Viva Q&A

  • Q1: How do you access a single element from a matrix?

    • Use mat[row, col]
    • Example: mat[2, 3] gets element at 2nd row, 3rd column
    • Both row and column indices must be specified
    • Negative index EXCLUDES: mat[-2, -3] removes row 2 and col 3

  • Q2: How do you select an entire row?

    • mat[row_number, ] — leave column index BLANK
    • Empty index means "all" for that dimension
    • Example: mat[3, ] selects entire 3rd row as a vector
    • Multiple rows: mat[c(1,3), ] selects rows 1 and 3

  • Q3: How do you select an entire column?

    • mat[ ,col_number] — leave row index BLANK
    • Example: mat[ ,4] selects entire 4th column as a vector
    • Multiple columns: mat[ ,c(2,4)] selects columns 2 and 4
    • Result is a vector (loses matrix structure unless drop=FALSE)

  • Q4: How do you select a sub-matrix?

    • Use ranges for both row and column
    • Example: mat[1:2, 2:3] gives a 2×2 sub-matrix
    • Rows 1-2 and columns 2-3
    • Can also use named rows/cols if dimnames are set

  • Q5: What does mat[-2, ] do?

    • Negative index EXCLUDES that row/column
    • mat[-2, ]: all rows EXCEPT row 2
    • mat[ ,-3]: all columns EXCEPT column 3
    • mat[-c(1,3), ]: excludes rows 1 and 3

Unit I Introduction to Data Science
Unit I — Syllabus Quick Overview
  • Data Science: Interdisciplinary field — stats + CS + domain knowledge to extract insights from data
  • Data types: Structured (tables/CSV), Unstructured (text/images/video), Semi-structured (JSON/XML)
  • Variables: Nominal (labels), Ordinal (ranked), Interval (no true zero), Ratio (true zero)
  • Data Analytics: Focuses on answering specific business questions using existing data
  • Data Processing Chain: Collect → Clean → Transform → Analyze → Visualize → Interpret
  • Data Distributions: Normal (bell curve), Skewed (left/right), Uniform (flat), Binomial (discrete)
  • Paths to Data Science: Data Analyst → Data Scientist → ML Engineer → Data Engineer → Architect
  • Data Mining: Discovering patterns in large datasets using statistics & ML (part of KDD)
  • Data Warehouse: Stores historical data for analytics (OLAP); Database stores current data (OLTP)
  • Cloud in DS: AWS/GCP/Azure provide scalable compute, storage, ML services for data workflows
  • AI vs ML vs DS: AI is the broad goal; ML is a technique to achieve it; DS uses both for insights
  • Supervised Learning: Labelled data → Classification / Regression (e.g., spam filter, house prices)
  • Unsupervised Learning: No labels → Clustering / Association (e.g., customer segmentation)
  • Real-world apps: Healthcare (diagnosis), Finance (fraud), E-commerce (recommendations), Social media

Q1: What is Data Science?

  • Interdisciplinary field combining statistics, computer science, and domain expertise
  • Extracts knowledge and insights from structured and unstructured data
  • Involves: data collection, cleaning, analysis, visualization, interpretation
  • Used in healthcare, finance, e-commerce, social media, etc.

Q2: What is Data Mining?

  • Process of discovering patterns, correlations, anomalies from large datasets
  • Uses statistical and ML techniques
  • Examples: finding shopping patterns, detecting fraud, predicting customer churn
  • Part of the KDD (Knowledge Discovery in Databases) process

Q3: What is the difference between a Database and a Data Warehouse?

  • Database: stores CURRENT operational data (OLTP — transactions)
  • Data Warehouse: stores HISTORICAL aggregated data (OLAP — analysis)
  • Database: frequently updated; Warehouse: updated in batches
  • Database: optimized for speed; Warehouse: optimized for analysis

Q4: What is Machine Learning? What are its types?

  • Supervised: trains on labeled data (input + correct output)
  • Unsupervised: finds patterns in unlabeled data (no correct answers)
  • Reinforcement: learns through trial and error with rewards
  • Example: Email spam = supervised; Customer grouping = unsupervised

Q5: What is the difference between Supervised and Unsupervised Learning?

  • Supervised: model learns from labeled examples
  • Examples: Classification (spam/not spam), Regression (price prediction)
  • Unsupervised: model finds hidden patterns without labels
  • Examples: Clustering (customer segments), Dimensionality reduction

Q6: What is the Data Processing Chain?

  • Collection → Cleaning (handle missing/wrong data) → Transformation (normalize, encode)
  • Analysis (statistics, ML models) → Visualization → Insights/Decisions
  • Each step is critical: "garbage in, garbage out"
  • Data cleaning typically takes 60-80% of the time

Q7: What is Cloud Computing in Data Science?

  • Remote servers store and process large datasets
  • No need for expensive on-premise hardware
  • Examples: AWS S3 (storage), Google BigQuery (analysis), Azure ML
  • Enables scalability — pay for what you use

Q8: What is Artificial Intelligence? How does it relate to Data Science?

  • AI: broad field of making machines perform human-like tasks
  • Data Science: provides data and insights to power AI systems
  • Machine Learning is the bridge — it's a subset of AI and a tool of Data Science
  • Data Science fuels AI; AI applies the insights

Q9: What is the difference between Data Science, Data Analytics, and Data Engineering?

  • Data Science: builds predictive models and discovers insights from data (ML, statistics)
  • Data Analytics: analyses existing data to answer specific business questions (SQL, Excel, dashboards)
  • Data Engineering: builds and maintains data pipelines and infrastructure (ETL, databases)
  • Data Engineers prepare data → Data Scientists model it → Data Analysts interpret it

Q10: What is EDA (Exploratory Data Analysis)?

  • Initial analysis of data to summarize its main characteristics — often visually
  • Goals: detect outliers, find patterns, check assumptions, understand distributions
  • Techniques: histograms, box plots, scatter plots, correlation matrices, summary statistics
  • In R: summary(), str(), hist(), pairs() are common EDA tools

Q11: What is the CRISP-DM model?

  • Cross-Industry Standard Process for Data Mining — 6-phase lifecycle
  • Business Understanding → Data Understanding → Data Preparation → Modelling → Evaluation → Deployment
  • Iterative process — can go back to earlier phases
  • Most widely used data science project framework in industry

Q12: What is Overfitting vs Underfitting?

  • Overfitting: model learns training data too well including noise — poor performance on new data
  • Underfitting: model too simple — fails to capture the pattern even in training data
  • Goal: find the sweet spot (Bias-Variance tradeoff)
  • Solutions: regularization, cross-validation, more data (overfitting); complex model, more features (underfitting)

Q13: What are Data Distributions?

  • Normal Distribution: symmetric bell curve around mean μ; most natural phenomena follow this
  • Skewed Distribution: Right-skewed (tail to right, mean > median); Left-skewed (tail to left)
  • Uniform Distribution: all outcomes equally likely; flat histogram (e.g., rolling a fair die)
  • Binomial Distribution: counts successes in n independent trials; discrete; shape depends on p
  • Purpose: understanding distribution shape guides choice of statistical tests and ML models

Q14: What are the career paths / roles in Data Science?

  • Data Analyst: cleans and analyzes data; creates dashboards and reports; uses SQL, Excel, Power BI
  • Data Scientist: builds predictive models; uses Python/R, ML algorithms, statistics
  • ML Engineer: deploys and scales ML models in production; focuses on engineering, not research
  • Data Engineer: builds data pipelines, ETL processes, data warehouses; uses Spark, Kafka, SQL
  • Data Architect: designs the overall data infrastructure and strategy for an organization

Q15: What are the real-world applications of Data Science?

  • Healthcare: disease diagnosis, drug discovery, patient outcome prediction
  • Finance: fraud detection, credit scoring, algorithmic trading, risk assessment
  • E-commerce: product recommendation systems, demand forecasting, dynamic pricing
  • Social Media: content personalization, sentiment analysis, targeted advertising
  • Transport: route optimization (Google Maps), self-driving cars, traffic prediction

Q16: What is a Variable in Data Science? What are its types?

  • A variable is any characteristic, number, or quantity that can be measured or counted
  • Nominal: categories with no order (e.g., Gender: Male/Female, Colors)
  • Ordinal: categories with a meaningful order but unequal intervals (e.g., Rank: 1st, 2nd, 3rd)
  • Interval: ordered, equal intervals, no true zero (e.g., Temperature in °C, IQ scores)
  • Ratio: ordered, equal intervals, true zero exists (e.g., Height, Weight, Age)

Q1: What is R?

  • Free, open-source programming language for statistical computing
  • Developed by Ross Ihaka and Robert Gentleman (New Zealand)
  • Very popular in data science, statistics, and bioinformatics
  • CRAN repository has 18,000+ packages for R

Q2: What are the main data types in R?

  • Numeric: 3.14 (decimal numbers)
  • Integer: 3L (whole numbers, L suffix)
  • Character: "hello" (text strings)
  • Logical: TRUE/FALSE | Complex: 2+3i | Check type with class() or typeof()

Q3: What is a Vector in R?

  • Fundamental data structure in R
  • Ordered collection of elements of the SAME type (atomic)
  • Created with c() function
  • Example: c(1,2,3) is numeric vector; all elements auto-coerced to same type

Q4: What is a List in R?

  • Ordered collection of elements of DIFFERENT types
  • Can contain vectors, data frames, numbers, strings, even other lists
  • Created with list() function
  • More flexible than vector — like a container of containers

Q5: What is a Factor in R?

  • Categorical variable with a fixed set of possible values (called levels)
  • Used for categorical data like gender, city, grade
  • Created with factor()
  • Internally stored as integers with labels — memory efficient

Q6: What are special values in R?

  • NA: missing/unknown value
  • NULL: empty/absent object (no value at all)
  • NaN: Not a Number (e.g., 0/0)
  • Inf/-Inf: positive/negative infinity | Check with is.na(), is.null(), is.nan(), is.infinite()

Q7: What is a Data Frame?

  • 2D table where each column can have a different type
  • Like a spreadsheet or SQL table
  • Each column is a vector of the same length
  • Most common structure for data analysis — real-world datasets use data frames

Q8: What is an IDE for R?

  • RStudio: most popular IDE, provides console + script editor + environment viewer + plot panel
  • VS Code with R extension: also widely used
  • Jupyter Notebook: supports R kernel for interactive analysis
  • RStudio Cloud: web-based, no installation needed

Q9: What is a Matrix in R? How is it created?

  • 2D array where all elements are of the SAME type
  • Created with matrix(data, nrow, ncol, byrow)
  • Example: matrix(1:6, nrow=2, ncol=3) creates 2×3 matrix
  • Access: m[row, col] — row first, column second

Q10: What is the difference between a Matrix and a Data Frame?

  • Matrix: all elements must be the same type; mathematical operations supported
  • Data Frame: each column can be different type; ideal for real-world datasets
  • Matrix operations: matrix multiplication, transpose; Data Frame: merge, filter, aggregate
  • Use matrix for numerical computation; use data frame for data analysis

Q11: How does R handle missing values (NA)?

  • NA propagates: NA + 5 = NA, mean(c(1,NA,3)) = NA
  • To ignore NAs: mean(x, na.rm=TRUE)
  • Check for NA: is.na(x) returns logical vector
  • Remove rows with NA: na.omit(df) or complete.cases(df)

Q12: What is the difference between == and identical() in R?

  • ==: element-wise comparison, returns logical vector, treats NA specially
  • identical(): checks if two objects are exactly the same (type + value + attributes)
  • NA == NA returns NA; identical(NA, NA) returns TRUE
  • Use identical() for strict equality checks in functions/tests

Q13: What is the History of R? Who created it?

  • R was created by Ross Ihaka and Robert Gentleman at the University of Auckland, New Zealand
  • Named "R" after their first initials; developed as a free implementation of the S language (Bell Labs, 1970s)
  • First public version released in 1993; R 1.0.0 released on February 29, 2000
  • CRAN (Comprehensive R Archive Network) launched to host packages; now has 18,000+ packages
  • Governed by the R Foundation for Statistical Computing (Vienna, Austria)

Q14: How do you install R and packages? What is CRAN?

  • Install R: Download from cran.r-project.org; run the installer; then install RStudio IDE
  • CRAN: Comprehensive R Archive Network — official repository for R packages; thousands of maintained packages
  • Install a package: install.packages("ggplot2") — downloads and installs from CRAN
  • Load a package: library(ggplot2) — makes the package available in the session
  • Check installed: installed.packages(); Update: update.packages()

Q15: What are Arrays in R? How do they differ from matrices?

  • Array: Multi-dimensional data structure; all elements must be of the same type
  • Created with: array(data, dim=c(rows, cols, layers))
  • Matrix: Special case of an array with exactly 2 dimensions (rows × columns)
  • Array: Can have 3 or more dimensions (e.g., dim=c(2,3,4) = 2 rows, 3 cols, 4 layers)
  • Access element: arr[row, col, layer]

Q16: How do you get Help in R?

  • ?mean or help("mean"): opens documentation for the exact function
  • ??keyword or help.search("keyword"): searches all docs for the keyword
  • args(lm): shows the arguments of a function without full docs
  • example(plot): runs the built-in examples for a function
  • Online: rdocumentation.org, Stack Overflow, CRAN package vignettes
Unit III Programming using R
Unit III — Syllabus Quick Overview
  • Function creation: f <- function(x, y=1) { x + y }; default args supported
  • Scripts: .R files run top-to-bottom; source("file.R") executes a script from console
  • Logical operators: & (AND, vectorized), | (OR, vectorized), ! (NOT), &&/|| (scalar, short-circuit)
  • Conditional: if(cond){} else if(){} else{}; also ifelse(cond, yes, no) for vectors
  • Loops: for, while, repeat; controlled with next (skip) and break (exit)
  • switch(): switch(x, "a"="Alpha", "b"="Beta") — alternative to if-else chains
  • Creating a list: lst <- list(name="Rex", age=20); access with lst$name or lst[["name"]]
  • Common list ops: length(), names(), append(), modifyList()
  • Recursive list: List containing other lists; create with nested list() calls
  • Data Frame ops: subset(), merge(), rbind(), cbind(), na.omit()
  • lapply/sapply: Apply a function to each list/vector element; sapply simplifies result to vector
  • S3 Classes: Informal OOP; add class attribute; define generics with UseMethod()
  • S4 Classes: Formal OOP; setClass(), setGeneric(), setMethod(); strict type checking
  • Reference Classes (R5): Mutable OOP similar to Java/Python; fields & methods defined together; $new()

Q1: What are the types of loops in R?

  • for: iterates over each element of a sequence
  • while: runs while a condition is TRUE
  • repeat: infinite loop until break statement
  • Apply family (lapply, sapply, tapply): functional alternatives to loops

Q2: What is the difference between lapply() and sapply()?

  • lapply(): always returns a LIST
  • sapply(): tries to SIMPLIFY result to vector/matrix
  • Example: sapply(1:5, function(x) x^2) returns numeric vector
  • Use lapply when you want consistent list output

Q3: What are conditional statements in R?

  • if (condition) { } else { }: basic conditional
  • ifelse(condition, true_val, false_val): vectorized (works on entire vector at once)
  • switch(expr, case1=result1, case2=result2): multi-way branching
  • Use ifelse() instead of if-else inside data frame operations

Q4: What are S3 and S4 Classes in R?

  • S3: informal OOP system; class assigned with class() attribute
  • S4: formal, stricter OOP; defined with setClass(), accessed with slot()
  • S3 is simpler and more common in base R
  • S4 is used in Bioconductor and formal packages

Q5: How do you create a function in R?

  • Syntax: myFunc <- function(param1, param2) { body; return(result) }
  • Return is optional — R returns the last expression automatically
  • Functions are objects and can be passed to other functions
  • Example: square <- function(x) x^2 then square(5) gives 25

Q6: What is the switch() statement?

  • Used for multi-way selection based on a value
  • Cleaner alternative to multiple if-else
  • Example: switch(day, "Mon"="Monday", "Tue"="Tuesday", "Unknown")
  • Default (unnamed last argument) returned if no match

Q7: How do you create a list in R?

  • myList <- list(name="Alice", age=20, scores=c(85,90,78))
  • Elements can be different types
  • Access: myList$name, myList[["name"]], myList[[1]]
  • names(myList) shows element names

Q8: What is the difference between [ ] and [[ ]] for list access?

  • Single [ ]: returns a sub-list (still a list object)
  • Double [[ ]]: extracts the actual element (its native type)
  • Example: myList["age"] returns a list; myList[["age"]] returns the number
  • Always use [[]] when you want the actual value

Q9: What is tryCatch() in R? How is error handling done?

  • tryCatch(expr, error=function(e){...}, warning=function(w){...})
  • Allows code to continue even when an error occurs
  • Example: tryCatch(log(-1), warning=function(w) "got a warning")
  • Similar to try-catch in Java/Python — prevents program crash

Q10: What is vapply() vs sapply()?

  • sapply(): tries to simplify output but return type can be unpredictable
  • vapply(): safer — you specify the expected return type explicitly
  • Example: vapply(list, fun, FUN.VALUE=numeric(1)) always returns numeric vector
  • Use vapply() in production code for safer, predictable output

Q11: What is a closure in R?

  • A function that "closes over" (remembers) variables from its enclosing environment
  • The inner function has access to the outer function's variables even after it returns
  • Example: counter <- function() { n<-0; function() { n<<-n+1; n } }
  • Used to create function factories, stateful functions, and generators

Q12: What are next and break in R loops?

  • break: exits the loop immediately
  • next: skips the current iteration and goes to the next
  • Similar to break/continue in other languages
  • Example: for(i in 1:10){ if(i==5) break; print(i) } prints 1 to 4

Q13: What are Logical Operators in R?

  • & / |: element-wise AND / OR — compare entire vectors, return logical vector
  • && / ||: scalar AND / OR — short-circuit, only compares first element; used in if()
  • !: NOT — negates a logical value; !TRUE → FALSE
  • xor(x, y): TRUE only when exactly one of x, y is TRUE (exclusive OR)
  • any() / all(): returns TRUE if any/all elements of a logical vector are TRUE

Q14: What is a Recursive List and common List operations in R?

  • Recursive list: A list that contains other lists as elements — any depth of nesting is allowed
  • Example: student <- list(name="Rex", scores=list(math=90, sci=85))
  • Access nested: student$scores$math or student[["scores"]][["math"]]
  • Common ops: length(lst), names(lst), append(lst, list(new=val)), lst$key <- NULL (delete)
  • unlist(lst): flattens a list into a named vector

Q15: What are Reference Classes (R5) in R?

  • The most OOP-like system in R — similar to Java or Python classes
  • Created with setRefClass(); instances created with ClassName$new()
  • Key difference from S3/S4: Reference class objects are mutable — methods modify the object in place using <<-
  • Fields (attributes) and methods are defined together inside setRefClass()
  • Example: Counter <- setRefClass("Counter", fields=list(n="numeric"), methods=list(inc=function(){n<<-n+1}))

Q16: What is an R Script? How does it differ from console?

  • R Script: A plain text file with .R extension containing R code; runs top to bottom
  • Run from RStudio: click "Run" or Ctrl+Enter (line by line) or Ctrl+Shift+Enter (whole file)
  • Run from command line: Rscript myfile.R
  • Console: Interactive — code runs immediately one line at a time; history not automatically saved
  • Scripts are preferred for reproducibility — same result every time; share with others; version-control with git
Unit IV Data Handling, Mathematical & Statistical Concepts
Unit IV — Syllabus Quick Overview
  • Save/Load R data: save(obj, file="data.RData") / load("data.RData"); or saveRDS() / readRDS()
  • CSV import: read.csv("file.csv"); CSV export: write.csv(df, "out.csv", row.names=FALSE)
  • ODBC: Open Database Connectivity; connect R to external databases using RODBC or odbc package
  • Debugging: debug(fn), browser() (pause inside fn), traceback() (show call stack after error)

  • ggplot2: Grammar of Graphics; ggplot(df, aes(x,y)) + geom_point(); layered approach

  • Frequency distribution: table(x) — counts; prop.table(table(x)) — proportions; hist() — visual
  • Central tendency: mean(x), median(x); mode = most frequent (no built-in; use table())
  • Hypothesis testing: One-sample: t.test(x, mu=50); Two-sample: t.test(x, y); p-value < 0.05 → reject H₀

  • Regression: lm(y ~ x, data=df); summary(model) for R², coefficients, p-values
  • Normal distribution: dnorm(), pnorm(), qnorm(), rnorm()
  • Other distributions: Binomial (dbinom), Poisson (dpois), t (dt), chi-sq (dchisq), F (df)

Q1: How do you read a CSV file in R?

  • read.csv("filename.csv"): reads CSV with header row
  • read.table("file.txt", header=TRUE, sep="\t"): general delimited files
  • readr::read_csv(): faster, part of tidyverse
  • Result is a data frame object

Q2: How do you save/load R data?

  • Save: save(obj, file="data.RData") or saveRDS(obj, "data.rds")
  • Load: load("data.RData") restores object with original name
  • readRDS("data.rds"): loads and can assign new name
  • RDS format preferred for single objects

Q3: What is ggplot2?

  • Powerful R package for data visualization
  • Based on "Grammar of Graphics" — builds plots layer by layer
  • Part of the tidyverse collection
  • Basic structure: ggplot(data, aes(x,y)) + geom_point() + other layers

Q4: What types of plots can R create?

  • Base R: plot() (scatter), hist() (histogram), boxplot() (box), barplot() (bar)
  • ggplot2: geom_point(), geom_line(), geom_bar(), geom_histogram(), geom_boxplot()
  • ggplot2 produces publication-quality graphics
  • Choose chart type based on data type (categorical vs continuous)

Q5: What is Hypothesis Testing?

  • Statistical method to test an assumption about a population
  • Two hypotheses: H₀ (null — status quo) and H₁ (alternative — what we want to prove)
  • If p-value < significance level (0.05): reject H₀
  • Examples: t-test, chi-square test, ANOVA

Q6: How do you find correlation in R?

  • cor(x, y): Pearson correlation coefficient
  • cor(x, y, method="spearman"): Spearman rank correlation
  • Range: -1 (perfect negative) to +1 (perfect positive)
  • cor.test(x, y): also gives p-value for significance

Q7: What is ODBC in R?

  • Open Database Connectivity — standard interface to connect to databases
  • RODBC package: older R package for ODBC connections
  • Modern approach: odbc + DBI packages
  • Used to import data from MySQL, SQL Server, Oracle into R

Q8: What is the difference between Base R graphics and ggplot2?

  • Base R: simpler syntax, built-in, fast for quick plots
  • ggplot2: layered approach, highly customizable, aesthetically superior
  • Base R: hist(data) — one line
  • ggplot2: ggplot(df, aes(x=col)) + geom_histogram() — more verbose but more control

Q9: What is dplyr and its 5 key verbs?

  • dplyr: core tidyverse package for data manipulation
  • filter(): select rows by condition
  • select(): choose columns
  • mutate(): create/modify columns | arrange(): sort rows | summarise(): aggregate data (often with group_by())

Q10: What is the pipe operator %>% in R?

  • Passes the left-hand side as the first argument of the right-hand side function
  • x %>% f() is equivalent to f(x)
  • Makes code readable left-to-right: df %>% filter(age>18) %>% select(name)
  • From magrittr package; R 4.1+ has native pipe |>

Q11: How do you perform a t-test in R?

  • One-sample: t.test(x, mu=value) — tests if mean equals value
  • Two-sample: t.test(x, y) — tests if two group means are equal
  • Paired: t.test(before, after, paired=TRUE)
  • Output: t-statistic, degrees of freedom, p-value, confidence interval

Q12: What is lm() in R?

  • Fits a linear regression model: lm(y ~ x, data=df)
  • summary(model): shows coefficients, R², p-values, residuals
  • predict(model, newdata): predicts values for new data
  • Formula syntax: y ~ x1 + x2 for multiple regression

Q13: What are Debugging Techniques in R?

  • traceback(): shows the call stack after an error — tells you where it crashed
  • debug(fn): marks a function for debugging; next call enters line-by-line debug mode; undebug(fn) to remove
  • browser(): inserts a breakpoint inside a function; execution pauses there; type n (next), c (continue), Q (quit)
  • message("val:", x): prints diagnostic info to console without stopping execution
  • tryCatch(): catches errors and warnings programmatically without crashing the script

Q14: What are Measures of Central Tendency in R?

  • Mean: mean(x) — arithmetic average; sensitive to outliers
  • Median: median(x) — middle value when sorted; robust to outliers
  • Mode: Most frequent value; R has no built-in mode() for this (it returns data type); use names(sort(table(x), decreasing=TRUE)[1])
  • When to use: Symmetric data → mean; Skewed data → median; Categorical data → mode
  • Also useful: summary(x) gives min, Q1, median, mean, Q3, max in one call

Q15: What is Frequency Distribution in R?

  • Frequency table: table(x) — counts occurrences of each unique value
  • Relative frequency: prop.table(table(x)) — proportions summing to 1
  • Histogram: hist(x) — visual frequency distribution for continuous data; breaks arg controls bins
  • Cumulative: cumsum(table(x)) — cumulative frequencies
  • For two variables: table(x, y) creates a cross-tabulation (contingency table)

Q16: What are the main Statistical Distributions in R?

  • R provides 4 functions per distribution: d(density), p(CDF), q(quantile), r(random)
  • Normal: dnorm(x, mean, sd), pnorm(), qnorm(), rnorm(n)
  • Binomial: dbinom(x, size, prob) — discrete; n trials, probability p
  • Poisson: dpois(x, lambda) — counts of rare events; mean = variance = λ
  • t-distribution: dt(x, df) — used for small sample hypothesis testing
  • Chi-square: dchisq(x, df) — used for goodness-of-fit and independence tests
