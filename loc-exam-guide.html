DMA (Direct Memory Access) transfers data directly between high-speed peripherals and main memory without passing through the CPU, saving clock cycles.

Why DMA gets Priority The CPU is patient because its instructions can wait. High-speed I/O peripherals are volatile and impatient — if incoming data blocks are not read instantly, they will overwrite and be lost. Thus, DMA holds higher bus priority.

Pipeline Hazards & Flynn's Classification ⭐ Repeated 3×

Flynn's Parallel Taxonomy Classifies parallel computers by Instruction streams vs Data streams: (1) SISD: Uni-core PC, (2) SIMD: Vector/GPU graphics processor, (3) MISD: Redundant spacecraft systems (rare), (4) MIMD: Modern multicore processors.

The Three Pipeline Hazards

  • Structural Hazard: Hardware conflicts. E.g., when instruction-fetch and data-memory access try to use a single-port memory block at the same time. Resolved by splitting into Instruction and Data caches (Harvard architecture).
  • Data Hazard: Operands not ready. An instruction depends on a previous instruction's result that has not yet completed the write-back stage (RAW, WAR, WAW). Resolved by forwarding or stalling (bubbles).
  • Control Hazard: Branch delays. Decisions on jumps affect which instructions to fetch next. Resolved by branch prediction or branch target buffers.

A physical address is partitioned into distinct fields to query cache rows:

Set-Associative Address Fields

Direct Mapping vs Associative • Direct Mapping has a dedicated LINE field; a block can map to exactly one cache line, causing thrashing.
• Set-Associative allows a block to reside in any line within the target SET, balancing search logic complexity with higher hit rates.

The Algorithms • FIFO (First-In-First-Out): Evicts oldest page. Suffers from Belady's Anomaly (faults can rise with frame count).

Define a Program Status Word (PSW). What are its primary flags?

The PSW is a small CPU register. It holds status bits (flags). The ALU updates these bits after every operation. Main x86 flags:

  • CF: Carry Flag. Set when unsigned add overflows the top bit.
  • ZF: Zero Flag. Set when the result is zero.
  • SF: Sign Flag. Copies the top bit (sign) of the result.
  • OF: Overflow Flag. Set on signed (2's-complement) overflow.
  • PF: Parity Flag. Set when the low byte has even parity.

What are addressing modes? Describe the 9 standard addressing modes with their EA formulas. Provide a worked numerical trace for direct, relative, and register-indirect effective addresses on a memory system.
Introduction

Addressing modes are rules built into the ISA. They tell the CPU how to find the effective address (EA) of an operand from the bits in the instruction. The instruction has only a small address field (often 8–16 bits). The real memory address can be 32 or 64 bits wide. Addressing modes bridge this gap. They also let the CPU trade code size, speed, flexibility, and decoder cost. Example: a smartphone CPU runs the same instruction set on millions of programs only because addressing modes hide the address gap.

  • Use 1 — smaller code: a short address field can reach a big memory via indirection or indexing.
  • Use 2 — flexible compilers: indexed and indirect modes let one piece of code walk arrays, structs, linked lists without rewriting.
  • Use 3 — relocation: relative and base-register modes let a program run at any memory address. No re-linking needed.
  • Use 4 — recursion: stack-based addressing handles nested calls with local frames.
  • Use 5 — ISA style: CISC (x86) keeps the full set; RISC (ARM, RISC-V) keeps only a few for pipeline simplicity.

Number of Addresses in an Instruction
  • 3-address (RISC, MIPS): ADD R1, R2, R3 → R1 = R2 + R3. Long instruction word but no implicit register.
  • 2-address (x86): ADD AX, BX → AX = AX + BX. One source is destroyed. Common in CISC.
  • 1-address (8085-style): ADD B → AC = AC + B. Implicit accumulator.
  • 0-address (Stack / JVM): ADD → pops top two, pushes sum. Uses RPN.
Modern Relevance
  • x86-64 (CISC): keeps the full set: direct, indirect, scaled-indexed ([base + index*scale + disp]), RIP-relative for PIE binaries, segment (FS/GS) for thread-local data.
  • ARM AArch64 (RISC): only load-store + immediate + register + register-shifted + PC-relative. No indirect, no direct-to-memory ALU ops. Every memory touch is an explicit LDR/STR.
  • RISC-V: even leaner. Only immediate + register + register+offset + PC-relative. No indexed, no auto-increment. This simplicity is why a RISC-V core can be taped out in months, not years.
  • Apple Silicon M-series hits 5+ GHz because ARM64's small mode set lets the front-end decode 8 instructions per cycle without blowing up the decoder.
  • JVM / .NET CLR / WebAssembly all use a zero-address (stack) ISA. RPN maps neatly onto a stack. Bytecode runs anywhere.
Common pitfalls (1) Immediate vs Direct: in Immediate the value is the operand (no memory read). In Direct the operand sits at the address inside the instruction (one memory read). (2) Register-Indirect ([BX]) reads memory at the register's value. This is different from Register (BX), which uses the register itself. (3) Examiners cut marks for "Indirect = EA + R". Indirect is M[A]. Register-Indirect is R. (4) Relative always uses the next PC (after fetch), not the current PC. (5) Indexed and Base look the same in math but differ by use: indexed walks data, base relocates code.
Conclusion

Addressing modes are the ISA's contract with the compiler. They trade decoder cost for code density and flexibility. Knowing the EA formula for each mode is a must for any compiler or low-level systems engineer.

(a) IEEE-754 Single Precision Format — Layout

IEEE-754 single-precision (32-bit) splits a real number into three parts: sign, exponent, mantissa. The value is (-1)^S × 1.M × 2^(E - 127) for normal numbers. The leading 1 of the mantissa is hidden (not stored). The bias is 127.

(b) Booth's Algorithm — Why & How

Booth's algorithm does signed (2's-complement) multiplication. It looks at the last two bits Q₀Q₋₁ of the multiplier, picks one of four actions, then shifts right. A long run of 1s becomes one subtract + one add instead of many adds. So multiplying negative numbers gets faster. No sign-extension needed.

Q₀Q₋₁ pairAction on AInterpretation
00No operationMiddle of a run of 0s
01A ← A + MEnd of a run of 1s — add the multiplicand
10A ← A − MStart of a run of 1s — subtract the multiplicand
11No operationMiddle of a run of 1s

Then perform Arithmetic Shift Right (ASR) on the combined A · Q · Q₋₁ register, preserving the sign bit of A. Repeat for n iterations where n = word length of multiplier.

(b) Stack-CPU + RPN Evaluation: (A+B)*(C-D)

A stack-based CPU (zero-address machine) keeps all operands on a hidden operand stack. Instructions name no operand. They just pop the top of stack (TOS), do the work, and push the result. RPN (postfix) fits this style. The operator comes right after its two operands. Think of it like a cinema multiplex queue: last in, first out.

Infix → RPN conversion: (A+B)*(C-D)  ⟶  A B + C D - *

1
PUSH A → Stack: [A].
2
PUSH B → Stack: [A, B].
3
ADD → pop B, pop A, push A+B. Stack: [A+B].
4
PUSH C → Stack: [A+B, C].
5
PUSH D → Stack: [A+B, C, D].
6
SUB → pop D, pop C, push C-D. Stack: [A+B, C-D].
7
MUL → pop C-D, pop A+B, push product. Result on TOS: (A+B)*(C-D).
Stack-CPU Characteristics & Trade-offs
  • Zero-address instructions: math opcodes carry no operand field. So instruction words are short and bytecode is dense.
  • Hidden operand source: ADD/SUB/MUL always pop two from TOS and push the result. Only PUSH and POP touch memory.
  • RPN fits naturally: a postfix expression read left-to-right is the program.
  • Recursion is easy: stack frames support nested calls. No need to save/restore registers.
  • Cost: every temporary goes on the stack. Slower than register machines. But the bytecode is portable (JVM, Wasm).
  • Pop order trap: for SUB and DIV, the second pop is the first operand. The first pop is the second operand.
Modern Relevance
  • IEEE-754 is everywhere: every commodity CPU FPU uses it. ARM Cortex-M0/M3 ship single-precision FPUs. Cortex-M7 adds double. x86 SSE/AVX work on packed singles/doubles.
  • Booth in silicon: modern multipliers use Radix-4 Booth (looks at 3 bits) plus Wallace-tree reduction. They compute 64-bit products in about 3 cycles. The classic 2-bit Booth still shows up in low-power DSPs.
  • Stack machines in production: JVM bytecode (every .class file), .NET CIL, WebAssembly, PostScript, Forth — all run on virtual stack machines. Wasm's stack ISA lets a 2 MB binary run the same on x86, ARM, RISC-V.
  • FMA fusion: AVX-512 and ARM NEON fuse multiply-add into one rounding (a*b + c). More precision, half the instructions for matrix work.
Common pitfalls (1) Bias trap: write the biased exponent E + 127, not the raw exponent. For E = -3, the field stores 124. (2) Hidden bit: the leading 1. of a normal mantissa is hidden. Never write it into the 23 stored bits. (3) Booth: the starting Q₋₁ = 0 bit is a must. Skip it and every pass shifts wrong. (4) Booth uses Arithmetic Shift Right (keeps the sign bit), not Logical Shift Right. (5) RPN: do not flip operand order on SUB. The second-popped value is the minuend. The first-popped is the subtrahend.
Conclusion

IEEE-754 made floating-point one standard across all CPUs after 1985. Booth's encoding cut signed-multiply adds by about 50%. It is still the base of every modern multiplier tree. The zero-address stack-CPU model made portable bytecode (JVM, Wasm) possible. All three ideas — float format, signed multiplier, evaluator stack — live inside every modern CPU and runtime.

Introduction

A Control Unit (CU) makes the timing and control signals. It tells the CPU when to fetch, decode, route operands, latch results, and step PC. There are two ways to build a CU:

  • Hardwired CU — fixed logic gates. Inputs: opcode + flags + step counter. Outputs: hard-wired control lines.
  • Micro-programmed CU (Wilkes, 1951) — control signals are fetched as microinstructions from a small fast Control Memory (a ROM). One main instruction expands into a microroutine: a list of microinstructions that drive the data path one register-transfer at a time.

Inside the microprogrammed CU, the CAR (Control Address Register) (also called μPC) holds the address of the microinstruction being read from Control Memory. After every microcycle, the Address Sequencer updates the CAR. It picks from four next-address sources. The CDR (Control Data Register) latches the fetched microinstruction so its bits can drive control lines in the same clock cycle.

Address Sequencing — the 4 Next-Address Sources

After each microinstruction, the Next-Address Generator picks the new CAR value from one of four sources:

  • (1) Increment: CAR ← CAR + 1. The default. Used inside a microroutine to step linearly. A simple +1 adder does this.
  • (2) Mapping logic (Opcode-Map): the opcode is padded with zeros (e.g. 00·OPCODE·00) or fed through a PLA to give the start address of that instruction's microroutine. This is the FETCH ⇒ EXECUTE jump.
  • (3) Unconditional Branch: CAR ← absolute address from the microinstruction's branch field. Used to jump between microroutines (e.g. EXECUTE ⇒ FETCH).
  • (4) Conditional Branch: CAR ← address-field if the selected status bit (Zero, Carry, Overflow, Interrupt) is true. Else CAR + 1. This is how the CU handles conditional opcodes like JZ, BCS.

A 2-bit field inside every microinstruction picks which of the four sources is used. The Next-Address MUX decodes it.

Mapping Logic — Worked Example

Vertical vs Horizontal Microinstruction Encoding
  • Horizontal microcode: one bit per control line, no decoding. Wide microinstruction (~100 bits). Max parallel work. Low decode delay. Used in mainframe CPUs.
  • Vertical microcode: packed fields decoded into mutually exclusive control signals. Narrow (~16–32 bits). Less parallel work. Smaller ROM. Used in small CISC chips.
  • Nano-programming: two-level. Vertical microinstructions index into a horizontal nanoroutine. Trades one extra ROM read for far fewer total ROM bits.

The reward is flexibility. That is why x86 stayed binary-compatible for 45 years, and why the 2018 Spectre microcode patches shipped in days, not years.

Introduction

An I/O transfer scheme is the rule the CPU uses to move data between a device and RAM. Device speeds went from punch-card readers (10 char/s) to NVMe SSDs (7 GB/s). Three schemes grew over time. Each one uses more hardware to give the CPU more free time. Programmed I/O wastes CPU cycles in a poll loop. Interrupt-driven I/O frees the CPU between transfers. DMA takes the CPU out of the data path completely. Think of DMA like a bus depot: trucks load and unload by themselves while the driver rests.

Interrupt Mechanics — Step-by-Step
  • 1. Interrupt Request (IRQ): the device raises an INT line to the interrupt controller.
  • 2. Acknowledgement (INTA): CPU finishes the current instruction, raises INTA. The device puts its vector number on the data bus.
  • 3. Save context: PC and PSW are pushed to the stack. The IF (Interrupt-enable Flag) is cleared to block new interrupts.
  • 4. Vectored dispatch: CPU looks up the interrupt vector table at IVT_BASE + vector × 4. It loads the ISR start address into PC.
  • 5. ISR execution: the handler serves the device, clears the IRQ source. It can re-enable IF for nested handling.
  • 6. Return (IRET): pops PSW and PC. CPU resumes the old program smoothly.

Maskable interrupts (INTR) respect IF. Non-maskable interrupts (NMI) ignore IF. NMI is used for serious events like memory parity errors or watchdog timeout.

The Three DMA Transfer Modes
  • 1. Burst mode (Block transfer): The DMA controller grabs the bus with HOLD. It moves the full block (1 KB–4 MB) without breaks. Then it lets go with HLDA. CPU waits during this time. Best for fast bulk transfers (disk-to-RAM, SATA, NVMe).

Modern Relevance
  • NVMe SSDs: use queue-pair DMA. Host writes commands to queues in RAM. The SSD's DMA engine pulls commands and pushes data straight between flash and RAM. Hits 7+ GB/s on PCIe Gen4 x4 with the CPU idle.
  • GPU DMA: NVIDIA CUDA and AMD ROCm both use PCIe DMA for host-to-device transfers. New GPUDirect RDMA lets a NIC DMA straight into GPU memory, skipping host RAM. Critical for AI training clusters.

  • Embedded ARM Cortex-M: ships with a built-in DMA Controller (e.g. STM32 DMA1/DMA2). A serial port can feed ADC samples to RAM while the M4 core sleeps. Saves battery.
Common pitfalls (1) Priority inversion: a high-priority handler stuck waiting on a low-priority handler's spinlock. This caused the Mars Pathfinder 1997 watchdog reboot. Fix: priority inheritance. DMA splits data movement from instruction work. This was the biggest speed gain in computer organisation since Eckert-Mauchly. The three transfer modes trade CPU stalls for I/O delay. All modern I/O (PCIe MSI-X, NVMe queue-pair DMA, GPUDirect RDMA, IOMMU isolation) is built on these 1970s ideas.

Introduction

Flynn's taxonomy (1966) sorts parallel computers by two axes: the number of parallel Instruction Streams and the number of parallel Data Streams. The 2×2 box it forms still names every modern parallel system, from a tiny embedded chip to a GPU shading 4096 pixels per clock.

Data \ InstructionSingle Instruction (SI)Multiple Instruction (MI)
Single Data (SD)SISD — Standard uni-core von Neumann CPUMISD — Rare; spacecraft fault-tolerant pipelines (e.g. Space Shuttle 4-AP1)

  • SISD: classic Von Neumann machine. One IF/ID/EX/MEM/WB pipeline. One operand per cycle. Examples: Intel 8086, ARM Cortex-M0.
  • SIMD: one control unit sends one instruction to N data lanes. Each lane runs it on its own element. Examples: NVIDIA GPU warps (32-wide SIMT), Intel AVX-512 (8 doubles per instruction), Cray-1 vector processor.
  • MISD: N instruction streams work on the same data. Used in redundant systems where each stream checks the others. Examples: Space Shuttle's quad-redundant 4-AP1 voting computers, signal-processing systolic arrays.
  • MIMD: N cores each fetch their own instructions and work on their own data. Examples: every modern multi-core x86/ARM CPU, message-passing clusters, NUMA servers.
  • SPMD (Single Program Multiple Data) is a software pattern that sits on top of MIMD. Same binary, different data slices. OpenMP, MPI, and CUDA all use this.
5-Stage Pipeline Timing — 4 Instructions, No Hazards
Pipeline Hazards & Resolutions
  • 1. Structural Hazard: two pipeline stages want the same hardware in the same cycle.
    Examples: single-port memory needs both IF (instruction fetch) and MEM (load/store) at once. One FPU serving both ADD.F and MUL.F.
    Fixes: (a) split into separate instruction and data caches (Harvard architecture); (b) add more units (extra FPUs); (c) insert a bubble (NOP) to stall the clashing stage.
  • 2. Data Hazard: instruction j needs a result from instruction i that has not yet reached WB.
    Three sub-types:
    • RAW (Read-After-Write): j reads what i writes. A true dependency. Example: I1: ADD R1, R2, R3; I2: SUB R4, R1, R5.
    • WAR (Write-After-Read): j writes a register i has not yet read. Anti-dependency. Only matters in out-of-order pipelines.
    • WAW (Write-After-Write): two instructions write the same register. Output dependency. Only matters in out-of-order.
    Fixes: (a) Operand forwarding (bypassing): route the EX/MEM result straight back to the next instruction's ALU input; (b) compiler scheduling: move independent instructions between i and j; (c) stalling: insert pipeline bubbles; (d) register renaming: kill WAR/WAW by giving each write a fresh physical register.
  • 3. Control Hazard (Branch Hazard): branch and jump change the PC after the next instruction has already started its fetch. Wrong instructions are already in the pipe.
    Fixes:
    • Branch Prediction (BTB + 2-bit saturating counter): guess taken / not-taken from history. About 95% right for normal loops.
    • Delayed Branch (MIPS): the ISA promises that the instruction after the branch always runs. The compiler fills that delay slot.
    • Branch Target Buffer (BTB): stores recent branch targets so IF can redirect in the same cycle the branch decodes.
    • Speculative Execution: fetch and run the predicted path before the branch resolves. Squash and replay on mis-predict.
RAW Hazard Trace — Forwarding vs Stall

With EX→EX forwarding, I1's ALU result is routed straight to I2's EX input. So 0 stall. For a load-use pair (I1 = LW R1, 0(R6)), forwarding cannot remove the load delay. Modern Relevance
  • Deep pipelines: Intel Skylake ~14 stages, AMD Zen 4 ~19 stages, Apple M3 ~16 stages. Deeper pipelines raise clock speed but waste more cycles on a branch mispredict (≥ k cycles lost).
  • Out-of-order issue + register renaming: Modern cores (Zen 4, M3, Skylake) rename to 200+ physical registers and schedule 6–10 µops per cycle. This hides cache-miss delay.
  • SIMD is still alive: Intel AVX-512 (16 floats per instruction), ARM SVE / SVE2 (up to 2048 bits), NVIDIA SIMT warps (32-wide). One add instruction can do 64 FP ops on H100 tensor cores.
  • MIMD is everywhere: every smartphone has at least 4 cores. Data-centre Sapphire Rapids has 60 P-cores. Apple M3 Max = 12 P + 4 E cores.
  • Spectre / Meltdown (2018): speculative execution leaks data via cache timing side-channels. Wrong-path loads dirty the cache before squash. Hardware fixes (eIBRS, STIBP, Retpoline) still cost 5–30% on syscall-heavy workloads.
  • VLIW (cousin of SIMD): Itanium and Qualcomm Hexagon DSPs pack many parallel ops in one instruction word. The compiler handles the parallel work, not the hardware.
Common pitfalls (1) RAW vs WAR vs WAW: RAW is a true dependency. It needs forwarding or stall. WAR and WAW are false dependencies. Fix them with register renaming, not forwarding. (2) Forwarding cannot remove load-use stalls. A load result is ready only at the end of MEM. So the next ALU op must wait one cycle. (3) Speedup formula: use n·k / (k + n − 1), not the lazy k. Conclusion

Flynn's 1966 2×2 box still names every parallel machine built today. SISD chips boot. SIMD chips render. MIMD chips serve. MISD chips fly spacecraft. Pipelining is the trick that made SISD 5× faster. Hazards are the cost. Forwarding, branch prediction, and register renaming push the pipeline toward the limit Sp = k. But Spectre showed that aggressive speculation creates security holes that hardware fixes (eIBRS, STIBP) cannot fully seal.

Introduction — Cache Mapping

A cache is a small fast memory between the CPU and main memory. It uses temporal and spatial locality to give a quick answer. Think of it like a library card desk that keeps the most-asked books near the counter. The physical address from the CPU is split so the cache can answer two questions in one cycle: "Is this block here?" (tag compare) and "Which byte in the block?" (byte offset). The split depends on the mapping scheme:

  • Direct-mapped: address = TAG · LINE · WORD. Each block maps to one fixed cache line.
  • Fully associative: address = TAG · WORD. A block can sit in any line. Needs N-way parallel tag compare.
  • k-way set-associative: address = TAG · SET · WORD. Each block maps to a SET (group of k lines). Any line within the set is fine. A middle ground between direct and fully associative.
(a) Cache Bits Computation — Given Configuration

Given: Cache size = 64 KB, block size = 16 B, 2-way set-associative, physical address = 32 bits.

Belady's Optimal is the theoretical floor. LRU (and its CLOCK approximation) gets within a few percent. FIFO survives only where simplicity wins over correctness. The OS kernel still solves this tradeoff inside mm/swap.c every day.

  • CAR / CDR: Control Address Reg + Control Data Reg. μPC drives Control Memory.
  • Address sequencing: 4 sources — Increment, Mapping logic, Uncond branch, Cond branch.
  • Horizontal vs Vertical microcode; Nano-programming (two-level).
  • I/O techniques: Programmed I/O (poll), Interrupt-driven, DMA.
  • Interrupts: IRQ → INTA → save context → IVT lookup → ISR → IRET. Maskable / NMI.
  • DMA controller: Address reg + Word count + Control reg. Three modes: Burst / Cycle-stealing / Transparent.
  • I/O bus structure, daisy chaining, priority arbitration.
Unit III · Pipelining + Parallel + Coherence
  • Flynn's taxonomy: SISD, SIMD, MISD, MIMD. SPMD as MIMD software pattern.
  • Pipelining: 5-stage RISC (IF-ID-EX-MEM-WB). Speedup S = nk/(k+n−1).
  • Hazards: Structural (resource clash), Data (RAW/WAR/WAW), Control (branch).
  • Resolutions: Forwarding (bypassing), Stalls, Compiler scheduling, Register renaming.
  • Branch handling: Prediction (2-bit counter), Delayed branch, BTB, Speculative execution.
  • Vector / Array processors: Cray-1, SIMD lanes, AVX-512, NVIDIA SIMT warp (32-wide).
  • Cache coherence: MESI / MOESI protocols. Bus snooping, directory-based.
  • Multiprocessors: UMA, NUMA, message-passing clusters.
Unit IV · Memory Organisation
  • Memory hierarchy: Reg → L1 → L2 → L3 → DRAM → SSD → HDD. Cost ↑, speed ↓.
  • Locality: Temporal (re-access soon) + Spatial (neighbour access).
  • Cache mapping: Direct, Fully associative, k-way Set-associative.
  • Address split: TAG · SET · WORD. WORD = log₂(block); SET = log₂(lines/k); TAG = remainder.
  • Hit ratio: H = hits / (hits + misses). Effective access time T = H·Tc + (1−H)·Tm.
  • Memory interleaving: low-order vs high-order; banked DRAM for parallel access.
  • Virtual memory: Paging, Page table, TLB. Page fault → OS swap.
  • Page replacement: FIFO (Belady's anomaly), LRU, Optimal, CLOCK / Second-Chance.
  • Write policies: Write-through (RAM always synced) vs Write-back (lazy, dirty bit).

🧠 If You Remember Only 5 Things — Walk in With These
  1. Micro-programmed CU + CAR trace. The CAR holds the next microinstruction address. It is updated by one of 4 sources: Increment, Mapping (opcode → start), Uncond Branch, Cond Branch. Walk the ADD R1, R2 trace: addr 0 (FETCH-1) → 1 → 2 (decode, map) → 40 → 41 → 42 → 43 → back to 0.

  4. DMA cycle-stealing + 3 modes. DMA has bus priority because I/O data is volatile. Three modes: Burst (full block, CPU stalled), Cycle-stealing (1 cycle per word, CPU slows), Transparent (only on idle CPU cycles, zero impact). Block diagram: Address reg + Word count + Control reg.
