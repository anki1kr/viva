Eight anchors that buy you 18+ marks

  1. RAM cycle (RTL) for fetch: MAR←PC · MBR←M[MAR] · IR←MBR · PC←PC+1 — every numerical / theory answer that mentions execution cycle opens here.
  2. Master mnemonic — pipeline stages: I-D-X-M-W · Instruction-fetch, Decode, Execute, Memory, Write-back. Hazards = 3 types: Structural · Data · Control. Memorise Speedup = n·k / (k+n−1).
  3. Cache mapping triad — DAS: Direct · Associative · Set-associative. TAG/SET/WORD split: WORD = log₂(words per block), SET = log₂(number of sets), TAG = rest of main-memory address bits. Always state hit-time, miss-penalty, replacement.
  4. Booth's recoding table (qi, qi−1): 00→0 · 01→+M · 10→−M · 11→0. Arithmetic shift right after each pass. Works for signed 2's-complement multiplicands.
  5. Addressing modes mnemonic — "I-D-I-R-IND-IDX-BAS-REL-STK" (Immediate · Direct · Indirect · Register · Register-indirect · Indexed · Base-register · Relative · Stack). Each mode has 1 EA formula — write it.
  6. DMA priority rule: "I/O is slow, CPU is patient" — but DMA gets priority because peripheral data is volatile (lost if not serviced). 3 DMA modes: burst · cycle-stealing · transparent.
  7. Floating-point IEEE-754 single: 1 sign · 8 exponent (bias 127) · 23 mantissa · normalised form 1.f × 2^(E−127). To normalise: shift mantissa until leading 1 is implicit, adjust exponent.
  8. Write policy decision tree: latency-sensitive ⇒ write-through + write-allocate; bandwidth-sensitive ⇒ write-back + dirty bit. Coherence in multi-core ⇒ MESI on write-back.

1.2 Stored-program control concept PYQ

Definition The stored-program control concept — proposed by John von Neumann (1945) in the EDVAC report — states that both the program (instructions) and the data on which it operates are stored in the same read-write main memory. The CPU fetches an instruction, decodes it, executes it, and repeats — guided only by the program counter.

Five defining properties

  1. Single memory for both instructions & data.
  2. Sequential execution driven by a program counter (PC).
  3. Instructions are binary patterns — treatable as data (self-modifying code is possible, though discouraged).
  4. Fetch–decode–execute is the universal control loop.
  5. Same hardware can be reprogrammed by loading a different program — making the machine general-purpose.
Why it matters Every modern CPU (Intel, ARM, RISC-V) is a descendant of von Neumann's model. The only modification at scale is the Harvard architecture (separate instruction & data caches) — used in DSPs and the L1 of almost every modern CPU — to break the von Neumann bottleneck (single bus is the throughput limit).
How prof asks this Asked as 1.5-mark short in LOC_CSE24 Q1(a). Senior profs want the year (1945, EDVAC), the single-bus model, and "instructions-as-data" to be named explicitly. A two-sentence answer scores full 1.5 only if all three are present.

The four blocks A computer comprises four interconnected blocks: (1) CPU — fetches, decodes, executes; (2) Memory — stores instructions & data; (3) I/O subsystem — bridges the machine to the outside world; (4) Control unit — sequences operations by generating timing signals. They communicate over three buses: address, data, control.

1.4 CPU registers · why they exist PYQ

Definition A register is a fast, small (typically 1-128 words) storage location built directly inside the CPU — accessed in a single clock with zero memory-bus delay. Registers hold operands, addresses, status flags, and control state.

Why registers are used — 5 reasons (LOC23 Q1d, 1.5 M)

  1. Speed gap — register access is < 1 ns; main memory access is 50-100 ns. Computation parks operands in registers to avoid the memory wall.
  2. Reduce memory traffic — a value reused 5× is fetched once and cached in a register.
  3. Hold control state — PC, IR, flags can't live in memory because the CPU needs them every cycle.
  4. Addressing modes — register-indirect, indexed, base-register modes all need registers as part of the EA computation.
  5. Compiler optimisation — register allocation algorithms (graph colouring) yield the biggest single performance win for compiled code.

1.5 Program Status Word (PSW) PYQ

Definition The Program Status Word is a special CPU register that collects status flags set by the ALU and system-state bits set by the CU into a single word. It is saved on interrupt entry and restored on interrupt return, preserving the program's full execution context.

Standard PSW fields (8086 / generic)

  • Z — Zero flag (result was zero)
  • S / N — Sign / Negative flag (MSB of result)
  • C — Carry flag (unsigned overflow out of MSB)
  • V / O — Overflow (signed overflow)
  • P — Parity (8086 sets P=1 when low byte has even number of 1s)
  • AC / A — Auxiliary carry (carry from bit 3 to 4 — used by BCD adjust)
  • I / IF — Interrupt enable
  • T / TF — Trap (single-step)
  • D / DF — Direction (string ops)
  • Mode — User vs Kernel (privileged) bit on most architectures

How prof asks this LOC_CSE24 Q1(b) asks "What is program status word?" (1.5 M) and Q5(a) gives a numerical: "perform addition on −81 and FE (hex). What is the resultant FLAGS value?" — full flag-by-flag walk-through follows.

Worked: 8086 flag computation (LOC_CSE24 Q5a, 5 M)

Perform 16-bit addition of −81 (decimal) and FE (hex). Determine CF, ZF, SF, OF, PF, AF.

Encode operands. −81 in 16-bit 2's complement: +81 = 0000 0000 0101 0001; invert = 1111 1111 1010 1110; +1 = 1111 1111 1010 1111 = FFAF16. FE16 = 0000 0000 1111 1110.
Add. FFAF + 00FE: bit-by-bit gives 1 0000 0000 1010 1101 → result = 00AD, carry-out = 1.
CF = carry out of bit 15 = 1.
ZF = (result == 0)? 00AD ≠ 0 → ZF = 0.
SF = MSB of result = bit 15 of 00AD = 0.
OF = signed-overflow: signed inputs −81 and +254; signed result = +173 fits the 16-bit signed range → OF = 0.
AF = carry from bit 3 to bit 4 in the low nibble. Low nibble of FFAF = F (1111), of 00FE = E (1110). 1111 + 1110 = 1 1101 → carry into bit 4 → AF = 1.
PF = even parity of low byte. Low byte 0xAD = 1010 1101 has 5 ones (odd) → PF = 0 (PF=1 only when ones-count of the low byte is even).

FLAGS: CF=1, ZF=0, SF=0, OF=0, AF=1, PF=0.

1.6 Instruction execution cycle & RTL CRITICAL

Definition The instruction execution cycle is the repetitive sequence by which the CPU processes each instruction, expressible as four micro-phases: fetch → decode → execute → write-back (with optional indirect and interrupt sub-cycles). Each phase is described in Register-Transfer Language (RTL) — a notation where X ← Y denotes a single-clock register transfer.

RTL for a complete cycle

FETCH PHASE T1: MAR ← PC ; address of instruction onto MAR T2: MBR ← M[MAR], PC ← PC + 1 ; read memory, advance PC T3: IR ← MBR ; decode-ready in IR DECODE PHASE T4: decode IR(opcode) ; CU determines class of instruction if indirect bit = 1 → INDIRECT cycle INDIRECT PHASE (optional) T5: MAR ← IR(address) T6: MBR ← M[MAR] T7: IR(address) ← MBR EXECUTE PHASE — example for ADD M T8: MAR ← IR(address) T9: MBR ← M[MAR] T10: AC ← AC + MBR update PSW(Z,S,C,V) INTERRUPT PHASE (if pending) T11: MBR ← PC T12: MAR ← SP, SP ← SP − 1 T13: M[MAR] ← MBR T14: PC ← interrupt-vector address
Common mistake Students write PC ← PC + 1 at the start of fetch. Wrong — PC must be copied into MAR first, then incremented after the memory read is issued. Reversing the order causes the wrong instruction to be fetched on the next cycle.

Worked numerical (LOC_CSE24 Q1(i), 1.5 M)

An instruction is stored at location 200 with the address field at 201. The value of the address field is 320. Processor register R1 contains 500. Find EA for: (i) Direct (ii) Relative (iii) Register-indirect.

Direct: EA = A = 320. The address field is the EA.
Relative: EA = PC + A. After fetch, PC has advanced to next instruction (instruction = 2 words at 200, 201, so PC=202). EA = 202 + 320 = 522.
Register indirect: EA = (R1) = 500. The address field is ignored; the EA comes from the register's contents.

EA values: 320, 522, 500.

Mnemonic "I-D-I-R-IND-IDX-BAS-REL-STK" — say it like a rap. Immediate · Direct · Indirect · Register · Register-INDirect · IndeXed · BAS-register · RELative · STacK.
Common mistake For relative addressing, students forget that PC has already advanced by the time the EA is computed. The PC value to use is address of the next instruction, not the current one.

Stack item count (LOC23 Q1(h), 1.5 M)

Let SP = 0000 in the stack. How many items are there in the stack if FULL = 1 and EMPTY = 0?

SP is an n-bit counter that wraps around. SP = 0000 normally means stack empty (next-PUSH location).
But FULL = 1 and EMPTY = 0 simultaneously means the stack has wrapped — it's full, not empty.
For an n-bit SP, the stack capacity is 2n. With SP=0000 indicating completion of a full wrap, the stack holds 2n items.

For SP = 4 bits, stack holds 16 items (24). For SP = 8 bits, it holds 256 items.

How prof asks this Classic Morris-Mano problem. The trick is recognising the wrap-around: 0000 alone is ambiguous, but FULL=1 + EMPTY=0 disambiguates. Always state the SP bit-width assumption.

Worked: subtract (36)10 − (47)10 using 2's complement (loc25 Q2a)

Compute 36 − 47 in 8-bit 2's complement.

Represent 36 = 00100100 ; 47 = 00101111.
2's complement of 47 = invert(00101111)+1 = 11010000 + 1 = 11010001.
Add: 00100100 + 11010001 = 11110101. No carry out → result is negative.
Interpret: invert(11110101)+1 = 00001010 + 1 = 00001011 = 1110. So result = −11.

36 − 47 = −11, represented as 11110101 in 8-bit 2's complement.

Worked: base conversions (loc24 Q2b style, 5 M)

(i) (79.4375)10 → octal · (ii) (5D.8F)16 → binary · (iii) (10111011.1101)2 → decimal.

(i) 79 to octal. 79÷8 = 9 r 7; 9÷8 = 1 r 1; 1÷8 = 0 r 1. Read remainders bottom-up → integer = 117. Fraction 0.4375: 0.4375×8 = 3.5 → digit 3, remainder 0.5; 0.5×8 = 4.0 → digit 4, stop. Result: (117.34)8.
(ii) 5D.8F to binary. Each hex digit → 4 bits. 5 = 0101, D = 1101, 8 = 1000, F = 1111. Result: (0101 1101 . 1000 1111)2.
(iii) 10111011.1101 to decimal. Integer 10111011 = 128 + 0 + 32 + 16 + 8 + 0 + 2 + 1 = 187. Fraction .1101 = 1/2 + 1/4 + 0 + 1/16 = 0.5 + 0.25 + 0.0625 = 0.8125. Result: (187.8125)10.

Rule: integer part by repeated division (read remainders bottom-up); fraction by repeated multiplication (read integer parts top-down).

1.12 Fixed & Floating-point representation · IEEE-754 PYQ ×3

Definition Fixed-point representation places the binary point at an implicit fixed position — fast but limited dynamic range. Floating-point representation stores a number as (−1)S × 1.M × 2E−bias, giving a vast dynamic range by sacrificing uniform precision.

IEEE-754 Single precision (32-bit)

Layout · 32 bits ┌───┬──────────┬──────────────────────────┐ │ S │ Exp (8b) │ Mantissa / Fraction (23b) │ └───┴──────────┴──────────────────────────┘ bit 31 30-23 22-0 Value = (-1)ˢ × 1.M × 2^(E - 127) ← bias = 127 Special encodings: E=0, M=0 → ±0 E=0, M≠0 → denormal: (-1)ˢ × 0.M × 2^(-126) E=255, M=0 → ±∞ E=255, M≠0 → NaN

Worked: represent −262.12510 in IEEE-754 single (LOC_CSE24 Q2b)

Convert −262.125 to single-precision IEEE-754.

Sign: negative ⇒ S = 1.
Magnitude in binary: 262 = 256+4+2 = 100000110. Fraction 0.125 = 1/8 = 0.001. So 262.125 = 100000110.001.
Normalise: 100000110.001 = 1.00000110001 × 28. Implicit leading 1, so mantissa stored = 00000110001 padded with zeros to 23 bits: 00000110001 0000000000 00.
Exponent: E = 8 + 127 (bias) = 135 = 10000111.
Concatenate S | E | M:
1 | 10000111 | 00000110001 0000000000 00
= 1100 0011 1000 0011 0001 0000 0000 0000
= 0xC3830000.

−262.125 = 0xC3830000 in IEEE-754 single precision.

Normalisation & addition of two FP numbers (LOC23 Q2a)

A floating-point number is normalised when its leading mantissa digit is non-zero — in IEEE-754 binary, the mantissa is always 1.xxx (implicit leading 1). Addition follows four steps:

  1. Align exponents — shift the mantissa of the smaller exponent right until exponents match.
  2. Add the aligned mantissas.
  3. Normalise the result — shift mantissa & adjust exponent until leading bit is 1.
  4. Round the mantissa to fit the target precision.

Add 0.9504×101 and 0.8200×101, give the normalised result.

Exponents are equal (both 101) — no alignment needed.
Add mantissas: 0.9504 + 0.8200 = 1.7704.
Result: 1.7704 × 101. Not normalised (mantissa ≥ 1) — shift right, increment exponent.
Normalised: 0.17704 × 102.

Sum = 0.17704 × 102.

1.12b Character representation · ASCII & Unicode Syllabus

Definition ASCII (American Standard Code for Information Interchange) is a 7-bit character code that maps 128 symbols (codes 0–127) to printable letters, digits, punctuation, and control characters. Extended ASCII uses the eighth bit to reach 256 characters (codes 128–255) for international glyphs and box-drawing symbols.

Two categories of ASCII codes

  • Printable (0x20–0x7E, i.e. 32–126) — space, digits, letters, punctuation. Visible on a terminal.
  • Control / non-printable (0x00–0x1F and 0x7F) — line feed, carriage return, bell, escape, tab, backspace. Used for device control and stream framing, not displayed as glyphs.

Worked examples

  • 'A' = 0x41 = 6510, 'a' = 0x61 = 9710 — note the constant offset of 32 between upper and lower case (bit 5 toggles case).
  • '0' = 0x30 = 4810, '9' = 0x39 = 5710 — the low nibble is the digit value (subtract 0x30 to get the numeric value).
  • String "Hi" = 0x48 0x69 = two bytes in memory.

Successors of ASCII

  • EBCDIC (IBM, 8-bit) — used on mainframes; non-contiguous letter codes make sorting harder than in ASCII.
  • Unicode — universal code-point space (currently > 149,000 characters across all scripts). The first 128 code-points are identical to ASCII.
  • UTF-8 — variable-length (1–4 bytes) encoding of Unicode. ASCII text is valid UTF-8 byte-for-byte, which is why ASCII still matters for memory layout and I/O.
Why this matters in LOC Character storage at the byte level explains string layout in main memory, how I/O peripherals frame text streams (LF / CR control codes), and why a 1 KB cache block holds ~1024 ASCII characters but only ~256–1024 UTF-8 code-points depending on script. The bit-level structure of 'A' vs 'a' also surfaces in toggle-case tricks at the instruction level (XOR with 0x20).

1.13 Ripple-carry adder · Carry-look-ahead adder PYQ

Definition An n-bit ripple-carry adder (RCA) chains n full-adders, propagating the carry from bit 0 up to bit (n−1). It is simple but slow — worst-case delay is n · tFA. The carry-look-ahead adder (CLA) computes carries in parallel using generate (Gi = AiBi) and propagate (Pi = Ai⊕Bi) signals, reducing delay to O(log n).

RCA equations (single full-adder)

Sum : Si = Ai ⊕ Bi ⊕ Ci Carry: Ci+1 = AiBi + (Ai⊕Bi)Ci

CLA generate & propagate

An n-bit adder/subtractor reuses an RCA: control line M selects add (M=0) or subtract (M=1). Each Bi XORs with M, and C0 = M (forces 2's-complement of B when subtracting).

Why CLA wins RCA delay is O(n) — for 64-bit operands, this becomes the critical path. CLA reduces delay to O(log n) by computing all carries in parallel from G/P signals. Modern CPUs use 4-bit CLA blocks chained with carry-select for the best speed/area trade-off.

1.15 Carry-save multiplier · (3:2) compressors & final CPA Syllabus

Definition A carry-save multiplier reduces the n partial products of an n-bit multiplication to two values — a sum vector S and a carry vector C — using a tree of carry-save adders (CSAs). A single carry-propagate adder (CPA) at the bottom then combines S + C into the final product. The trick is that each CSA layer takes O(1) time regardless of operand width, so n partial products collapse in O(log n) levels instead of O(n).

Key idea · the (3:2) compressor

Each CSA cell is built from one full-adder treated not as an adder but as a (3:2) compressor: it takes three input bits in the same bit-column and produces two output bits — a sum bit that stays in this column, and a carry bit that moves one column left (to be consumed by the next CSA layer, not propagated this layer). Because the carry never propagates within a layer, every column resolves in one full-adder delay no matter how wide the operands are.

CSA cell · column i Inputs : xi, yi, zi (three bits, same column) Outputs: si = xi ⊕ yi ⊕ zi ← sum, stays at column i ci+1 = xiyi + yizi + xizi ← carry, shifts to column i+1 (NEXT layer)

Dataflow · n=3 partial-product example

Advantages over ripple-add of partial products

  • Within each CSA layer carry propagation is hidden — the carry bit shifts between layers rather than rippling through one layer. Layer delay is one full-adder, independent of operand width.
  • Carry propagation happens exactly once, in the final CPA — that single CPA can itself be a fast adder (CLA, carry-select, prefix).
  • For n partial products: ripple-add path is O(n) full-adder delays; CSA-tree path is O(log3/2 n) ≈ O(log n). Wallace and Dadda trees realise this asymptotic win.

The same (3:2) compressor reappears in the multi-operand adders used by accumulators and the MAC unit at the heart of every signal-processing kernel.

2.1 CPU control unit — hardwired vs microprogrammed PYQ ×3

Definition The control unit (CU) generates the timed sequence of control signals that drive every other CPU block (ALU mux selects, register loads, memory R/W, bus enables). Two design philosophies: hardwired — CU is a fixed combinational + sequential circuit; microprogrammed — CU executes a tiny "program" stored in a fast read-only control memory.

Hardwired control unit

  1. Inputs: opcode (from IR), timing signals (T1..Tn from a ring counter), status flags.
  2. Outputs: control lines (e.g. MAR_load, ALU_add, MEM_read).
  3. Realised as a sum-of-products PLA / random logic — one Boolean equation per control line.
  4. Fast (one gate delay per signal) but inflexible — adding an instruction requires redesigning logic.

Microprogrammed control unit · key components

  • Control memory (CM) — ROM holding micro-instructions. Each micro-instruction is a horizontal bit-vector specifying control signals.
  • Control Address Register (CAR) — points to the next micro-instruction.
  • Control Buffer Register (CBR) — holds the current micro-instruction.
  • Sequencer / Next-address generator — decides where CAR points next (increment, branch, map from opcode, subroutine return).

2.2 Address sequencing in microprogrammed CU PYQ · 15 M

Definition Address sequencing is the mechanism by which the CAR (Control Address Register) is updated each cycle to fetch the next micro-instruction from control memory. Four sources of the next address feed a multiplexer driving CAR.

Four ways the next address is produced

  1. Incrementing CAR (CAR ← CAR + 1) — default sequential flow inside a routine.
  2. Unconditional branch — branch address from the current μ-instruction's BRA field.
  3. Conditional branch — branch taken only if a status flag (Z, C, V, S) matches the BRA-condition field; else CAR+1.
  4. Mapping from opcode — at the start of EXECUTE, the opcode in IR is fed through a small mapping ROM that produces the start address of that opcode's microroutine.
  5. Subroutine return — a microsubroutine register (SBR) saves return address before branching into a shared micro-routine (e.g. FETCH).

Worked: CAR trace for ADD M (concrete μ-routine)

Trace CAR values through one execution of ADD M (memory-reference, direct addressing). Assume FETCH lives at μ0–μ1 and MAP yields μ23 for the ADD opcode.

μ0 (CAR = 0) FETCH: MAR ← PC; M ← read; PC ← PC + 1. Next-address = CAR + 1 → CAR = 1.
μ1 (CAR = 1) FETCH: IR ← MBR. Next-address = MAP(opcode) → CAR = 23.
μ23 (CAR = 23) ADD-routine: MAR ← IR.addr. Next = CAR + 1 → CAR = 24.
μ24 (CAR = 24) ADD-routine: MBR ← M[MAR]. Next = CAR + 1 → CAR = 25.
μ25 (CAR = 25) ADD-routine: AC ← AC + MBR; update PSW. Next = unconditional branch → CAR = 0 (back to FETCH).

CAR sequence: 0 → 1 → 23 → 24 → 25 → 0. Shows all 4 address sources: increment (μ0→1, μ23→24, μ24→25), opcode-map (μ1→23), and unconditional branch (μ25→0).

2.6 Memory organisation & bus structure

A semiconductor memory chip exposes three interfaces:

  • Address bus — log₂(N) lines for an N-word memory.
  • Data bus — word-width lines (8 / 16 / 32 / 64).
  • Control bus — R/W, CE (chip enable), OE (output enable).

  1. Word-width expansion — join chips side-by-side to widen the data bus. Two 1K × 4 chips → 1K × 8.

  3. Both — combine the above for arbitrary memory size.

Why DMA has priority over CPU (LOC23 Q5a)

When both CPU and DMA request the bus, the arbiter grants the bus to DMA first. Reason: peripheral data is volatile — if the DMA doesn't service an incoming disk sector within the rotational deadline, the data is lost. The CPU, by contrast, can simply wait an extra cycle without harm. So bus priority follows the rule "slow but deadline-strict peripherals beat fast but deadline-flexible CPU".

DMA vs Programmed/Interrupt I/O — saved cycles For a 512-byte disk sector transfer: programmed I/O = 512 polling loops; interrupt-driven = 512 interrupt cycles; DMA = 2 interrupts (start, end). DMA wins by 256× on CPU overhead alone.

Priority interrupts

When multiple devices interrupt simultaneously, a priority interrupt controller (PIC, e.g. Intel 8259A) arbitrates. Two common schemes:

  • Daisy chain — INTA signal flows serially through devices; first to grab it wins.
  • Parallel priority encoder — encoder converts active IRQ lines into a binary priority code; highest-priority IRQ is granted.

Role of interrupts in process-state transitions

A process in a multitasking OS moves between these states: New → Ready → Running → Waiting → Terminated. Interrupts drive most transitions:

  • Running → Ready — timer interrupt at end of quantum (preemption).
  • Running → Waiting — software interrupt (system call: read disk).
  • Waiting → Ready — hardware interrupt (disk DMA complete).
  • Running → Terminated — exception (illegal opcode, divide-by-zero).
Interrupts drive process-state transitions The OS uses interrupts to change a process's lifecycle state: Timer interrupt → preempt running process → READY; I/O completion interrupt → wake a process from BLOCKED → READY; Page-fault exception → suspend RUNNING → BLOCKED until the disk-service ISR completes. Without interrupts the kernel cannot regain CPU — every modern OS scheduler is interrupt-driven.
Privileged vs non-privileged instructions Most CPUs split instructions into privileged (only runnable in kernel mode — I/O, interrupt control, page-table writes) and non-privileged (user mode — arithmetic, branches). Attempting a privileged instruction from user mode raises an exception, transferring control to the OS — the foundation of OS protection.

2.11 Handshaking · Synchronous vs Asynchronous data transfer PYQ

Handshake sequence (sender → receiver)

  1. Sender places data on bus and asserts DV.
  2. Receiver sees DV, latches data, asserts DA.
  3. Sender sees DA, deasserts DV.
  4. Receiver sees DV low, deasserts DA. Cycle complete.

3.1 Basic concept of pipelining

Definition Pipelining overlaps the execution of multiple instructions by splitting each instruction into a sequence of k stages and processing successive instructions in successive stages — much like an assembly line. The number of instructions completed per unit time (throughput) grows ≈ k-fold without changing the latency of any single instruction.

Classical 5-stage RISC pipeline

  1. IF — Instruction Fetch (from I-cache)
  2. ID — Instruction Decode + register read
  3. EX — Execute / ALU operation
  4. MEM — Memory access (load/store)
  5. WB — Write-back result to register file

3.2 Throughput · Speedup · Efficiency PYQ

Three pipeline metrics For a k-stage pipeline processing n tasks, with cycle time τp (pipelined) and τn (non-pipelined):
Throughput = n / [(k + n − 1) · τp] Speedup Sk = Tseq / Tpipe = (n · k · τp) / [(k + n − 1) · τp] = (n · k) / (k + n − 1) For n → ∞: Sk → k (theoretical max = number of stages) Efficiency = Sk / k = n / (k + n − 1)

Worked: speedup of a 4-stage pipeline on 100 instructions (LOC_CSE24 Q1h)

A 4-stage pipeline executes 100 instructions, each taking 4 ns per stage. Compute speedup vs non-pipelined.

Pipelined time = (k + n − 1) · τp = (4 + 99) · 4 = 412 ns.
Non-pipelined time = n · k · τp = 100 · 4 · 4 = 1600 ns.
Speedup = 1600 / 412 = 3.88.

Speedup ≈ 3.88 (theoretical limit = 4 stages; 97% of ideal).

Worked: efficiency and throughput (predicted 5 M)

A 6-stage pipeline processes 50 instructions with stage cycle τp = 2 ns. Compute (a) total time, (b) throughput, (c) speedup, (d) efficiency.

Total time = (k + n − 1) · τp = (6 + 49) · 2 = 110 ns.
Throughput = n / total time = 50 / 110 = 0.4545 instr/ns ≈ 454.5 MIPS.
Non-pipelined time = n · k · τp = 50 · 6 · 2 = 600 ns. Speedup = 600 / 110 = 5.45.
Efficiency = S / k = 5.45 / 6 = 0.909 = 91 % (since n = 50 ≫ k = 6, fill/drain overhead is small).

As n grows, efficiency approaches 100% — fill-and-drain overhead amortises across the workload.

Why speedup < k always Three penalties stop you from reaching the theoretical limit: fill time (first k−1 cycles produce nothing), drain time (last k−1 cycles trail off), and stalls from hazards. The first two vanish for large n; the third doesn't.

3.3 Pipeline hazards PYQ · 15 M

Definition A pipeline hazard is a situation that prevents the next instruction from executing in its designated cycle, forcing the pipeline to stall (insert a "bubble"). Three categories:

(1) Structural hazards

Two instructions need the same hardware resource in the same cycle (e.g. one memory unit shared between IF and MEM). Resolved by:

  • Adding hardware (split I-cache and D-cache — the Harvard L1).
  • Stalling the conflicting instruction by one cycle.

(2) Data hazards

An instruction depends on the result of an earlier in-flight instruction. Three sub-types:

RAW (Read After Write) — true dependence I1: ADD R1, R2, R3 ; R1 written in WB I2: SUB R4, R1, R5 ; needs R1 in ID — stalls WAR (Write After Read) — antidependence I1: ADD R4, R1, R5 ; reads R1 I2: ADD R1, R2, R3 ; writes R1 — not a problem in in-order pipelines; matters in OoO WAW (Write After Write) — output dependence I1: ADD R1, R2, R3 ; writes R1 I2: SUB R1, R4, R5 ; also writes R1 — only matters in multi-issue / OoO

Resolutions: data forwarding (bypass), pipeline interlock (stall until producer's result is available), compiler scheduling (instruction reordering).

(3) Control hazards (branch hazards)

Branches change PC, but the pipeline has already fetched the next 1-3 instructions. Resolutions:

  • Stall on branch — freeze pipeline until branch resolves. Costly.
  • Branch prediction — guess taken/not-taken, squash on misprediction. 90%+ accuracy in modern CPUs.
  • Delayed branch — execute the instruction after the branch unconditionally (MIPS-style). Compiler fills the delay slot.
  • Branch target buffer (BTB) — small cache of branch destinations indexed by PC.

3.6 Concurrent memory access

In a multiprocessor or DMA-equipped system, several masters may try to access main memory simultaneously. Concurrent memory access is resolved by:

  • Memory interleaving — split memory into banks; consecutive addresses go to different banks, allowing parallel access.
  • Memory-bus arbitration — central arbiter grants the bus one master at a time.
  • Multi-port memory — physically separate read/write ports (dual-port SRAM used in cache & FPGAs).
  • NUMA partitioning — each CPU has its own "local" memory; remote access is slower.
Why interleaving works Bank access has roughly equal latency. If consecutive memory accesses target different banks, the bus utilisation approaches 100% — k-way interleaving theoretically gives k× peak bandwidth.

3.7 Cache coherence PYQ

Definition Cache coherence is the property that all CPU caches in a multiprocessor maintain a consistent view of shared memory: any read of a memory location returns the value of the most recent write, regardless of which cache the writer/reader uses.

The coherence problem

Two cores each cache the same line X = 5. Core A writes X = 7 to its cache (write-back). Core B reads X — sees stale 5. This is a coherence violation.

Two protocol families

  • Bus-snooping protocols — every cache controller watches (snoops) all bus transactions. On seeing another core write to a line it has cached, it invalidates its copy. Most common: MESI.
  • Directory-based protocols — a central directory tracks which caches hold each line. Used in NUMA / large multiprocessor systems.

4.4 Cache memory — hit ratio, miss penalty PYQ ×2

Definition Cache memory is a small (KB-MB), fast (SRAM) memory placed between CPU and main memory. It holds recently/frequently used data so subsequent accesses bypass the slow DRAM path. Performance is dominated by the hit ratio h — the fraction of accesses that find their data in the cache.

Average access time formula

Tavg = h · Tcache + (1 − h) · Tmain h = hit ratio ; 0 ≤ h ≤ 1 Tcache = cache access time Tmain = main-memory access time (includes miss penalty) Miss rate = 1 − h Miss penalty = Tmain − Tcache (extra time the miss costs)

Locality of reference (LOC23 Q1b)

Principle The locality of reference principle says memory accesses cluster — most references in a short interval are to a small set of addresses. Two flavours:
  • Temporal locality — a word accessed now is likely to be accessed again soon.
  • Spatial locality — addresses near a recent access are likely to be accessed soon.
Cache memory exploits both: hold recently-touched words (temporal) in cache lines of 32-128 bytes (spatial).

Improving cache performance (LOC23 Q5b)

Hit ratio is increased by:

  • Increasing cache size — diminishing returns; doubling size raises hit-ratio by ~1-2 percentage points.
  • Increasing associativity — direct ↔ 2-way ↔ fully associative reduces conflict misses.
  • Increasing block size — exploits spatial locality, but raises miss penalty.
  • Prefetching — speculatively fetching the next block.
  • Victim cache — small fully-associative cache for recently evicted blocks.

Worked: average access time

Cache access time = 5 ns; main-memory access time = 100 ns; hit ratio h = 0.95. Find Tavg and the miss-penalty contribution.

Tavg = h·Tcache + (1 − h)·Tmain = 0.95·5 + 0.05·100 = 4.75 + 5 = 9.75 ns.
Miss-penalty contribution = (1 − h)·Tmain = 5 ns — half the average even though only 5 % of accesses miss.
If h drops to 0.90: Tavg = 4.5 + 10 = 14.5 ns — a 5-point hit-rate drop costs ~50 % extra time. Hit ratio matters non-linearly.

Cache effectiveness depends on h staying above ≈0.95 — that is why doubling cache size yields only 1-2 % h improvement but is still worth it.

4.5 Cache mapping — Direct · Associative · Set-associative CRITICAL

Definition A mapping function determines which cache slot can hold a given main-memory block. Three schemes:

(1) Direct mapping

Block j of main memory maps to a single cache line i = j mod c (c = number of cache lines). Address splits into TAG · LINE · WORD.

┌────────┬──────┬──────┐ │ TAG │ LINE │ WORD │ └────────┴──────┴──────┘ WORD = log₂(words per block) LINE = log₂(number of cache lines) TAG = remaining bits Hit if cache[LINE].tag == address.TAG AND valid bit set
  • Pros: Cheapest, 1 comparator.
  • Cons: Highest conflict misses (two hot blocks mapping to same line thrash).

(2) Fully associative mapping

Any block may sit in any cache line. Address splits into TAG · WORD only. On lookup, all tags are compared in parallel.

  • Pros: Zero conflict misses.
  • Cons: Expensive (one comparator per line). Used only for very small caches (TLBs).

(3) Set-associative mapping

Cache lines grouped into s sets of k lines each (k-way set associative). Block j maps to set j mod s; within the set it may sit in any of the k ways. Address splits into TAG · SET · WORD.

┌────────┬──────┬──────┐ │ TAG │ SET │ WORD │ └────────┴──────┴──────┘ SET = log₂(number of sets) k comparators per set (k-way)

Modern L1 caches are 4-way to 8-way; L2/L3 are 8-way to 16-way. Best balance of cost and miss rate.

UNIVERSAL CACHE-SIZING RECIPE 1. Compute main-memory address-width: log₂(main-memory in addressable units) 2. WORD = log₂(words per block) ; block-offset bits 3. Compute total cache lines = cache size / block size 4. For DIRECT: LINE = log₂(cache lines); TAG = total − LINE − WORD 5. For SET-ASSOC: sets = cache lines / k; SET = log₂(sets); TAG = total − SET − WORD 6. For FULLY-ASSOC: TAG = total − WORD (no LINE/SET field)

Worked: cache design (LOC23 Q5b, 10 M)

A 2-way set-associative cache uses blocks of 4 words and can hold 2048 words from a 128K × 32 main memory. Compute (i) all sizes (ii) TAG / SET / WORD bit-widths.

Cache lines = total words / block size = 2048 / 4 = 512 lines.
Sets (2-way) = 512 / 2 = 256 sets.
Address bits. Main memory has 128K = 217 words → 17-bit word address. Word size 32 bits = 4 bytes; if address is byte-addressable, multiply by 4 → 19 bits. The classical Mano problem is word-addressable, so we use 17 bits.
WORD = log₂(4) = 2 bits. SET = log₂(256) = 8 bits. TAG = 17 − 2 − 8 = 7 bits.
Cache memory size (data only) = 2048 × 32 = 65,536 bits = 8 KB. Including tag (7 bits × 512 lines = 3584 bits) and valid bit (512), the total SRAM is ≈ 8 KB + 512 bytes ≈ 8.5 KB.

512 lines · 256 sets · TAG = 7 b · SET = 8 b · WORD = 2 b · cache ≈ 8 KB.

4.7 Cache size vs block size

Two design parameters interact:

  • Cache size ↑ reduces capacity misses, but raises hit time (larger SRAM = slower) and silicon cost. Doubling cache typically improves hit rate by 1–2 percentage points after the working-set fits.
  • Block size ↑ exploits spatial locality (one miss brings in more useful data), but:
    • Raises miss penalty (more bytes to transfer).
    • Wastes cache space if locality is poor.
    • Eventually miss rate rises beyond a sweet-spot block size — see the classic "U-shaped" miss-rate vs block-size curve.
Sweet spot Modern CPUs use 64-byte cache lines (8 words on a 64-bit machine). This balances DRAM burst behaviour (which favours 64-byte transfers) with the diminishing-returns of spatial locality.

4.8 Write policies — write-through & write-back PYQ

Definition A write policy specifies what happens when the CPU writes to a cached location: should the write also propagate to main memory immediately, or be deferred?

Write-through

  • Every write hit updates both the cache and main memory immediately.
  • Pros: Memory always consistent. Simple coherence in multi-core. Easy crash recovery.
  • Cons: Heavy memory bus traffic — every store costs a memory write. Mitigated by a write buffer that holds pending writes.

Write-back

  • Writes update only the cache; a "dirty bit" marks the line. The line is flushed to main memory only when evicted.
  • Pros: Massive bandwidth saving — many writes coalesce. Faster.
  • Cons: Memory temporarily inconsistent (matters for DMA/multiprocessor — needs MESI). Complex eviction logic.

Write-miss policies

  • Write-allocate — on a write miss, load the line into cache first, then write. Pairs naturally with write-back.
  • No-write-allocate — on a write miss, write straight to main memory, skip cache. Pairs with write-through.

What are the benefits of studying computer architecture?

Computer architecture is the programmer-visible design layer of a CPU. Studying it gives four concrete benefits:

  2. Compiler & OS design — system software lives at the architecture interface.

Topic 1.1 recap · architecture vs organisation
  • Architecture = programmer-visible attributes — ISA, registers, addressing modes, data formats; the answer to "what does the machine do?"
  • Organisation = hardware-level realisation — bus widths, ALU layout, pipeline depth, cache hierarchy, control signals; the answer to "how is it built?"
  • Same architecture → many organisations: x86 ISA realised by Intel Core, AMD Ryzen with vastly different microarchitectures.

May 2024 · Unit I · 1.5 M · LOC_CSE24 Q1(a)
What do you mean by stored program control concept?

The stored-program control concept — proposed by von Neumann (1945) — states that both the program (instructions) and the data on which it operates are stored in the same read-write main memory. The CPU fetches each instruction sequentially by address held in the program counter, decodes it, and executes it. Because instructions are themselves binary patterns, the machine is reprogrammable just by loading a different program — the foundation of all general-purpose computers. See topic 1.2.

Topic 1.2 recap · von Neumann model
  • Proposed by John von Neumann (1945, EDVAC report) — single memory for both instructions and data.
  • Five defining properties: single memory · sequential execution by PC · instructions are binary patterns · fetch-decode-execute loop · reprogrammable hardware.
  • Single shared bus connects CPU · memory · I/O — the "von Neumann bottleneck" (one bus is the throughput limit).
  • Variant: Harvard architecture separates I and D caches (used in DSPs and modern L1) to break the bottleneck.
  • Every modern CPU (Intel, ARM, RISC-V) is a descendant.

Draw the flowchart explaining the process of non-restoring division algorithm.

Non-restoring division avoids the "restore-then-resubtract" wasted cycle by remembering the sign of the partial remainder. Each cycle does exactly one add OR subtract instead of subtract-then-maybe-restore. See full flowchart at topic 1.14, Fig 1.14b.

  • Initialise A = 0, M = divisor, Q = dividend, n = bit-width.
  • Loop: shift-left (A,Q); if sign(A) = +, A ← A − M else A ← A + M; Q0 ← sign(A); n−−.
  • Fix-up: if at end A < 0, perform A ← A + M.
Topic 1.14 recap · restoring vs non-restoring
  • Restoring: subtract divisor; if remainder negative, restore by adding back → up to 2 ops per cycle.
  • Non-restoring: subtract divisor; if negative, next step adds instead → exactly 1 add OR subtract per cycle.
  • Non-restoring is ~2× faster on average; only final fix-up required if last remainder negative.
  • Flowchart kernel: A=0, M=divisor, Q=dividend; loop = shift-left (A,Q) → if sign(A)=+ then A←A−M else A←A+M → Q0←sign(A) → n−−; fix-up at end.
  • Worked 4-bit example 7÷3: quotient=2, remainder=1 (verify 3·2+1=7).

Why are registers used in computers?

Five reasons: (1) Speed — register access is sub-nanosecond, vs 50-100 ns for main memory. (2) Reduce memory traffic — reused values stay cached in a register. (3) Hold control state — PC, IR, flags must be in registers for the CPU to access them every cycle. (4) Support addressing modes — register-indirect, indexed, base-register modes all need registers. (5) Enable compiler optimisation — register allocation is the single biggest compile-time win. See topic 1.4.

  • PC (Program Counter) — address of next instruction to fetch.
  • IR (Instruction Register) — instruction currently being decoded.
  • MAR / MBR — Memory Address Register drives address bus; Memory Buffer Register buffers data.
  • AC (Accumulator) — default ALU destination (single-accumulator machines).
  • SP (Stack Pointer) — top-of-stack; PSW = status flags + mode bits.
  • R0..Rn — general-purpose registers (8/16/32 typical); register-file access ≤ 1 ns vs 50-100 ns for DRAM.

What is bus arbitration?

Bus arbitration is the protocol that decides which of several bus-master devices (CPU, DMA, co-processor) gets ownership of a shared bus on a given cycle, preventing contention. Three schemes: daisy-chain (serial grant signal), centralised parallel (one arbiter, dedicated request lines), and distributed polling (each device runs its own arbitration). See topic 1.9.

Topic 1.9 recap · 3 arbitration schemes
  • Daisy-chain: grant flows serially through devices; first requester takes it. Few lines, simple, fixed priority; single fault breaks chain.
  • Centralised parallel: one arbiter with dedicated request lines per device; flexible priority, fast; arbiter is single point of failure.
  • Distributed polling: each device runs its own arbitration logic, ties broken by address. No central arbiter, scalable; complex per-device logic.
  • Common-bus via MUX: for k registers of n bits, use n k:1 MUXes (one per bit); 2 select lines for k=4.

Explain the concept of cache coherence.

Cache coherence is the guarantee that all caches in a multiprocessor see a consistent value for every shared memory location. If one core writes X = 7, every other core's subsequent read of X must return 7 (or an even more recent value), not the stale value its own cache may hold. Coherence is maintained by bus-snooping protocols (e.g. MESI) or directory-based protocols. See topic 3.7.

Topic 3.7 recap · MESI states

  • E · Exclusive: matches memory, held only here; first read miss with no other holder.
  • S · Shared: matches memory, may exist in other caches; read miss with another responder.
  • I · Invalid: empty/stale; triggered by snooping a remote write.
  • Two protocol families: bus-snooping (MESI) for small multi-core; directory-based for NUMA/large systems.

Explain the ripple-carry adder/subtractor using the circuit diagram.

An n-bit ripple-carry adder/subtractor reuses the same RCA for both operations via a control input M:

  • Each Bi XORs with M before entering FAi. When M = 1, every Bi is inverted.
  • C0 = M. When M = 1, this becomes the +1 needed to complete 2's-complement of B.
  • Hence A − B = A + (−B) = A + 1's-complement(B) + 1, all in one circuit.

Topic 1.13 recap · RCA equations

  • n-bit RCA delay: worst-case n · tFA (carry ripples through all stages).
  • Adder/subtractor: XOR each Bi with mode M, set C0=M; M=0 ⇒ A+B; M=1 ⇒ A−B = A + 1's-comp(B) + 1.
  • CLA: generate Gi=Ai·Bi, propagate Pi=Ai⊕Bi; computes all carries in O(log n).

  • ① Sender places data on bus and asserts DV (data-valid).

(1) Definition

A DMA controller is an auxiliary processor that, once initialised by the CPU (source address, destination, count, direction), transfers a block of data directly between memory and an I/O device — bypassing the CPU for each individual word.

  3. DMA asserts Bus-Request to CPU.
  4. CPU finishes current bus cycle, asserts Bus-Grant, tri-states its bus drivers.

(1) Definition

Cache memory is a small fast SRAM placed between CPU and main DRAM. It holds recently used data, exploiting temporal and spatial locality. Performance metric: hit ratio h; Tavg = h·Tc + (1−h)·Tm.

Larger cache (capacity), higher associativity (reduces conflict misses), larger block (spatial locality, but watch miss penalty), prefetching, victim cache, write-back to reduce bus traffic.

Explain interrupts in detail — classification, priority schemes, and role in process-state transitions.

(1) Definition

An interrupt is an asynchronous event that suspends the running program, transfers control to a predefined ISR (interrupt service routine), and on completion restores the saved context.

  • Hardware — external device on INT pin (timer, keyboard, disk).

(3) Priority schemes

  • Timer interrupt → Running → Ready (preemption).

  • DMA-complete hardware interrupt → Waiting → Ready.

Explain write-back and write-through policies in cache memory.

(1) Write-through

Every cache write also updates main memory immediately. Pros: always consistent, simple coherence. Cons: heavy memory bus traffic. Mitigated by a write buffer.

(2) Write-back

Writes update only cache; a dirty bit marks the line. Line is flushed to memory only on eviction. Pros: massive bandwidth saving, faster. Cons: memory temporarily stale, needs MESI in multi-core.

LIKELY Unit IV predicted 5 – 10 M
Explain memory interleaving (low-order & high-order). Show how 4-way interleaving accelerates a 16-word cache-line fill. Discuss bank-conflict edge cases.
Why predicted — signals 4 + 3 (4) Format rotation — appeared only as 1.5 M short (LOC_CSE24 Q1j), due to scale up. (3) High-yield syllabus topic ("memory interleaving" is in the very first line of Unit IV) but under-asked.
(1) Definition

Memory interleaving splits main memory into k banks; address bits select the bank. Bank accesses overlap in time → ~k× bandwidth.

(2) Low-order vs high-order

See full table at topic 4.3. Low-order = consecutive words different banks (best for sequential). High-order = each bank holds contiguous range (best for fault isolation, multiprogramming).

(3) 4-way example for 16-word fill

16 words / 4 banks = 4 cycles per bank. With 4-way low-order interleave, all 4 banks read in parallel → total time = 4 cycles instead of 16 → 4× speedup.

(4) Bank conflicts

If consecutive accesses target the same bank (e.g. stride-4 in 4-way interleave = always bank 0), no parallelism is achieved. This is a real problem for some matrix-multiply patterns; solved by skewed indexing or larger k.

Key takeaways · revise these 5
  • Interleaving splits main memory into k independent banks; address bits select the bank, overlapping bank accesses → ~k× bandwidth.
  • Low-order interleaving: low address bits = bank ID → consecutive words across different banks; best for sequential / streaming / cache-line fill.
  • High-order interleaving: high address bits = bank ID → each bank holds a contiguous range; best for fault isolation, multiprogramming.
  • 4-way example: 16-word cache-line fill = 4 cycles instead of 16 → 4× speedup, provided no bank conflicts.
  • Bank conflict: stride-k access pattern always hits one bank — kills parallelism; solved by skewed indexing or larger k.

Compare hardwired and microprogrammed control unit designs. For a hypothetical 8-instruction CPU, sketch both implementations and discuss when to choose each.
Why predicted — signals 3 + 5 (3) Microprogrammed CU is asked frequently, but the comparison with hardwired is overdue (last directly compared in earlier corpora). (5) Senior profs love the design-tradeoff angle.

See full topic at topic 2.1. For the design sketch:

Hardwired

For 8 instructions, build a 3-bit opcode decoder feeding into combinational logic: each control line is a Boolean function of (opcode, T-state, status flags). Total: ~30-50 gates per control signal × 20 signals = 600-1000 gates.

Microprogrammed

Build a 256-word control memory (each μ-instruction ~40 bits); CAR (8 bit) + CBR (40 bit) + MUX + opcode mapper. Total: ~10 KB ROM + few hundred gates of sequencer.

When to choose which

Small simple ISA (RISC) → hardwired (fast, area-efficient). Large complex ISA (CISC, microcontrollers with rich peripheral instructions) → microprogrammed (flexible, easy to patch).

Derive the speedup formula for a k-stage pipeline on n tasks. Compute speedup and efficiency for k=5, n=100. What is the bottleneck when n is small?
Why predicted — signal 5 Signature topic for the parallel-computing unit. Numerical pipeline speedup hasn't been asked at length yet; teachers like the derivation + numerical pair.

Full derivation at topic 3.2. For k=5, n=100:

  • Pipelined cycles = k + n − 1 = 104.
  • Sequential cycles = n · k = 500.
  • Speedup = 500 / 104 = 4.81 (~96% of ideal 5).
  • Efficiency = 4.81 / 5 = 96%.

Key takeaways · revise these 5
  • Pipelined cycles = k + n − 1 (k-stage fill + 1 task per cycle thereafter); sequential cycles = n · k.
  • Speedup Sk = (n·k) / (k + n − 1); efficiency E = Sk / k.
  • For k=5, n=100: S = 500/104 = 4.81 (≈96% of ideal 5); efficiency = 96%.
  • Throughput = n / [(k + n − 1) · τp] where τp is the slowest-stage clock period.
  • Bottleneck for small n: the (k−1)-cycle fill dominates → at n = k speedup is only k/2 (half ideal); pipelining wins only on long streams (n ≫ k).

Part I · Short-answer drill (10 × 1.5 M)

D1 · What is the difference between RISC and CISC?

RISC = small uniform instruction set, fixed-length, hardwired control, register-to-register ops (e.g. ARM, MIPS). CISC = large complex instruction set, variable length, microprogrammed control, memory-touching ops (e.g. x86). See topic 1.8.

D2 · Define cache coherence.

The property that every cache in a multiprocessor sees a consistent value for each memory location; if core A writes X=7, every subsequent read of X (from any core) must return 7 (or a more recent write). Maintained by MESI snooping or directory protocols. See topic 3.7.

D3 · List the steps of the instruction execution cycle.

Fetch (MAR←PC; MBR←M[MAR]; IR←MBR; PC←PC+1) → Decode (CU interprets opcode) → optional Indirect → Execute (compute EA; perform op; update PSW) → optional Interrupt service. See topic 1.6.

D4 · What is bus arbitration? Name three schemes.

The protocol that grants a shared bus to one of several requesting masters. Three schemes: daisy-chain (serial grant), centralised parallel encoder, distributed polling. See topic 1.9.

D5 · State the IEEE-754 single-precision layout.

32 bits: 1 sign · 8 exponent (bias 127) · 23 mantissa with implicit leading 1. Value = (−1)S · 1.M · 2E−127. Special: E=0,M=0 → ±0; E=0,M≠0 → denormal; E=255,M=0 → ±∞; E=255,M≠0 → NaN. See topic 1.12.

D6 · Why does DMA have priority over the CPU?

Peripheral data is volatile — if a sector/packet word isn't latched before the next rotational/clocking event, it's lost. CPU can wait an extra cycle without harm. Hence the arbitration rule: deadline-strict peripherals beat deadline-flexible CPU. See topic 2.9.

D7 · What is locality of reference?

The principle that memory accesses cluster: temporally (recent access likely to repeat soon) and spatially (addresses near a recent access likely to be touched). The locality principle justifies caching — it ensures 95-99% hit rate at the small top tier of the memory hierarchy. See topic 4.4.

D8 · Distinguish write-back and write-through.

Write-through: every write also updates main memory immediately — consistent but bus-heavy. Write-back: writes update only cache + dirty-bit; line flushed on eviction — bandwidth-efficient but needs coherence protocol. Modern L1-L3 are write-back. See topic 4.8.

D9 · What are pipeline hazards? Name the three types.

A hazard is a situation that stalls the next instruction from entering its designated stage. Three types: structural (resource contention), data (RAW/WAR/WAW), control (branch redirect). See topic 3.3.

D10 · How do you compute speedup of a k-stage pipeline on n tasks?

Sk = (n·k) / (k+n−1). As n → ∞, Sk → k. Efficiency = Sk/k = n/(k+n−1). See topic 3.2.

D11 · [Unit I · 5 M] Write a short note on ASCII and Unicode character representations.

ASCII = 7-bit code (128 chars) extended to 8-bit (256 chars). Two categories: printable 0x20–0x7E (letters, digits, punctuation) and control 0x00–0x1F + 0x7F (LF, CR, BEL, ESC). Examples: 'A' = 0x41 = 65, 'a' = 0x61 = 97, '0' = 0x30 = 48; upper↔lower differ by bit 5 (XOR 0x20). Unicode supersedes ASCII with > 149,000 code-points across all scripts; UTF-8 encodes Unicode in 1–4 bytes and is byte-identical to ASCII for the first 128 code-points. Why LOC cares: byte-level memory representation, string storage, and I/O stream framing (LF / CR). See topic 1.12b.

D12 · [Unit I · 5 M] Explain the carry-save multiplier with a diagram.

A carry-save multiplier reduces n partial products to two vectors (sum S, carry C) using a tree of CSAs, then a final CPA sums them. Each CSA cell is a (3:2) compressor: three input bits in one column give a sum bit (stays at this column) and a carry bit (shifts one column left into the next layer). Because carry never propagates inside a layer, every layer is one full-adder delay regardless of operand width — n partial products collapse in O(log n) levels instead of O(n). Final CPA does the single end-to-end carry propagation. Used in fast multipliers (Wallace, Dadda), DSP MAC units, pipelined FP multipliers. See topic 1.15, Fig 1.15.
