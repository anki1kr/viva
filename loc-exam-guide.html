    IEEE-754 Floating Point & Signed multiplication

    IEEE-754 Single Precision (32-bit) Bit layout: 1-bit Sign (S) · 8-bit Biased Exponent (E) (Bias=127) · 23-bit Mantissa (M). Formula: Value = (-1)ˢ × 1.M × 2ᴱ⁻¹²⁷.
    Booth's Signed Multiplier Mnemonic Recode multiplier bits $Q_i, Q_{i-1}$ as follows:
    • 00 or 11: Do nothing, shift arithmetic right.
    • 01: Add multiplicand ($A \leftarrow A + M$), shift arithmetic right.
    • 10: Subtract multiplicand ($A \leftarrow A - M$), shift arithmetic right.

    Hardwired vs Microprogrammed Control Units ⭐ Repeated 3×

    Microprogrammed Address Sequencing The Control Address Register (CAR) stores the address of the next microinstruction. The 4 next-address sources are: (1) Increment (CAR + 1), (2) Opcode Mapping (determines start address of instruction execution microroutine), (3) Unconditional Branch, (4) Status-conditional Branch.
    FeatureHardwired Control UnitMicroprogrammed Control Unit
    SpeedExtremely fast (implemented entirely in gate-logic silicon).Slower (requires a memory read access to Control ROM).
    FlexibilityRigid. Modifying commands requires redesigning the logic board.Flexible. Updates require changing code in Control Memory.
    ApplicationUsed in RISC processors where speed is critical.Used in complex CISC systems with massive instruction sets.

    DMA Controllers & I/O Transfer Modes ⭐ Guaranteed 10M

    DMA (Direct Memory Access) transfers data directly between high-speed peripherals and main memory without passing through the CPU, saving clock cycles.

    Why DMA gets Priority The CPU is patient because its instructions can wait. High-speed I/O peripherals are **volatile** and impatient — if incoming data blocks are not read instantly, they will overwrite and be lost. Thus, DMA holds higher bus priority.
    Unit III

    03Pipelining &
    Parallel Processors

    Throughput calculations, hazard resolution, and multicore parallel execution classifications.

    Pipeline Hazards & Flynn's Classification ⭐ Repeated 3×

    Flynn's Parallel Taxonomy Classifies parallel computers by Instruction streams vs Data streams: (1) SISD: Uni-core PC, (2) SIMD: Vector/GPU graphics processor, (3) MISD: Redundant spacecraft systems (rare), (4) MIMD: Modern multicore processors.

    The Three Pipeline Hazards

    • Structural Hazard: Hardware conflicts. E.g., when instruction-fetch and data-memory access try to use a single-port memory block at the same time. Resolved by splitting into Instruction and Data caches (Harvard architecture).
    • Data Hazard: Operands not ready. An instruction depends on a previous instruction's result that has not yet completed the write-back stage (RAW, WAR, WAW). Resolved by **forwarding** or stalling (bubbles).
    • Control Hazard: Branch delays. Decisions on jumps affect which instructions to fetch next. Resolved by branch prediction or branch target buffers.
    Unit IV

    04Memory Organisation,
    Cache & Paging

    Sizing cache mapping fields and tracking page replacement algorithms.

    Set-Associative Cache Mapping ⭐ Guaranteed 10M

    A physical address is partitioned into distinct fields to query cache rows:

    Set-Associative Address Fields
    • WORD = log₂ (Block Size in Bytes)
    • SET = log₂ (Total Cache Size / (Block Size × Set Size))
    • TAG = Address Width − SET − WORD
    Direct Mapping vs Associative • Direct Mapping has a dedicated LINE field; a block can map to exactly one cache line, causing thrashing.
    • Set-Associative allows a block to reside in any line within the target SET, balancing search logic complexity with higher hit rates.

    Page Replacement Tracing

    The Algorithms • FIFO (First-In-First-Out): Evicts oldest page. Suffers from Belady's Anomaly (faults can rise with frame count).
    • LRU (Least-Recently-Used): Evicts page untouched for longest time. Most realistic.
    • Optimal: Evicts page used farthest in future. Provides theoretical lower bound baseline.

    Section A (Compulsory · 1.5M Each)
    Q1(a)1.5M
    Define a Program Status Word (PSW). What are its primary flags?
    The PSW is a dedicated CPU register storing status bits (flags) updated by ALU operations. Primary flags in x86 include: Carry Flag (CF), Zero Flag (ZF), Sign Flag (SF), Overflow Flag (OF), and Parity Flag (PF).
    Q1(b)1.5M
    Explain the Stored-Program concept.
    Proposed by John von Neumann in 1945, the Stored-Program concept specifies that program instructions and operand data are stored together in the same read-write memory address space, allowing instructions to be manipulated as data.
    Q1(c)1.5M
    Differentiate between Direct and Indirect addressing modes.
    In Direct Mode, the instruction address field directly contains the Effective Address of the operand (EA = A). In Indirect Mode, the address field points to a memory location containing the EA of the operand (EA = M[A]).
    Q1(d)1.5M
    Why does a DMA controller have priority over the CPU for bus control?
    I/O peripheral data is highly **volatile**. If incoming high-speed peripheral data is not read instantly from the channel, it will overwrite and be lost. The CPU's execution, however, can be paused without data loss.
    Q1(e)1.5M
    List the three primary pipeline hazards with a single sentence each.
    (1) Structural Hazard: Hardware conflicts (e.g. concurrent instruction-fetch and data-access). (2) Data Hazard: Instructions attempting to read data before a previous write completes. (3) Control Hazard: Delays caused by jump and branch decisions.
    Q1(f)1.5M
    Define spatial and temporal locality of reference.
    Temporal Locality: A memory location accessed once is highly likely to be accessed again in the near future. Spatial Locality: Memory locations physically adjacent to a recently accessed address are highly likely to be accessed next.
    Q1(g)1.5M
    Write the formula for pipeline speedup S_p and define its variables.
    Formula: S_p = (n × k) / (k + n − 1), where n is the total number of instructions, and k is the total number of stages in the pipeline.
    Q1(h)1.5M
    Explain the difference between write-through and write-back cache policies.
    In **Write-Through**, cache and main memory are updated simultaneously on writes. In **Write-Back**, updates occur only in the cache; the main memory block is updated only when that cache line is evicted (dirty bit tracking).
    Q1(i)1.5M
    Convert decimal (19.625)₁₀ to binary.
    Integer: 19 = 16 + 2 + 1 = 10011. Fraction: 0.625 = 0.5 + 0.125 = 2⁻¹ + 2⁻³ = .101.
    Result: 10011.101₂.
    Q1(j)1.5M
    What is the function of the Instruction Register (IR) and Program Counter (PC)?
    The **Program Counter (PC)** holds the memory address of the next instruction to be fetched. The **Instruction Register (IR)** holds the actual opcode of the instruction currently being decoded and executed.

    Q4Unit 2 Highlight15M
    Describe Microprogrammed Control Address Sequencing with a neat block diagram. Trace the CAR (Control Address Register) or μPC values through the execution cycle of an ADD M microroutine.
    Introduction

    In a microprogrammed control unit, control signals are generated by reading microinstructions from a dedicated Control Memory (ROM). The Control Address Register (CAR) stores the address of the microinstruction currently being executed.

    Address Sequencing & Mappings

    The control unit determines the next CAR value using 4 distinct paths:
    (1) Increment: CAR + 1. (2) Opcode Map: decodes the instruction register opcode to jump to the instruction's dedicated microroutine start address. (3) Unconditional Branch: absolute address. (4) Conditional Branch: jumps based on status flags.

    CAR Trace for ADD M Microroutine

    Assume FETCH starts at Control Memory address 0, and the ADD instruction opcode maps to address 42. The instruction adds memory value to the AC register:

    0
    FETCH 1: MAR ← PC. Next address = increment. CAR updates: CAR = 1.
    1
    FETCH 2: MBR ← M[MAR]; PC ← PC + 1. Next = opcode map. CAR updates: CAR = 42.
    42
    ADD 1: MAR ← MBR(Address). Next = increment. CAR updates: CAR = 43.
    43
    ADD 2: MBR ← M[MAR]. Next = increment. CAR updates: CAR = 44.
    44
    ADD 3: AC ← AC + MBR; Update Flags. Next = Branch to FETCH. CAR updates: CAR = 0.

    This trace demonstrates the precise coordination of register transfer logic through all four sequencing operations.

    DMA Controller Block Diagram
    The Three DMA Transfer Modes
    • 1. Burst Mode: The DMA controller gains absolute control of the system bus and transfers the entire data block continuously. The CPU is completely starved of the bus and sits idle. Used for high-speed disk writes.
    • 2. Cycle-Stealing Mode: The DMA steals a single bus clock cycle from the CPU to transfer a single word, then immediately releases the bus back. This interleaves transfer with CPU instruction execution, minimizing CPU starvation.
    • 3. Transparent Mode: The DMA controller transfers data only during CPU cycles when the processor is executing internal instructions (e.g. Decode) and does not require the external system bus. Has zero impact on CPU performance.

    Q6Unit 3 Highlight15M
    Describe Flynn's Classification of parallel computer architectures with a neat diagram. Discuss the three pipeline hazards (structural, data, control) and how each is resolved.
    Flynn's Parallel Classification

    Flynn's taxonomy classifies computer architectures based on the number of concurrent **Instruction Streams** and **Data Streams** processed:

    Data \ InstructionSingle Instruction (SI)Multiple Instruction (MI)
    Single Data (SD)SISD (Standard Uni-core PC)MISD (Spacecraft fault-tolerance, rare)
    Multiple Data (MD)SIMD (GPUs, vector array processors)MIMD (Modern multi-core processors)
    Pipeline Hazards & Resolutions
    • 1. Structural Hazard: Occurs when two pipeline stages attempt to access the same physical hardware resource simultaneously.
      Example: Instruction Fetch and Data Memory access both querying a single-port memory block.
      Resolution: Implementing separate Instruction and Data caches (Harvard Architecture) or inserting a pipeline bubble.
    • 2. Data Hazard: Occurs when an instruction depends on the result of a previous instruction that has not yet completed execution and write-back.
      Types: Read-After-Write (RAW, most common), Write-After-Read (WAR), Write-After-Write (WAW).
      Resolution: Data Forwarding (bypassing the register write-back stage and routing the ALU output directly to the ALU input for the next cycle), or stalling via compiler-inserted NOP instructions.
    • 3. Control Hazard: Occurs when branch or jump instructions modify the Program Counter (PC), making fetched instructions in the pipeline invalid.
      Resolution: Branch Prediction (predicting whether a branch is taken or not), using Branch Target Buffers (BTB), or employing Delayed Branching (filling delay slots with useful instructions).

    (a) Cache Bits Computation
    Step 1
    Compute WORD bits: Block size = 16 bytes. WORD = log₂ (16) = 4 bits.
    Step 2
    Compute Cache lines: Total cache lines = Cache size / Block size = 64 KB / 16 B = 65,536 / 16 = 4096 lines.
    Step 3
    Compute SET bits: Since it is 2-way set-associative, Set count = Total lines / Set size = 4096 / 2 = 2048 sets. SET = log₂ (2048) = 11 bits.

    Compute TAG bits: Physical address width = 32 bits. TAG = 32 − SET − WORD = 32 − 11 − 4 = 17 bits.
