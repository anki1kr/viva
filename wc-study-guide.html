01 Wireless Networking UNIT IV

1.4 Traffic Routing in Wireless Network PYQ 5(b) — 7 marks

Definition Traffic routing is the process of selecting a path for a call/data packet from the calling mobile, through the wireless infrastructure, to the destination — possibly across the PSTN.

Steps in routing a mobile call

  1. Call origination: MS sends a request on the control channel to the BTS.
  2. Authentication & registration: MSC checks HLR/VLR — is the user valid, where is the called party?
  3. Destination lookup: If the called number is mobile, the GMSC queries the called user's HLR; HLR returns a routing number from the current VLR.
  4. Channel assignment: A free traffic channel is assigned in the cell.
  5. Path setup: A circuit is created MS → BTS → BSC → MSC → (PSTN or other MSC) → called party.
  6. Conversation: Voice/data flows on the assigned channel.
  7. Handover routing: If the user moves, traffic is re-routed to the new BTS.
  8. Release: Channel freed when call ends.

Routing Strategies

StrategyHow it works
Fixed routingSame predetermined path every call (used in legacy PSTN)
Dynamic routingPath chosen based on real-time traffic / congestion
Alternate routingTry primary path; if busy, fall back to alternates
Hierarchical routingLocal → tandem → toll switches in tiers

1.5 Common Channel Signaling (CCS / SS7)

Definition Common Channel Signaling separates the signaling (call setup messages) from the voice channel — all signaling is sent on a dedicated network (SS7). This is faster, supports more services, and prevents fraud.
  • CAS (Channel Associated Signaling): signaling carried in-band with each voice circuit (legacy).
  • CCS / SS7: signaling on a separate packet network. Used in GSM, ISDN, modern PSTN.
  • Enables: caller-ID, call forwarding, roaming, SMS, prepaid billing.

1.6 Wireless Data Services

  • SMS – short text up to 160 chars over signaling channel.
  • MMS – multimedia (image/audio/video).
  • GPRS / EDGE – packet data on 2.5G.
  • WAP – simplified Web for old mobiles.
  • VoLTE – voice over LTE packet data.

1.7 ISDN — Integrated Services Digital Network PYQ 1(i) — 1.5 marks

Definition ISDN is a set of standards for the simultaneous digital transmission of voice, video, data, and signaling over the traditional telephone network using digital circuits end-to-end.

ISDN Channels

ChannelSpeedPurpose
B (Bearer)64 kbpsCarries user voice/data
D (Delta)16 / 64 kbpsSignaling & control
H384 kbps – 1.92 MbpsHigh-speed bulk data (H0,H11,H12)

ISDN Interfaces

  • BRI (Basic Rate Interface): 2B + 1D = 144 kbps — for homes & small offices.
  • PRI (Primary Rate Interface): 23B + 1D (US, 1.544 Mbps) or 30B + 1D (Europe, 2.048 Mbps) — for businesses.

1.8 Advanced Intelligent Networks (AIN)

Definition AIN is an architecture where the call-handling logic (intelligence) is moved out of the switching nodes into a separate Service Control Point (SCP). New services can be deployed without upgrading every switch.

Components: SSP (Service Switching Point), STP (Signal Transfer Point), SCP (Service Control Point), IP (Intelligent Peripheral). Enables 800-numbers, prepaid mobile, virtual private numbers, calling-card services.

Quick Quiz — Unit IV

Q1. What does the D channel of ISDN carry?
Signaling and control information (16 kbps in BRI, 64 kbps in PRI).
Q2. Two ways routing in wireless differs from fixed routing.
(1) Path may change mid-call due to handover. (2) HLR/VLR lookups locate a moving subscriber, not a fixed phone number.
Q3. BRI total bandwidth?
2 × 64 + 16 = 144 kbps.

02 Multiple Access Techniques UNIT III

Multiple-access techniques let many users share the limited radio spectrum. The four core schemes — FDMA, TDMA, CDMA, SDMA — and Spread Spectrum (FHSS / DSSS) appear almost every year.

2.1 What is Multiple Access?

Definition Multiple access means dividing a shared communication resource (frequency band) among many users simultaneously, without their signals interfering with each other.

The three basic resources we can divide are: Frequency, Time, Code — giving FDMA, TDMA, CDMA. A fourth uses Space → SDMA.

2.2 FDMA — Frequency Division Multiple Access

Definition FDMA divides the total available bandwidth into many narrow non-overlapping frequency bands; each user gets one band for the entire duration of the call.
FDMA — Frequency Division Multiple Access
Each user owns a narrow sub-band continuously for the entire call

Used in: 1G AMPS, satellite, FM radio, cable TV.

Pros: simple hardware, no synchronisation needed.
Cons: guard bands waste spectrum; poor flexibility; idle channels can't help busy ones.

2.3 TDMA — Time Division Multiple Access

Definition TDMA gives each user the entire frequency band but only for a short time slot in a repeating frame. Slots cycle through users so every user gets a regular turn.
TDMA — Time Division Multiple Access
Single frequency, time slots cycle U1 → U4 → U1 → U4 …

Used in: GSM (8 slots/carrier), DECT, 2G PDC.

Pros: efficient spectrum use, supports variable bit-rate per user.
Cons: needs strict time synchronisation; sensitive to multipath and the near-far effect.

2.4 Near/Far Effect in TDMA PYQ 1(h) — 1.5 marks

Definition The near-far effect happens when a transmitter close to the receiver overpowers a far-away transmitter on adjacent slots/codes, causing the weaker signal to be lost.

In TDMA each slot is on the same frequency, so a strong signal in one slot can leak into adjacent slots if timing is slightly off. Mitigation: tight power control (each MS adjusts power so the BTS receives all users at the same level) and accurate timing advance.

Where it hurts mostCDMA suffers worse than TDMA because all CDMA users share frequency and time; in TDMA only timing/leakage between slots matters.

2.5 CDMA — Code Division Multiple Access PYQ 2(a) — 10 marks

Definition CDMA lets all users transmit at the same frequency at the same time, but each user's data is multiplied by a unique spreading code (PN sequence). The receiver multiplies the received mix by the same code to recover that user's data; other users appear as background noise.
CDMA — Codes Sum in Air, RX Recovers with Same Code
Other users appear as background noise — orthogonal codes cancel them out

Properties

  • Spread bandwidth (much wider than data bandwidth).
  • Soft capacity: more users → more noise, no hard cut-off.
  • Universal frequency reuse (N = 1).
  • Soft handoff (talk to two BTS at once).
  • Resistant to jamming, multipath, eavesdropping.
  • Strong power control mandatory to handle near-far.

Areas where CDMA is used

  • 2G mobile: IS-95 (cdmaOne).
  • 3G: CDMA2000, WCDMA / UMTS.
  • GPS satellites (each satellite has its own PN code).
  • Military comms (anti-jamming).
  • Wi-Fi 802.11b uses DSSS (a form of CDMA).
  • Bluetooth uses FHSS — also a spread-spectrum technique.

2.6 SDMA — Space Division Multiple Access

Definition SDMA reuses the same frequency in different geographical areas / antenna beams. Cellular frequency reuse and smart-antenna beamforming are SDMA examples.

2.7 Packet Radio

Definition In packet radio, users transmit small data packets when they have data — there is no fixed slot or channel allocation. Examples: ALOHA, slotted ALOHA, CSMA/CA (Wi-Fi), GPRS.

Pure ALOHA efficiency = 18%. Slotted ALOHA = 36%.

2.8 Spread Spectrum PYQ 6(a) — 10 marks

Definition Spread spectrum intentionally spreads a narrow-band signal across a much wider bandwidth. The signal looks like low-power noise to anyone who doesn't know the spreading pattern.

Need / Why we spread

  • Anti-jamming: a narrow-band jammer can't kill a wide signal.
  • Anti-interception / privacy: looks like noise without the code.
  • Multiple access: different codes = different users (CDMA).
  • Multipath resistance: reflected copies arrive de-correlated.
  • Low power spectral density: co-exists with narrowband systems.

Types of Spread Spectrum

TypeHow it spreadsExample
FHSS — Frequency HoppingCarrier hops between many frequencies on a fast pseudo-random patternBluetooth, military
DSSS — Direct SequenceEach data bit is XORed with a fast PN chip sequencecdmaOne, Wi-Fi 802.11b, GPS

2.9 FHSS — Frequency Hopping Spread Spectrum (Detail) PYQ 6(a)

Definition In FHSS the carrier frequency hops over a set of channels in a pseudo-random pattern known to both transmitter and receiver. Each "hop" lasts a few ms (slow FH) or less than one bit (fast FH).
FHSS — Carrier Hops Pseudo-Randomly Across Channels
Each block = one hop dwell; sequence is known to TX and RX only

  • Strong jamming resistance — jammer must cover whole hop set.
  • Privacy — sequence acts as a key.
  • Co-existence with other systems on the same band.
  • Less affected by narrow-band fading.

2.10 Capacity of a Cellular System

Capacity C = M · k · Nc   users
where M = number of cells, k = channels per cell, Nc = number of clusters

For Frequency Reuse with cluster size N and total channels S: k = S / N (channels per cell). Smaller N → higher capacity but more co-channel interference.

Quick Quiz — Unit III

Q1. Which scheme has a soft capacity limit?
CDMA — adding more users only raises the noise floor, no hard cut.
Q2. Why does FHSS resist narrow-band jamming?
A narrowband jammer covers only one of many hopping channels, so most hops are unaffected.
Q3. The near-far problem is worst in which access scheme?
CDMA. All users share the frequency, so a strong nearby user drowns weak distant users unless tight power control is applied.

7.2 4-Mark Mini Answers (half-of-8-mark slots)

Each card = 1-line definition + 3–4 bullets + tiny example. Perfect when an 8-mark question splits as 4+4. 10 cards, all from Unit III/IV.

M1 · FDMA — definition + working [Unit III]

FDMA divides total bandwidth into narrow non-overlapping bands; each user owns one band for the entire call duration.

  • Used in 1G AMPS, FM radio, satellite uplinks.
  • Needs guard bands between channels — wastes spectrum.
  • No time synchronisation needed across users.
  • Cannot share idle bands with busy users (low utilisation).
M2 · TDMA — definition + working [Unit III]

TDMA gives every user the entire band but only for a short repeating time slot in a frame.

  • Used in GSM (8 slots per 200 kHz carrier) and DECT.
  • Requires strict timing advance at the BTS.
  • Supports variable bit-rate by allocating multiple slots.
  • Susceptible to multipath delay spread → equaliser needed.
M3 · CDMA — definition + working [Unit III]

CDMA gives every user the same band and the same time slot, separated only by a unique orthogonal PN/Walsh code.

  • Universal frequency reuse (N = 1).
  • Soft capacity — adding users raises noise, no hard cut-off.
  • Soft handoff — talks to two BTS at once.
  • Power control mandatory to fight near-far problem.
M4 · SDMA — definition + working [Unit III]

SDMA reuses the same frequency / time / code in different spatial regions using directional antennas, sectoring or beam-forming.

  • Examples: 3-sector cell, satellite spot beams, 5G massive MIMO.
  • Increases capacity without extra spectrum.
  • Combined with FDMA/TDMA/CDMA in real systems.
  • Requires antenna arrays and beam tracking.
M5 · Spread Spectrum — need [Unit III]

Spread spectrum stretches a narrow signal over a much wider band using a code known to both TX and RX.

  • Anti-jamming — narrow jammer can't kill a wide signal.
  • Privacy / LPI — looks like background noise.
  • Multiple access via codes (CDMA).
  • Multipath fading resistance via decorrelation.
M6 · FHSS vs DSSS [Unit III]

Two main spread-spectrum techniques.

  • FHSS — carrier hops over many frequencies on a PN pattern (Bluetooth, military).
  • DSSS — each data bit XORed with high-rate PN chips (IS-95, Wi-Fi 802.11b, GPS).
  • FHSS resists narrowband jammers; DSSS gives higher processing gain.
  • FHSS easier to sync; DSSS gives better data rate.
M7 · ISDN — channels & interfaces [Unit IV]

ISDN carries voice + data + signaling end-to-end over digital telephone lines.

  • B channel = 64 kbps user data.
  • D channel = 16 / 64 kbps out-of-band signaling.
  • BRI = 2B + 1D = 144 kbps (home).
  • PRI = 23B+1D (1.544 Mbps US) or 30B+1D (2.048 Mbps EU).
M8 · WAN vs WLAN [Unit IV]

WAN spans countries; WLAN covers a building / campus.

  • WAN: MPLS, LTE, leased lines; carrier-owned.
  • WLAN: IEEE 802.11 Wi-Fi; user-owned APs.
  • WAN — low per-user speed, high cost.
  • WLAN — high speed, low cost, limited range (≈ 100 m).
M9 · Common Channel Signaling (SS7) [Unit IV]

CCS / SS7 sends call-control signaling on a separate dedicated channel — not in the voice path.

  • Faster call setup (≈ 1–2 sec) than in-band MF tones.
  • Enables roaming, SMS, IN services, number portability.
  • Used between MSCs, HLR/VLR queries, mobility management.
  • Layered protocol: MTP1/2/3 → SCCP → TCAP / ISUP / MAP.
M10 · Traffic Routing in Wireless Network [Unit IV]

How the network finds and reaches a moving subscriber.

  • Caller dials → GMSC queries called party's HLR.
  • HLR returns the address of the current VLR/MSC.
  • GMSC routes the call to that MSC; MSC pages the MS via BTS.
  • MS responds → traffic channel assigned → call connects.

Q2 · Explain CDMA — concept, properties and areas of use [Unit III]

Definition CDMA (Code Division Multiple Access) is a spread-spectrum scheme where every user transmits on the same frequency at the same time, but each user's bits are multiplied by a unique orthogonal pseudo-noise (PN) code. The receiver despreads with the same code to recover the data; other users appear as noise.

How it works: data bit (1 or −1) is multiplied with a high-rate PN sequence (chips). Bandwidth becomes Rchip (much wider than data rate). The mixed wideband signal is transmitted. At the receiver, multiplying by the same PN code (correlation) recovers the original bit; other users' codes are orthogonal and integrate to ≈ 0.

// Walsh-code example (2 users)
user1: data = +1, code = [+1 +1 +1 +1] -> tx = [+1 +1 +1 +1]
user2: data = +1, code = [+1 -1 +1 -1] -> tx = [+1 -1 +1 -1]
sum    on air      = [+2  0 +2  0]

// Receiver despreads with code1
rx · code1 = [+2 +0 +2 +0] · [+1 +1 +1 +1] = +4 -> bit = +1  (user1 data recovered)
rx · code2 = [+2 +0 +2 +0] · [+1 -1 +1 -1] = +4 -> bit = +1  (user2 data recovered)
Properties
  • Universal frequency reuse (N = 1) — all cells share the same band.
  • Soft capacity — adding users raises noise floor; no hard cut-off.
  • Soft handoff — talk to two BTS at once (make-before-break).
  • Resistant to multipath, jamming and eavesdropping.
  • Strong power control mandatory (near-far problem).
  • Wide bandwidth ≈ 1.25 MHz (IS-95) or 5 MHz (WCDMA).
Areas of use
  • 2G mobile — IS-95 / cdmaOne.
  • 3G — CDMA2000, WCDMA / UMTS.
  • GPS — every satellite has its own PN code.
  • Wi-Fi 802.11b — DSSS form of CDMA.
  • Military communications — anti-jam, low intercept probability.

  • Privacy / LPI — looks like noise without the code.

  • Low spectral density — co-exists with narrowband systems.

FHSS — detail

In Frequency Hopping Spread Spectrum, the carrier frequency hops over a set of channels in a pseudo-random pattern known to both TX and RX. Each "hop" lasts a few ms (slow FH) or less than one bit (fast FH).

// FHSS transmitter pseudo-flow
for each hop_interval in session:
    f = pn_sequence.next()           // pick channel
    carrier = generate(f)
    tx_signal = modulate(carrier, data_chunk)
    transmit(tx_signal)

// Receiver — must use SAME PN sequence
for each hop_interval:
    f = pn_sequence.next()
    rx_data = demod(receive(), generate(f))

Bluetooth FHSS: 79 channels × 1 MHz in the 2.4 GHz ISM band, hopping at 1600 hops/s.

Advantage / disadvantage + Robust against narrowband interference and jamming, low intercept probability, supports many users via different hop sequences.
− Synchronisation between TX and RX is critical, requires fast frequency synthesizer, lower data rate per channel than DSSS.

Q4 · Explain Traffic Routing in a Wireless Network with a diagram [Unit IV]

Definition Traffic routing in a wireless network is the procedure by which the network locates a moving subscriber and delivers an incoming call to the cell currently serving that subscriber. It relies on databases (HLR, VLR) and signaling (SS7/MAP) — not on a fixed wire.
Mobile-Terminated Call Flow — Traffic Routing
8-step Mobile-Terminated call: caller → GMSC → HLR (locate) → visited MSC → BTS → MS, then voice path back through
Step-by-step
  1. Caller dials the MS number (MSISDN); call enters the network at the GMSC.
  2. GMSC queries the called subscriber's HLR with the MSISDN.
  3. HLR looks up the current VLR address and asks it for an MSRN (Mobile Station Roaming Number) — a temporary routing number — and returns it to GMSC.
  4. GMSC routes the call to the visited MSC using the MSRN.
  5. Visited MSC consults its VLR to find the cell-area, then sends a paging message to all BTSs in that area.
  6. BTSs broadcast page on the air interface; the called MS responds.
  7. MSC assigns a traffic channel; authentication and ciphering complete.
  8. Voice path is established: caller ↔ GMSC ↔ MSC ↔ BTS ↔ MS.
Why this design HLR-VLR split keeps subscriber data near the user (low latency) while still globally reachable. SS7/MAP signaling carries all the queries on a separate channel from the voice — that is Common Channel Signaling.

Q5 · Explain ISDN — services, channels and reference architecture [Unit IV]

Definition ISDN (Integrated Services Digital Network) is a set of CCITT/ITU standards (1980s) that delivers voice, data and signaling end-to-end in digital form over the existing copper telephone loop, replacing the analog local subscriber line with a digital one.
Services
  • Bearer services — voice, data, fax, video at 64 kbps multiples.
  • Tele-services — telephony, telex, teletex, videotex, fax G3/G4.
  • Supplementary services — caller ID, call forwarding, conference, hold.
Channels
ChannelBandwidthPurpose
B64 kbpsUser data — voice, data, video (one call each).
D16 kbps (BRI) / 64 kbps (PRI)Out-of-band signaling (call setup, control). Can carry low-speed packet data.
H384 kbps – 1.92 MbpsHigh-speed bulk data (H0, H11, H12).
Interfaces
  • BRI (Basic Rate Interface): 2B + 1D = 2(64) + 16 = 144 kbps — for homes and small offices.
  • PRI (Primary Rate Interface):
    • US/Japan T1 — 23B + 1D = 23(64) + 64 = 1.544 Mbps.
    • Europe E1 — 30B + 1D = 30(64) + 64 = 2.048 Mbps.
ISDN Reference Architecture (Subscriber Side)
Reference points R, S, T, U separate ISDN equipment (TE/TA/NT2/NT1) and the loop to the local exchange
Numerical example A small office uses BRI: 2 × 64 + 16 = 144 kbps. A medium office on E1 PRI: 30 × 64 + 64 = 2.048 Mbps — 14× more capacity per loop.

Q6 · Differentiate WAN and WLAN — characteristics, advantages, applications [Unit IV]

Definition A WAN (Wide Area Network) spans large geographical areas — cities, countries, continents — using carrier infrastructure (leased lines, MPLS, cellular core, satellites). A WLAN (Wireless LAN) covers a small area (a building, a campus) using the unlicensed ISM band and the IEEE 802.11 standard.

Advantages of WLAN over WAN
  • Much higher per-user bandwidth (Wi-Fi 6 ≈ 9.6 Gbps vs typical WAN tens of Mbps).
  • No carrier fees — uses unlicensed spectrum.
  • Quick to deploy and reconfigure; supports many concurrent devices.
Where WAN beats WLAN
  • Geographic reach — only WAN connects offices across cities.
  • Guaranteed SLA / QoS via carrier contracts.
  • Wide-area mobility (e.g. cellular WAN follows you; Wi-Fi does not).
Practical pairing A modern office combines both — internal users connect over WLAN for high-speed local sharing, and the building's gateway uses a WAN (leased line / MPLS) to reach head office and the internet.

Final Sessional Drill — 30 marks worth (Unit III & IV)

Q1 (1m). Expand SDMA and give one example.
Space Division Multiple Access — reusing the same band/time/code in different spatial regions. Example: 3-sector cell or 5G massive-MIMO beamforming.
Q2 (1m). What is the chip rate vs bit rate ratio called?
Processing gain, Gp = Rchip / Rbit. Higher Gp → better jamming resistance and despreading gain.
Q3 (1m). BRI bandwidth in kbps?
2B + 1D = 2(64) + 16 = 144 kbps.
Q4 (1m). Why does CDMA need power control?
To fight the near-far problem — without it a nearby user's signal would drown distant users sharing the same band.
Q5 (4m). Differentiate FHSS and DSSS.
FHSS hops the carrier across many narrow channels on a PN pattern (used in Bluetooth, military). DSSS XORs each data bit with a high-rate PN chip sequence (used in IS-95, Wi-Fi 802.11b, GPS). FHSS resists narrowband jammers; DSSS gives higher processing gain and data rate but is more complex to sync.
Q6 (4m). List the steps of traffic routing for a mobile-terminated call.
Caller → GMSC → query HLR → HLR returns MSRN from VLR → GMSC routes to visited MSC → MSC pages BTS → BTS pages MS → MS replies → traffic channel assigned → voice path established.
Q7 (8m). Compare FDMA, TDMA and CDMA in terms of resource sharing, sync, capacity and applications.
FDMA splits frequency (no sync, guard bands, 1G AMPS), TDMA splits time (strict sync, GSM 8 slots/200 kHz), CDMA splits codes (universal reuse N=1, soft capacity, soft handoff, IS-95/CDMA2000/GPS). Capacity ranking: CDMA > TDMA > FDMA per MHz. CDMA needs strong power control; FDMA is simplest. Comparison table + diagram of frequency-time-code partitioning earns full marks.
Q8 (8m). Explain ISDN — services, channels, BRI / PRI bandwidths and reference points.
ISDN delivers voice + data + signaling end-to-end in digital form over the existing copper loop. Three service classes: bearer (raw data pipe), tele (telephony, fax G3/G4), supplementary (caller ID, forwarding). Channels: B = 64 kbps user data, D = 16/64 kbps signaling, H = 384 kbps–1.92 Mbps bulk. BRI = 2B+1D = 144 kbps; PRI = 23B+1D (1.544 Mbps US) or 30B+1D (2.048 Mbps EU). Reference points R/S/T/U separate TE2-TA-NT2-NT1-LE in the subscriber loop.
