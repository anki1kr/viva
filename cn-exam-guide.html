Unit I

01Data Communication
Components

Signals, models, media and multiplexing — how raw data becomes a stream on a wire and back.

Data Communication & Representation ★ Part-A

Definition Data communication is the exchange of data between two devices through a transmission medium (a wire or wireless link). For it to work the devices must be part of a communication system made of hardware and software.

Its effectiveness rests on four fundamental characteristics:

  • Delivery: data must reach the correct, intended destination.
  • Accuracy: data must be delivered without corruption (error-free).
  • Timeliness: data must arrive in a usable time — for video/audio this means real-time.
  • Jitter: the variation in packet arrival time must be small (uneven delay ruins streaming).

Five Components of a Data Communication System

  • Message — the information (text, number, image, audio, video).
  • Sender — device that sends the message.
  • Receiver — device that receives it.
  • Transmission medium — physical path (cable, air).
  • Protocol — set of rules both devices agree on.

Data Representation

All data is sent as a stream of bits (0s and 1s). Text uses character codes (Unicode, 140,000+ characters; first 128 = ASCII). Numbers are converted directly to binary. Images are grids of pixels (grayscale 0–255, or RGB three-channel). Audio and video are sampled analog waves turned into bits.

Exam Tip A 2-mark "define data communication" answer = the definition + the four characteristics (Delivery, Accuracy, Timeliness, Jitter). The five components secure a 5-mark version.

Connection Topologies ⭐ Repeated 2×

Definition Topology is the physical or logical layout — the way devices (nodes) are connected by links.
  • Bus: all nodes share one backbone cable. Cheap, easy; one cable break kills the whole network.
  • Star: every node connects to a central hub/switch. Easy to manage; hub failure = total failure. Most common LAN today.
  • Ring: nodes form a closed loop; data travels node-to-node. Uses a token; one break can break the ring.
  • Mesh: every node links to every other node. Most reliable (many paths) but most cabling — n(n-1)/2 links for n nodes.
  • Hybrid: a mix (e.g. star-bus). Combines strengths; used in large enterprise networks.
Memory hook Mesh links = n(n−1)/2. Star = single point of failure at hub. Ring = token passing. Bus = one backbone.

Protocols & Standards

A protocol is a set of rules that governs data communication. Its three key elements are:

  • Syntax: the format/structure of the data (which bits mean what).
  • Semantics: the meaning of each section + what action to take.
  • Timing: when data is sent and how fast.

Standards ensure interoperability between vendors. De facto standards arise from common use; de jure standards are legislated by bodies like ISO, IEEE, ITU-T, and ANSI.

OSI Reference Model ⭐ Guaranteed 10M

OSI Model The Open Systems Interconnection (OSI) model by ISO is a 7-layer framework that standardises how two systems communicate. Each layer does a specific job and serves the layer above it.
Mnemonic (top→bottom) All People Seem To Need Data Processing — Application, Presentation, Session, Transport, Network, Data link, Physical.
Why it matters The 10-mark "draw OSI + function of each layer" is the single most common Unit-I long answer. See the full predicted answer (Q2a) for every layer's job + PDU + devices.

Transmission Media ⭐ Repeated 2×

The physical path that carries the signal. Split into guided (wired) and unguided (wireless).

Guided Media (wired)

  • Twisted Pair: two insulated copper wires twisted to cancel noise. UTP (unshielded, cheap, telephone/LAN) and STP (shielded). Cheapest, easiest; limited distance & bandwidth.
  • Coaxial Cable: central copper core + insulation + mesh shield. Higher bandwidth than twisted pair. Used in cable TV, older Ethernet.
  • Optical Fibre: carries light through a glass/plastic core by total internal reflection. Huge bandwidth, immune to EMI, long distance, secure. Costly and fragile. Used in backbones.

Unguided Media (wireless)

  • Radio waves: omnidirectional, penetrate walls (AM/FM, cordless).
  • Microwaves: line-of-sight, directional (cellular, satellite uplink).
  • Infrared: short range, cannot pass walls (TV remote).
Professor Pattern "What transmission media are used?" or "Explain guided/unguided media" appears almost every paper. For the 5M version: 3 guided + 3 unguided with one strength + one use each.

Multiplexing — FDM, TDM, WDM ⭐ Repeated 2×

Multiplexing Multiplexing lets several signals share one physical link, improving bandwidth utilization. A MUX combines them at the sender; a DEMUX separates them at the receiver.
  • FDM (Frequency-Division): the link bandwidth is split into frequency bands; each signal rides its own band at the same time. Analog. Guard bands prevent overlap. Used in radio, cable TV.
  • TDM (Time-Division): the link time is split into time slots; each signal gets the full bandwidth in its turn. Digital. Synchronous TDM (fixed slots) vs Statistical TDM (slots on demand). Used in digital telephony (T1).
  • WDM (Wavelength-Division): FDM applied to optical fibre — different wavelengths (colours) of light carried together, combined/split by prisms. Huge capacity. Used in fibre backbones (DWDM).
One-liner FDM = share frequency (analog). TDM = share time (digital). WDM = FDM with light on fibre.

Spread Spectrum — FHSS & DSSS

Spread Spectrum A technique that spreads a signal over a much wider bandwidth than it needs, to resist jamming, interference and eavesdropping. Used in wireless (Wi-Fi, Bluetooth, GPS).
  • FHSS (Frequency Hopping): the carrier rapidly hops between many frequencies in a pseudo-random sequence known to both sides. A jammer on one frequency only catches a fraction. Used in Bluetooth.
  • DSSS (Direct Sequence): each data bit is multiplied by a high-rate pseudo-random chip code, spreading it across the band. Receiver re-multiplies to recover. Used in older Wi-Fi (802.11b), GPS.
Unit II

02Data Link Layer &
MAC Sublayer

Catching errors, controlling flow, and sharing one channel between many senders.

Error Detection & Correction Fundamentals ⭐ Guaranteed 15M

Definition An error occurs when the bit pattern the receiver gets differs from what the sender sent — caused by noise on the channel flipping a 0↔1.

Types of Errors

  • Single-bit error: exactly one bit flipped. Rare in serial transmission.
  • Burst error: two or more consecutive (or close) bits flipped. Common — noise lasts longer than one bit time.

Error Detection Methods

  • Simple Parity Check: add one parity bit so the total number of 1s is even (even parity). Detects all single-bit and odd-bit errors; misses even-numbered errors.
  • 2-D Parity Check: parity for each row + each column. Detects and even corrects some single-bit errors.
  • Checksum: sum data segments using 1's complement, send the complemented sum. Receiver re-sums; all-zeros = correct.
  • CRC (Cyclic Redundancy Check): based on binary division by a generator polynomial — most powerful, catches almost all burst errors. (Full worked example in predicted Q4.)
Detection vs Correction Detection only tells you something is wrong (then ask for resend). Correction (e.g. Hamming code) finds which bit is wrong and flips it back — no resend needed.

Block Coding, Hamming Distance & Hamming Code ⭐ Repeated 2×

Block Coding In block coding we split the message into k-bit blocks (datawords) and add r redundant bits to make n = k + r-bit blocks (codewords). Only some codewords are valid; receiving an invalid one signals an error.
  • Hamming distance d(x, y): the number of bit positions in which two codewords differ. Found by XOR then counting the 1s. Example: d(10101, 11110) = 3.
  • Minimum Hamming distance dmin: the smallest distance over all valid codeword pairs.
  • To detect s errors: need dmin ≥ s + 1.
  • To correct t errors: need dmin ≥ 2t + 1.
Hamming Code A linear error-correcting code that places parity (redundancy) bits at power-of-2 positions (1, 2, 4, 8…) to pin-point and fix a single-bit error. Redundancy bits r satisfy 2r ≥ m + r + 1 for an m-bit dataword. (Full placement + detect/correct walkthrough in predicted Q4.)

Cyclic Redundancy Check (CRC)

CRC treats the dataword as a polynomial and divides it (modulo-2, i.e. XOR) by an agreed generator polynomial. The remainder is the CRC.

1
Append (k−1) zeros to the dataword, where k = length of the generator.
2
Divide the result by the generator using modulo-2 (XOR) division.
3
The remainder = the CRC (always k−1 bits).
4
Codeword = original data + CRC. Receiver divides by the same generator: remainder 0 = no error.
Exam Tip The CRC numerical is a classic 7–8M sub-part. Always show the long-division XOR steps, not just the answer. See predicted Q4.

Flow & Error Control — ARQ Protocols ⭐ Guaranteed 15M

Flow control vs Error control Flow control stops a fast sender from drowning a slow receiver. Error control (ARQ — Automatic Repeat reQuest) detects lost/damaged frames and re-sends them.
  • Stop-and-Wait: send one frame, wait for its ACK before sending the next. Simple but very slow (link idle while waiting). Window size = 1.
  • Stop-and-Wait ARQ: adds sequence numbers (0/1), a timer, and retransmission on timeout or NAK.
  • Sliding Window: sender may have up to N unacknowledged frames in flight at once — fills the pipe, far higher throughput.
  • Go-Back-N ARQ: window of N. On a lost/erroneous frame, sender re-sends that frame and all frames after it. Receiver discards out-of-order frames (window = 1 at receiver). Sender window ≤ 2m−1.
  • Selective Repeat ARQ: window of N at both ends. Only the specific damaged frame is re-sent; receiver buffers out-of-order frames. More efficient, more complex. Window ≤ 2m−1.

Piggybacking ⭐ Repeated 2×

Piggybacking In two-way (duplex) communication, instead of sending a separate ACK frame, the receiver attaches (piggybacks) the acknowledgement onto its next outgoing data frame.
  • Why used: a standalone ACK wastes a whole frame and bandwidth. Piggybacking carries the ACK free inside a data frame that was going out anyway → better channel utilization.
  • Trade-off: if no return data is ready, the receiver waits briefly; if nothing comes, it sends the ACK alone (a small added delay).
Professor Pattern "Why is piggybacking used?" is a near-certain Part-A. Answer: combines ACK with outgoing data → saves bandwidth + improves efficiency in duplex links.

Medium Access Control — ALOHA, CSMA/CD, CSMA/CA ⭐ Repeated 2×

MAC Problem When many devices share one channel (multiple-access link), MAC protocols decide who transmits when to minimise collisions. Random access = no fixed schedule; any station may send.

ALOHA

  • Pure ALOHA: a station sends a frame whenever it has data. If frames overlap = collision; both wait a random time and resend. Max throughput ≈ 18.4% (S = G·e−2G, peak at G=0.5).
  • Slotted ALOHA: time is divided into slots; a station may send only at the start of a slot. This halves the collision window. Max throughput ≈ 36.8% (S = G·e−G, peak at G=1).

CSMA Family — "listen before talk"

  • CSMA/CD (Collision Detection): sense the channel; if idle, transmit; while sending, keep listening — if a collision is detected, stop immediately, send a jam signal, back off (binary exponential). Used in wired Ethernet.
  • CSMA/CA (Collision Avoidance): collisions can't be detected on wireless, so avoid them — sense channel, wait a random backoff + IFS, use RTS/CTS handshake and ACKs. Used in Wi-Fi (802.11).
CD vs CA CD = wired, detects collision during send. CA = wireless, tries to avoid collision before sending (can't detect on radio).
Unit III

03Network Layer

Logical addressing, packet headers, and getting a packet across many hops to the right host.

Switching

Switching Switching is moving data from one device/network to another via intermediate nodes (switches/routers) that pick the path.
  • Circuit Switching: a dedicated path is reserved end-to-end before any data flows, held for the whole call (telephone network). Wastes bandwidth in idle gaps; no setup delay during data.
  • Packet Switching: data is split into packets routed independently as resources allow. Used by the Internet. Two flavours:
    • Datagram (connectionless): each packet routed on its own; may arrive out of order (IP).
    • Virtual Circuit (connection-oriented): a logical path is set up first; all packets follow it (ATM, Frame Relay).
  • Message Switching: whole message stored-and-forwarded node to node (obsolete).

IPv4 Packet (Datagram) Format ⭐ Repeated 2×

An IPv4 datagram has a 20-to-60 byte header + data. Key fields:

  • Version (4 bits): = 4 for IPv4.
  • HLEN (4 bits): header length in 4-byte words (5–15).
  • Type of Service / DSCP (8 bits): priority / QoS.
  • Total Length (16 bits): whole datagram length in bytes (max 65,535).
  • Identification, Flags, Fragment Offset: control fragmentation and reassembly.
  • Time To Live — TTL (8 bits): hop limit; decremented each router, packet dropped at 0 (kills loops).
  • Protocol (8 bits): upper-layer protocol (TCP=6, UDP=17, ICMP=1).
  • Header Checksum (16 bits): error check over the header only.
  • Source & Destination Address (32 bits each): sender and receiver IPs.
  • Options + Padding: optional, variable.
Exam Tip "Draw IPv4 header + explain fields" is a strong 8–10M. Full labelled diagram is in predicted Q6a.

Physical-to-Logical Mapping — ARP, RARP, BOOTP, DHCP ⭐ Repeated 2×

  • ARP (Address Resolution Protocol): finds the physical (MAC) address for a known IP address. Broadcasts "who has IP X?"; the owner replies with its MAC. IP → MAC.
  • RARP (Reverse ARP): the opposite — a diskless machine that knows only its own MAC asks a server "what is my IP?". MAC → IP. Obsolete, replaced by BOOTP/DHCP.
  • BOOTP (Bootstrap Protocol): gives a booting host its IP + server + gateway from a static table. Static, manual mapping. Predecessor of DHCP.
  • DHCP (Dynamic Host Configuration Protocol): dynamically leases IP addresses from a pool + gives subnet mask, gateway, DNS. Plug-and-play (the DORA process: Discover, Offer, Request, Ack). Runs over UDP.
Professor Pattern "Distinguish ARP and RARP" is a frequent Part-A — answer in a 2-row matrix: ARP maps IP→MAC, RARP maps MAC→IP. See predicted Q1(h).

Delivery, Forwarding & Unicast Routing

  • Delivery: how a packet is handed to the final host — direct (same network) or indirect (through routers).
  • Forwarding: a router's per-packet decision: look up the destination in the routing table, send out the right interface.
  • Routing: building those tables. Unicast routing protocols:
    • RIP (Distance Vector): uses hop count, Bellman-Ford, shares whole table with neighbours. Max 15 hops.
    • OSPF (Link State): each router floods link costs, builds full map, runs Dijkstra's shortest path. Scales better.
    • BGP (Path Vector): routing between autonomous systems — the protocol of the Internet backbone.
Unit IV

04Transport &
Application Layer

Process-to-process delivery, reliable TCP, lightweight UDP, and the services users actually touch.

Process-to-Process Communication & Ports

Definition The network layer delivers to a host (IP address). The transport layer delivers to the right process inside that host using a port number (16-bit). IP + port = a socket.
  • Well-known ports (0–1023): HTTP 80, HTTPS 443, FTP 20/21, DNS 53, SMTP 25, TELNET 23, SNMP 161.
  • Multiplexing/demultiplexing: many processes share one network link using distinct ports.

UDP — User Datagram Protocol ⭐ Repeated 2×

UDP A connectionless, unreliable transport protocol. No handshake, no ACKs, no ordering, no flow control — just a thin wrapper over IP. Fast and low-overhead.

UDP header is a fixed 8 bytes, four 16-bit fields:

  • Source Port (16 bits)
  • Destination Port (16 bits)
  • Total Length (16 bits) — header + data, in bytes
  • Checksum (16 bits) — optional error check

Used by: DNS, DHCP, TFTP, RIP, NTP, SNMP, VoIP, video streaming, online games — anything where speed beats reliability. (Header numerical in predicted Q7b.)

TCP — Transmission Control Protocol ⭐ Repeated 2×

TCP A connection-oriented, reliable, byte-stream transport protocol. Guarantees in-order, error-free, complete delivery using sequence numbers, ACKs, retransmission, flow control and congestion control.
  • 3-way handshake (connection setup): SYN → SYN+ACK → ACK.
  • Reliability: sequence numbers order bytes; cumulative ACKs confirm receipt; lost segments retransmitted on timeout.
  • Flow control: sliding window using the receiver's advertised window size.
  • Connection release: 4-way FIN/ACK exchange.
  • Header: 20–60 bytes (ports, seq no, ack no, flags SYN/ACK/FIN/RST, window, checksum).

Used by: HTTP/HTTPS, FTP, SMTP, TELNET — anything needing correctness. (Full operation + congestion control in predicted Q7a.)

Congestion Control & QoS

Congestion Too many packets in the network → routers' queues overflow → packets dropped, delay rises. Congestion control keeps the load below capacity.
  • Slow Start: TCP starts with a small congestion window (cwnd) and doubles it each RTT (exponential) until a threshold.
  • Congestion Avoidance (AIMD): after the threshold, cwnd grows by 1 each RTT (additive increase); on loss, cwnd is halved (multiplicative decrease).
  • QoS (Quality of Service): guarantees on bandwidth, delay, jitter, loss — via traffic shaping (leaky/token bucket), priority queues, reservation (RSVP). Critical for VoIP and video.

SCTP — Stream Control Transmission Protocol

A newer transport protocol combining the best of TCP and UDP. It is reliable + connection-oriented like TCP, but adds multi-streaming (multiple independent streams in one connection — no head-of-line blocking) and multi-homing (a host can use several IPs for failover). Used in telecom signalling (SS7 over IP).

Application-Layer Services — DNS, HTTP, FTP, EMAIL, TELNET, WWW

  • DNS (Domain Name System): the Internet's phone book — translates a domain name (google.com) into an IP address. Hierarchical (root → TLD → authoritative). Runs over UDP port 53. DDNS = Dynamic DNS, auto-updates records when an IP changes.
  • HTTP (HyperText Transfer Protocol): the request/response protocol of the WWW; stateless, runs over TCP 80 (HTTPS 443). Client sends GET/POST, server returns a status code + resource.
  • WWW: the web of hyperlinked documents accessed by URLs over HTTP, served by web servers, rendered by browsers.
  • FTP (File Transfer Protocol): transfers files over TCP using two connections — control (port 21) + data (port 20).
  • EMAIL: SMTP sends mail (push, port 25); POP3/IMAP retrieve it (pull).
  • TELNET: remote terminal login over TCP 23 (insecure, plaintext; replaced by SSH).

SNMP & Bluetooth ★ Short note

  • SNMP (Simple Network Management Protocol): lets a manager monitor and control network devices (routers, switches) via agents. Data is stored in the MIB (Management Information Base) addressed by object IDs. Runs over UDP 161/162. Operations: GET, SET, TRAP.
  • Bluetooth (IEEE 802.15.1): a short-range (~10 m) wireless PAN technology in the 2.4 GHz ISM band using FHSS (1600 hops/sec). Devices form a piconet (1 master + up to 7 active slaves); linked piconets form a scatternet. Low power, used for headsets, keyboards, IoT, file transfer.

Q2★★★ Both years15M
(a) Draw the OSI reference model and explain the function of each of the seven layers [10M]. (b) Explain connection topologies (bus, star, ring, mesh, hybrid) with a diagram [5M].
Introduction

The OSI (Open Systems Interconnection) model, given by ISO, is a 7-layer reference framework that standardises how two systems communicate. Each layer performs a defined job, adds its own header (encapsulation), and serves the layer above it. The lower 4 layers move data; the upper 3 support the application.

(a) Function of Each Layer
  • 1. Physical: transmits raw bits over the medium. Defines voltages, data rate, pin layout, cables, connectors, transmission mode. Devices: hub, repeater, cable.
  • 2. Data Link: turns bits into frames; adds physical (MAC) addresses; provides framing, error control, flow control, and medium access. Devices: switch, bridge, NIC.
  • 3. Network: delivers packets host-to-host across networks; handles logical (IP) addressing and routing. Device: router.
  • 4. Transport: provides end-to-end (process-to-process) delivery; segmentation + reassembly; reliability, flow & error control; uses port numbers. Protocols: TCP, UDP.
  • 5. Session: establishes, manages, and terminates sessions/dialogs between applications; adds synchronisation checkpoints.
  • 6. Presentation: translation (data formats, character codes), encryption/decryption, and compression.
  • 7. Application: the interface to the user; provides network services — HTTP, FTP, DNS, SMTP, TELNET.

Q3★★★ Both years15M
(a) Explain multiplexing — FDM, TDM and WDM with diagrams [10M]. (b) Explain spread spectrum (FHSS and DSSS) [5M].
Introduction

Multiplexing is the set of techniques that lets several signals share one physical link, maximising bandwidth utilization. A multiplexer (MUX) combines signals at the sender; a demultiplexer (DEMUX) separates them at the receiver. The three classic methods divide the link by frequency, by time, or by wavelength.

(a) The Three Multiplexing Techniques
  • FDM (Frequency-Division Multiplexing): available bandwidth is divided into non-overlapping frequency bands, one per signal. All signals travel simultaneously. Guard bands stop adjacent channels interfering. Analog technique. Use: AM/FM radio, analog TV, first-gen cellular.
  • TDM (Time-Division Multiplexing): the link's time is divided into slots; each signal uses the full bandwidth during its slot, sources interleaved round-robin. Digital technique. Synchronous TDM = fixed slots (wasted if idle); Statistical TDM = slots given on demand. Use: digital telephony (T1/E1), SONET.
  • WDM (Wavelength-Division Multiplexing): conceptually FDM for optical fibre — different wavelengths (colours) of light share one fibre. Combined and split by prisms/diffraction gratings. DWDM packs 100+ channels. Use: high-capacity fibre backbones.
(b) Spread Spectrum

Spread spectrum deliberately spreads a signal over a much wider bandwidth than it needs, trading bandwidth for resistance to jamming, interference, and interception. Two methods:

  • FHSS (Frequency Hopping Spread Spectrum): the carrier rapidly hops between many frequencies following a pseudo-random sequence shared by sender and receiver. A jammer fixed on one frequency catches only a fraction of the data. Used in Bluetooth (1600 hops/sec).
  • DSSS (Direct Sequence Spread Spectrum): each data bit is XORed/multiplied by a high-rate pseudo-random chip code, spreading it across the band. The receiver multiplies by the same code to recover the bit; noise spreads out and is filtered. Used in 802.11b Wi-Fi and GPS.

Examiner Trap WDM is not a fourth idea — it is FDM, just applied to light on fibre. Say so explicitly to earn the conceptual mark.
Modern Relevance

WDM/DWDM carries virtually all long-haul Internet traffic on undersea and terrestrial fibre in 2026. TDM lives on in mobile backhaul and SONET/SDH. Spread spectrum underpins every wireless link you use — Bluetooth (FHSS), GPS (DSSS), and modern Wi-Fi/5G evolve these ideas into OFDM and CDMA.

Conclusion

Multiplexing is how one expensive link serves many users — by frequency (FDM), time (TDM), or light (WDM). Spread spectrum then makes wireless links robust by trading bandwidth for security and noise immunity. Both are core bandwidth-utilization techniques of Unit I.

Explain error detection and correction. (a) For dataword 1001101 and generator 1011, compute the CRC and form the codeword (show the binary-division steps). (b) For dataword 1011, construct the Hamming (even-parity) code, then detect and correct a single-bit error introduced at position 5.
Introduction

Channel noise can flip bits in transit (0↔1). Error detection finds that a frame is corrupt (then re-send); error correction finds which bit is wrong and fixes it without a re-send. Both add redundancy bits to the data. CRC is the strongest detection method (binary division); the Hamming code is the classic single-error correcting code.

Types of Error & Detection Methods
  • Single-bit error: one bit flipped. Burst error: several consecutive bits flipped (more common).
  • Parity check: add one bit for even/odd 1-count — detects single & odd errors only.
  • Checksum: sum segments in 1's complement, send complement; receiver re-sums to zero.
  • CRC: divide by a generator polynomial; non-zero remainder = error. Catches all burst errors shorter than the generator.
(a) CRC — Worked Example

Given: dataword = 1001101, generator = 1011 (length k = 4). Steps:

1
Append k−1 = 3 zeros to the dataword → augmented = 1001101000.
2
Divide 1001101000 by 1011 using modulo-2 (XOR) division — XOR with the generator when the leading bit is 1, else with 0000, then bring the next bit down.
      1001101000  ÷ 1011   (mod-2)

   1001  XOR 1011 = 0101  -> bring 1 -> 1010
   1010  XOR 1011 = 0001  -> bring 0 -> 0011  ... shown step-wise below

At the receiver: divide 1001101101 by 1011. If the remainder is 000, the frame is accepted; any non-zero remainder means an error → discard.

(b) Hamming Code — Single-Error Correction

Given: dataword = 1011 (m = 4 data bits). Find redundancy bits r from 2r ≥ m + r + 1: try r = 3 → 23 = 8 ≥ 4 + 3 + 1 = 8 ✓. So total codeword length n = 7. Parity bits sit at power-of-2 positions 1, 2, 4; data bits fill 3, 5, 6, 7.

1

p1 covers positions 1,3,5,7 → p1 = 1⊕0⊕1 = 0
p2 covers positions 2,3,6,7 → p2 = 1⊕1⊕1 = 1
p4 covers positions 4,5,6,7 → p4 = 0⊕1⊕1 = 0

Introduce error at position 5 (0 → 1). Received word (pos 1→7) = 0 1 1 0 1 1 1.

Recompute check bits at the receiver (even parity over each group):

Q5★★★ Both years15M
(a) Explain the flow/error-control protocols — Stop-and-Wait, Go-Back-N ARQ and Selective Repeat ARQ — with sliding-window diagrams [10M]. (b) Compare Pure ALOHA and Slotted ALOHA, including their throughput (18.4% vs 36.8%) [5M].
Introduction

Flow control stops a fast sender from overwhelming a slow receiver; error control (ARQ — Automatic Repeat reQuest) recovers lost or corrupted frames by retransmission. The three classic protocols trade simplicity for efficiency: Stop-and-Wait (one frame at a time) → Go-Back-N and Selective Repeat (a sliding window of many frames in flight).

(a) The Three ARQ Protocols
1 · Stop-and-Wait ARQ

The sender transmits one frame, then waits for its ACK before sending the next. Sequence numbers alternate 0/1; a timer triggers retransmission on loss. Window size = 1. Simple but very inefficient — the link sits idle during the round-trip wait.

2 · Go-Back-N ARQ

A sliding window of N frames may be sent without waiting. The receiver accepts frames strictly in order (its window = 1) and sends cumulative ACKs. On a lost/erroneous frame, the sender re-sends that frame and all frames after it — even ones the receiver discarded. Sender window ≤ 2m−1 (m = seq-number bits).

3 · Selective Repeat ARQ

Both sender and receiver keep a window of N. The receiver buffers out-of-order frames and ACKs each frame individually (NAK for missing ones). The sender re-sends only the specific damaged frame, not the whole tail. More bandwidth-efficient but needs receiver buffering + sorting. Window ≤ 2m−1.

(b) Pure ALOHA vs Slotted ALOHA

Both are random-access MAC protocols. The difference is when a station may start sending.

Modern Relevance

TCP itself is a sliding-window protocol — Selective Repeat ideas live in TCP SACK (selective acknowledgement) used by every browser in 2026. Slotted-ALOHA-style random access underpins RFID, satellite uplinks, and the initial random-access channel (RACH) of 4G/5G cellular.

Conclusion

ARQ protocols climb from Stop-and-Wait's safe-but-slow single frame to Selective Repeat's efficient buffered window — the trade is complexity for throughput. ALOHA shows the cost of uncoordinated access: even slotting only reaches 36.8% utilization, which is why scheduled and carrier-sense protocols (CSMA) replaced it on busy channels.

Introduction

The IPv4 datagram is the basic unit of the network layer. It has a 20–60 byte header followed by data. The header carries everything routers need to forward, fragment, and error-check the packet. Classless (CIDR) addressing then uses a /n prefix to carve the address space into right-sized blocks.

(a) IPv4 Header Format
  • Version (4): IP version = 4.
  • HLEN (4): header length in 4-byte words (min 5 = 20 bytes, max 15 = 60 bytes).
  • Type of Service / DSCP (8): priority & QoS handling.
  • Total Length (16): entire datagram (header + data) in bytes; max 65,535.
  • Identification, Flags, Fragment Offset: control fragmentation & reassembly of large packets.
  • TTL (8): hop limit; each router decrements it; dropped at 0 (prevents loops).
  • Protocol (8): upper-layer protocol — TCP = 6, UDP = 17, ICMP = 1.
  • Header Checksum (16): error check over the header only; recomputed at each hop.
  • Source & Destination Address (32 each): sender and receiver IPv4 addresses.
  • Options + Padding: optional fields (record route, timestamp), padded to a 4-byte boundary.
(b) Worked Subnetting — 167.199.170.82/27

The /27 prefix means the first 27 bits are the network part and the last 5 bits (32 − 27) are the host part.

1

2

IP last octet 82 = 0101 0010

Q7★★★ Both years15M
(a) Explain TCP — its operation, the three-way handshake, reliability mechanisms, and congestion control (slow start & AIMD) [10M]. (b) Explain UDP and the user datagram; for the UDP header hex CB84 000D 001C 001C find the source port, destination port, total length and data length. Add short notes on DNS, SNMP and Bluetooth [5M].
Introduction

The transport layer offers two main protocols. TCP is connection-oriented and reliable — it guarantees ordered, error-free delivery. UDP is connectionless and unreliable — fast and lightweight. The choice is a trade-off between reliability and speed.

(a) TCP — Operation

TCP provides a reliable, in-order byte-stream between two processes. It segments the stream, numbers every byte, and uses acknowledgements + retransmission to guarantee delivery.

Three-Way Handshake (connection setup)
Reliability Mechanisms
  • Sequence numbers: every byte is numbered so the receiver can reorder and detect gaps.
  • Acknowledgements (cumulative): the receiver ACKs the next byte it expects.
  • Retransmission: a timer (RTO) re-sends unacknowledged segments; fast-retransmit triggers on 3 duplicate ACKs.
  • Checksum: detects corrupted segments.
  • Flow control: the receiver's advertised window limits how much the sender may transmit (sliding window).
Congestion Control — Slow Start & AIMD
  • Slow Start: the sender starts with a tiny congestion window (cwnd = 1 MSS) and doubles it every RTT (exponential growth) until it hits the threshold ssthresh.
  • Congestion Avoidance (AIMD): after ssthresh, cwnd grows linearly (+1 MSS per RTT — additive increase). On a packet loss, cwnd is halved (multiplicative decrease) and the cycle restarts. This "saw-tooth" probes for the most bandwidth the network will allow without collapsing.
(b) UDP & Header Numerical

UDP is connectionless and unreliable: no handshake, no ACK, no ordering. Its header is a fixed 8 bytes = four 16-bit fields (Source Port, Destination Port, Length, Checksum). Each 16-bit field = 4 hex digits.

Given header: CB84 000D 001C 001C — split into four 4-hex-digit fields and convert each to decimal:

1
Source Port = CB8416 = (12×4096)+(11×256)+(8×16)+4 = 52100.
2
Destination Port = 000D16 = 13 (the Daytime service port).
3
Total Length = 001C16 = 28 bytes (header + data).
4
Data Length = Total Length − Header (8 bytes) = 28 − 8 = 20 bytes. (The last 001C is the Checksum field.)

Short Notes
  • DNS: Domain Name System — translates domain names to IP addresses. Hierarchical (root → TLD → authoritative); runs over UDP 53. DDNS auto-updates records for changing IPs.
  • SNMP: Simple Network Management Protocol — a manager monitors/controls devices via agents; data stored in the MIB. Operations: GET, SET, TRAP. Runs over UDP 161/162.
  • Bluetooth (802.15.1): short-range (~10 m) 2.4 GHz wireless PAN using FHSS; devices form a piconet (1 master + up to 7 slaves), linked piconets form a scatternet. Used in headsets, IoT.

Conclusion

TCP buys reliability with handshakes, sequence numbers, ACKs and AIMD congestion control; UDP trades all of that for speed and an 8-byte header (here: ports 52100/13, length 28, data 20). The right choice depends on whether the app needs correctness (TCP) or low latency (UDP) — the central decision of the transport layer.

⚡ Part-A 1.5M Quick-Fire Drill

Ten one-line ready answers — memorise the bold phrase, drop it straight into the answer booklet.

Data communication: exchange of data between two devices over a medium under a protocol. Key chars: delivery, accuracy, timeliness, jitter.
Analog vs Digital: analog = continuous infinite values (voice); digital = discrete 0/1, noise-immune (computers).
Piggybacking: attach the ACK to an outgoing data frame instead of sending a separate ACK → saves bandwidth in duplex links.
Hamming code: error-correcting code; parity at positions 1,2,4,8 locates and fixes a single flipped bit (no resend).
ALOHA: random-access MAC; send when ready, resend after collision. Slotted ALOHA only sends at slot start (doubles throughput).
FDM vs TDM: FDM shares frequency bands (analog); TDM shares time slots (digital).
ARP vs RARP: ARP maps IP→MAC; RARP maps MAC→IP (diskless host finds its own IP).
DNS: translates domain names → IP addresses; hierarchical; runs over UDP port 53.
IPv4 classes: A 0–127, B 128–191, C 192–223, D 224–239 (multicast), E 240–255 (reserved).
TCP vs UDP: TCP = reliable, connection-oriented, 20–60B; UDP = fast, connectionless, 8B fixed header.
🎯 If You Remember Only 5 Things

The five non-negotiables. If brain freeze hits, hold on to these.

1. OSI 7-Layer Model
App, Pres, Session, Transport, Network, Data link, Physical (top→down). Know each layer's job + PDU. The most common 10–15M long answer.
2. CRC + Hamming Numericals
CRC: append (gen−1) zeros, mod-2 divide, remainder = CRC. Hamming: parity at 1,2,4,8; syndrome points to the bad bit. Show every step.
3. ARQ + ALOHA
Stop-and-Wait (window 1) → Go-Back-N (resend tail) → Selective Repeat (resend one). Pure ALOHA 18.4%, Slotted 36.8%.
4. IPv4 Header + Subnetting
Header: Version, HLEN, Total Length, TTL, Protocol, Checksum, Src/Dst IP. /n → 2(32−n) addresses; network = IP AND mask, broadcast = host bits all 1.
5. TCP vs UDP
TCP = reliable, 3-way handshake, slow-start/AIMD congestion control. UDP = 8-byte header, fast (data = total − 8). Hex field = 4 hex digits.
