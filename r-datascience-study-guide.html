1.4 Supervised vs Unsupervised Learning

Supervised Learning

Trained on labelled data (input → known output). Goal: predict the label for new data.

Tasks: Classification, Regression

Algorithms: Linear Regression, Logistic Regression, Decision Tree, KNN, SVM, Random Forest

Example: Spam vs Not-Spam (you have labels)

Unsupervised Learning

Trained on unlabelled data. Goal: discover hidden structure / groups.

Tasks: Clustering, Association, Dimensionality Reduction

Algorithms: K-Means, Hierarchical Clustering, Apriori, PCA

Example: Group customers into segments without knowing them in advance

Quick test If the dataset has a "correct answer" column → Supervised. If it doesn't → Unsupervised.

02 R Environment & CRAN UNIT 3

Before writing R code you need to know what R is, how to extend it with packages, and what CRAN is.

2.1 What is R?

Definition R is an open-source programming language and environment specifically built for statistical computing and graphics. Created by Ross Ihaka and Robert Gentleman in 1993.

Why R for Data Science? Free, huge package ecosystem, built-in stats, excellent plotting (ggplot2), strong community.

2.2 CRAN — Comprehensive R Archive Network

Definition CRAN stands for Comprehensive R Archive Network. It is the official network of FTP and web servers around the world that store identical, up-to-date versions of code, packages, and documentation for R.

Purpose of CRAN

  • Distributes the R language itself
  • Hosts 20,000+ contributed packages
  • Provides documentation, manuals, and FAQs
  • Acts as a quality gate — submitted packages are checked before publication

2.3 Installing & Loading Packages

A package is a bundle of R code, data, and documentation that adds extra functions. You install it once, then load it every session.

Step-by-step

# 1. Install (only once, needs internet)
install.packages("ggplot2")

# 2. Load (in every R session you want to use it)
library(ggplot2)
# or
require(ggplot2)

# 3. Use a function from the package
qplot(mtcars$mpg, mtcars$hp)

# 4. Check installed packages
installed.packages()

# 5. Update everything
update.packages()

# 6. Remove a package
remove.packages("ggplot2")
Tip — install vs library install.packages() downloads from CRAN (do once). library() loads it into the current session (do every time).
Memory hook "Install once, library always" — quotes are needed in install.packages("x") but not inside library(x).

3.1 Vectors

Definition A vector is a 1-dimensional sequence of elements of the same type (numeric, character, logical, etc.). It is the most basic R data structure. 
# Numeric vector
v1 <- c(10, 20, 30, 40)

# Character vector
v2 <- c("R", "Python", "Julia")

# Logical vector
v3 <- c(TRUE, FALSE, TRUE)

# Sequence helpers
1:10                    # 1 to 10
seq(2, 20, by = 2)        # even numbers
rep(1:3, times = 2)       # 1 2 3 1 2 3

# Indexing — 1-based!
v1[2]              # 20
v1[2:4]            # 20 30 40
v1[v1 > 15]         # logical filter — 20 30 40

# Useful operations
length(v1)        # 4
sum(v1)           # 100
mean(v1)          # 25
sort(v1)          # ascending sort

3.2 Matrices

Definition A matrix is a 2-D arrangement of elements of the same type, organised in rows and columns. 
# Create a 3x3 matrix, filled column-wise (default)
m <- matrix(1:9, nrow = 3, ncol = 3)
print(m)
#      [,1] [,2] [,3]
# [1,]   1    4    7
# [2,]   2    5    8
# [3,]   3    6    9

# Fill row-wise
m2 <- matrix(1:9, nrow = 3, byrow = TRUE)

# Access
m[2, 3]      # row 2, col 3 → 8
m[1, ]        # entire row 1
m[, 2]        # entire col 2

# Operations
dim(m)        # 3 3
t(m)          # transpose
m + 10        # element-wise
m %*% t(m)   # matrix multiplication

4.3 switch()

Definition switch(expr, case1 = ..., case2 = ..., default) — evaluates expr and returns the value of the matching named case. Cleaner than long if-else chains. 
day <- "Mon"

result <- switch(day,
    "Mon" = "Start of week",
    "Wed" = "Mid week",
    "Fri" = "Almost weekend",
    "Sat" = , # falls through to next
    "Sun" = "Weekend!",
    "Unknown day"          # default
)
print(result)        # "Start of week"

05 Object-Oriented Programming in R — S3 vs S4 UNIT 3

R supports several OOP systems. The course focuses on the two classic ones: S3 (informal) and S4 (formal).

5.1 The S3 System

Definition S3 is R's original, informal object system. A class is just an attribute on an object. Methods are dispatched on the first argument's class via a generic function. 
# Create an S3 object — set the class attribute
emp <- list(name = "Asha", salary = 50000)
class(emp) <- "Employee"

# Define a method via the <generic>.<class> convention
print.Employee <- function(x, ...) {
    cat("Employee:", x$name, "\nSalary :", x$salary, "\n")
}

print(emp)            # dispatches to print.Employee()

5.2 The S4 System

Definition S4 is a formal, stricter system. You must declare the class with setClass(), fields are typed (called slots), and methods are defined with setGeneric() + setMethod(). 
# 1. Define the class with typed slots
setClass("Employee",
    representation(name = "character",
                   salary = "numeric"))

# 2. Create an object
e <- new("Employee", name = "Asha", salary = 50000)
e@name              # access slot with @

# 3. Define a generic + method
setGeneric("showInfo", function(x) standardGeneric("showInfo"))

setMethod("showInfo", "Employee", function(x) {
    cat("Name :", x@name, "\nSalary:", x@salary, "\n")
})

showInfo(e)

5.3 S3 vs S4 — Side-by-Side

AspectS3S4
StyleInformal, lightweightFormal, strict
Class definitionJust a class() attributeDeclared with setClass()
Slot/field access$@
Type checkingNoneYes (slot types enforced)
Method definitiongeneric.class <- function(...)setMethod()
Object creationclass(x) <- "C"new("C", ...)
Multiple dispatchNoYes
Used byMost base R packagesBioconductor, Matrix, formal pkgs
One-liner "S3 = Simple, S4 = Strict."

Quick Quiz — Topic 5

Q1. Which symbol accesses a slot in S4?
The @ symbol (S3 uses $).
Q2. Which OOP system enforces type checking on fields?
S4 — slot types are declared in setClass() and validated.

06 Data Handling — Frames, Filtering, Grading UNIT 4

The bread-and-butter of R: building a data frame, adding new columns (like a Grade column), filtering rows, and computing summaries.

6.1 Creating & Modifying a Data Frame

# Build the frame
students <- data.frame(
    Name  = c("Asha", "Ravi", "Meera", "Sahil", "Ria"),
    Roll  = 101:105,
    Marks = c(85, 42, 93, 68, 35)
)

# Add a column
students$Status <- ifelse(students$Marks >= 40, "Pass", "Fail")

# Modify (apply 5-mark bonus)
students$Marks <- students$Marks + 5

# Remove a column
students$Status <- NULL

# Add a row
students <- rbind(students,
                  data.frame(Name = "Vivek", Roll = 106, Marks = 77))

6.2 Adding a Grade Column (loop + condition)

Classic exam program — produce a Grade for each student.

students$Grade <- ""            # initialise empty column

for (i in 1:nrow(students)) {
    m <- students$Marks[i]
    if      (m >= 90) students$Grade[i] <- "A+"
    else if (m >= 75) students$Grade[i] <- "A"
    else if (m >= 60) students$Grade[i] <- "B"
    else if (m >= 40) students$Grade[i] <- "C"
    else              students$Grade[i] <- "Fail"
}
print(students)

Vectorised alternative (no loop)

students$Grade <- cut(students$Marks,
    breaks = c(-Inf, 40, 60, 75, 90, Inf),
    labels = c("Fail", "C", "B", "A", "A+"),
    right  = FALSE)

07 Statistics & Visualization (ggplot2) UNIT 4

Numbers tell only half the story — pictures finish it. We start with central tendency, frequency, correlation, then build charts in ggplot2.

7.1 Measures of Central Tendency

MeasureDefinitionR function
MeanArithmetic average — sum / countmean(x)
MedianMiddle value after sortingmedian(x)
ModeMost frequently occurring valueNo built-in — use table()
 
x <- c(2, 4, 4, 5, 7, 9, 9, 9)

mean(x)               # 6.125
median(x)             # 6

# Mode — custom function
get_mode <- function(v) {
    u <- unique(v)
    u[which.max(tabulate(match(v, u)))]
}
get_mode(x)           # 9
When to use what? Symmetric data → Mean. Skewed data (income, prices) → Median. Categories → Mode.

7.3 Histogram — Concept & Use

Definition A histogram is a graph that displays the frequency distribution of a continuous numeric variable by grouping values into bins and drawing bars whose height = frequency.

Use of histogram:

  • See the shape of a distribution (normal, skewed, uniform)
  • Spot outliers and gaps
  • Decide if a transformation is needed before modelling
  • Compare spread & central tendency visually
# Base R
marks <- c(35, 42, 68, 85, 93, 77, 55, 61, 88, 72)
hist(marks,
     main = "Marks distribution",
     xlab = "Marks",
     col  = "steelblue",
     breaks = 5)

7.4 Correlation Basics

Definition Correlation measures the strength and direction of the linear relationship between two numeric variables.

Correlation coefficient r ranges from -1 to +1:

ValueMeaning
+1Perfect positive linear (both rise together)
+0.7 to +1Strong positive
0No linear relationship
-0.7 to -1Strong negative (one rises, other falls)
-1Perfect negative linear
 
height <- c(150, 160, 170, 180, 190)
weight <- c(50,  60,  65,  75,  85)

cor(height, weight)              # Pearson by default
cor(height, weight, method = "spearman")
cor.test(height, weight)         # with p-value

# Quick visual check
plot(height, weight)
Pitfall Correlation ≠ Causation. Two things can rise together by coincidence (ice-cream sales & drowning rates both rise in summer — heat causes both).

Q3. Add a "Grade" column to a student data frame using a for loop and if-else logic.

    roll_no = c(101, 102, 103, 104, 105),
    name    = c("Asha", "Ravi", "Meera", "Sahil", "Ria"),
    marks   = c(85, 42, 93, 68, 35)
)

# 2. Initialise empty Grade column
students$Grade <- ""

# 3. Loop over each row
for (i in 1:nrow(students)) {
    m <- students$marks[i]
    if      (m >= 90) students$Grade[i] <- "A+"
    else if (m >= 75) students$Grade[i] <- "A"
    else if (m >= 60) students$Grade[i] <- "B"
    else if (m >= 40) students$Grade[i] <- "C"
    else              students$Grade[i] <- "Fail"
}

4     104 Sahil    68     B

Logic: The for loop visits each row index; the chained if / else if picks the first matching threshold (highest first); the result is written to students$Grade[i]. Order matters — checking ≥ 90 before ≥ 75 ensures A+ is not overwritten by A.

Q5. Write a complete R program that creates a student data frame, adds a Grade column, filters distinction students, and prints the class mean. (4 + 4)

Full Program 
# 1. Create the data frame
students <- data.frame(
  RollNo = 1:6,
  Name   = c("Asha", "Ben", "Cara",
             "Dev", "Esha", "Faiz"),
  Marks  = c(88, 42, 75, 30, 65, 95)
)

# 2. Add Grade column using a loop + if/else
grade <- character(nrow(students))
for (i in 1:nrow(students)) {
  m <- students$Marks[i]
  if (m >= 75)      grade[i] <- "A"
  else if (m >= 60) grade[i] <- "B"
  else if (m >= 40) grade[i] <- "C"
  else               grade[i] <- "Fail"
}
students$Grade <- grade

# 3. Filter — keep only Distinction (Marks > 80)
distinction <- students[students$Marks > 80, ]

# 4. Class mean & print
cat("Class mean :", mean(students$Marks), "\n")
print(students)
cat("\nDistinction students:\n")
print(distinction)

Common pitfalls in the exam hall
  • Writing = instead of <- for assignment — both work but examiners prefer <-.
  • Forgetting the comma inside data-frame indexing: df[df$x > 5, ] not df[df$x > 5].
  • Confusing $ (S3 / data frame) with @ (S4 slots).
  • Saying "R has a built-in mode() function" — it doesn't (the built-in mode() returns the storage mode, not the statistical mode).
  • Mixing up lapply (returns list) and sapply (returns vector).
